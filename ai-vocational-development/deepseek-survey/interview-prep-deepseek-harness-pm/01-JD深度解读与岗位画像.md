# 01 · JD 深度解读与岗位画像

> 目标：把 JD 从"招聘话术"还原成"团队真实需求"，挖出**显性能力 / 隐藏信号 / 反向问题**，并对应到你能讲的故事。

## 一、JD 全文（DeepSeek Agent Harness 产品经理）

> 你将加入 Harness 团队，与工程师、研究员紧密协作，参与 DeepSeek 桌面端 Agent 产品的全过程，定义 DeepSeek 对 Harness 的理解。
>
> **主要职责**
> 1. 规划 DeepSeek Harness 产品路线图，连接研究员、工程师、开源社区和广大用户。
> 2. 理解判断用户的最真实需求，定义与衡量"Agent 是否真的在更多场景下更深入的帮助到更多的人"的指标。
> 3. 与模型训练团队的研究员深度沟通与合作，实现模型与 Harness 的共同进化。
> 4. 帮助 Harness 产品内部落地，以内部真实任务做为 Harness 产品和模型相关能力训练的重要反馈源，持续迭代产品能力。
> 5. 维护 Harness 产品用户社群，从潜在海量用户群体中获取反馈、提取信号，指导产品迭代。
> 6. 协助项目管理相关工作。
>
> **任职要求**
> 1. 2 年以上产品经理从业经验，产品逻辑清晰。特别优秀候选人可放宽年限。
> 2. 知名高校本科学历及以上。
> 3. 能够使用 vibe coding 写代码，不一定需要技术背景。
> 4. 能够设计系统性的收集数据的方法（包括问卷、访谈、A/B 测试、灰度测试等），并使用统计学的工具严谨科学的分析数据。对此有系统性的思考和实践。
> 5. 是 Agent 产品的高强度用户，熟悉 Agent 产品的各种形态，对 Agent 有极大的热情，对模型行为有品味有判断力，对开发者体验及用户体验有强感知。深度使用过 Claude Code、Cowork、Codex、Cursor、OpenCode、GitHub Copilot、Manus、OpenClaw、Hermes 等类似产品，并将相关产品的使用融入到自己的工作和生活中。
> 6. 理解 LLM 以及 Agent 基本机制及其技术原理，包括 LLM API、KV Cache、Agent Loop、Tool Use、Reasoning、Planning、Skills、MCP、Memory、Subagent、Multi-Agent 等相关知识。对 Prompt Engineering、Context Engineering、Harness Engineering 等课题有第一手实践。
> 7. 具备 UI/UX 设计素养。能够在 AI 辅助下完成产品原型图设计、UI 设计等相关的产品设计工作。
> 8. 优秀的中文沟通能力。能用英文与开源社区、用户社群书面沟通。
>
> **加分项**
> - AI 行业从业经验、AI 相关产品经验。
> - 作为小团队中的产品经理主导产品路线的经验。
> - 与研究员深度协作的经验。
> - 深度参与开源社区或维护开源社区关系的经验。
> - 其它超乎常人的与此工作相关的才能。

## 三、主要职责逐句拆解

### 职责 1：规划 DeepSeek Harness 产品路线图，连接研究员、工程师、开源社区和广大用户。

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 写 1 年期 Roadmap、季度 OKR、版本规划、对外发版节奏；同时是开源/社区策略的 owner |
| **隐藏要求** | "连接"四方 = 你是 Hub，没有独立 KPI，能不能让其他人愿意听你的，看你的影响力 |
| **考察点** | 系统性思考、优先级判断、外部沟通（中英双语） |
| **你的故事素材** | 主导过一个跨 N 个团队的产品方向；社区/外部生态运营案例 |

### 职责 2：理解判断用户的最真实需求，定义与衡量"Agent 是否真的在更多场景下更深入的帮助到更多的人"的指标。

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 设计**北极星指标 + 二级指标 + 反向指标**；建数据看板；定 PMF 信号 |
| **隐藏要求** | 拆解关键词：「更多场景」「更深入」「更多人」——三轴：**广度 × 深度 × 用户数**。能不能想到比"DAU/留存"更适合 Agent 的指标？（见文档 13） |
| **考察点** | 指标体系设计、对 Agent 形态差异的理解 |
| **你的故事素材** | 你为一个产品定义过什么指标？为什么是这个？怎么落地的？踩过哪些坑（比如 Goodhart's Law）？ |

> **重点**：这是 JD 里最具体、最考"硬功夫"的一句，准备一个专门的回答（见文档 04-Q3）。

### 职责 3：与模型训练团队的研究员深度沟通与合作，实现模型与 Harness 的共同进化。

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 把用户反馈翻译成训练数据 / 奖励信号 / eval set；把模型新能力翻译成产品 feature |
| **隐藏要求** | "共同进化"= 你不是把需求扔给研究员，你是和研究员一起设计**Harness ↔ Model 的接口契约**（哪些事 Harness 兜底、哪些事让模型学会） |
| **考察点** | 能不能用研究员能听懂的话讲产品；懂不懂 SFT/RLHF/DPO 这套词汇 |
| **你的故事素材** | 和算法/研究员合作的真实项目；学会用他们的语言；曾经设计过什么 eval pipeline |

### 职责 4：帮助 Harness 产品内部落地，以内部真实任务做为 Harness 产品和模型相关能力训练的重要反馈源，持续迭代产品能力。

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 推 DeepSeek 内部工程师/研究员先用自己家产品做活儿（"吃狗粮"）；建反馈渠道；做内部分析 |
| **隐藏要求** | 你能不能"骚扰"高 P 的研究员让他们填问卷、参加访谈；能不能让他们的不爽变成训练信号 |
| **考察点** | 内部影响力 / 社群运营手感 / 数据收集严谨度 |

### 职责 5：维护 Harness 产品用户社群，从潜在海量用户群体中获取反馈、提取信号，指导产品迭代。

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | Discord/Slack/小红书/Twitter/微信群 多渠道运营；客服话术；KOL 关系 |
| **隐藏要求** | "海量"+"提取信号"=你要会**聚类、摘要、抽样**用户反馈（典型用 LLM 自动化处理 issue/tweet） |
| **考察点** | 海外社区经验 + 中文用户群 + 自动化处理 |

### 职责 6：协助项目管理相关工作。

| 维度 | 解读 |

## 五、JD 里没写但一定会考的"暗线"

### 暗线 1：你是不是"模型品味派"
JD 写"对模型行为有品味有判断力"——这个能力非常稀缺。陈德里这种研究员背景的 leader 会用 5 分钟测出你是不是"嘴上说用过"。

**测法**：让你现场点评某个产品的"系统提示词"，或者描述你最近一次被模型"骗到"的体验。

**应对**：每个主流 Agent 产品至少有 1 个具体可复述的"行为印象"。范例：
- "Claude Code 一旦你说 'no'，它就会快速换方向，不像 Codex 那种会跟你 argue 三轮；这反映出 Anthropic 在 SFT 时强化了 user-deference。"
- "Cursor 的 Agent Mode 在你跨文件让它做事时，会优先用 grep 而不是直接 read，是个明显的 token 节省 trick——但代价是它有时会漏掉只有 read 全文才能看到的细节。"

### 暗线 2：你和"研究员"能不能聊天
DeepSeek 是 research-first 文化。如果你说不出 SFT/DPO/eval/reward hacking/capability elicit/sycophancy/jailbreak 这些词，研究员面试官 2 分钟就能闻出来。

### 暗线 3：你 ship 不 ship
小团队最讨厌"很会聊不出活"。一定要带能跑起来的东西：
- 一个开源 repo（哪怕只有 100 行）
- 一份产品提案（见文档 12）
- 一份评测脚本 / leaderboard
- 一篇深度评测文（中或英）

### 暗线 4：你是不是"长期主义者"
DeepSeek 不发奖金不赛马，靠认同感留人。面试官会试探你的动机：
- 你为什么不去字节腾讯（钱更多）/ MiniMax 月之暗面（同梯队）？
- 你看 5 年后的 DeepSeek 长什么样？

**准备答案**："我看好 DeepSeek 把模型 + Harness 做成开源 community 的能力，这是中国能复制 Anthropic Labs 那种 research-driven product 的少数几支团队之一。"
