# 08 · 反问环节话术库

> 反问环节不是"礼节"，是 PM 候选人能拉差距的最后 5 分钟。**高质量反问 = 第二轮面试**。

## 核心原则

1. **每个面试官准备 2~3 个差异化问题**——不要对所有人问同一个
2. **问题要让对方"想答" + "答完愿意继续聊"**——这是双向选择
3. **绝不问百度能查到的事**（薪资、地点、上下班时间——HR 环节集中问）
4. **优先问 "听得到对方的判断而不只是事实" 的问题**
5. **每轮带 1~2 个"反向送球"问题**——给面试官炫他/她最近做对的事的机会

---

## 一、按面试官类型分

### 给 HR / 人力（first round）

**目的**：摸清流程、团队、对你这位候选人的初印象

1. "我看到 JD 强调'桌面端 Agent 产品'，这是 DeepSeek 第一次提到桌面端形态——是否能透露团队目前对桌面端的优先级，是与现有 Web 形态并行还是会替代？"
2. "Harness 团队刚组建，团队规模和未来 6~12 个月的招聘节奏大概是怎样的？"
3. "面试有几轮、分别由什么角色面？我想为后续轮次准备得更具体。"
4. "如果我顺利入职，团队 onboarding 一般是什么节奏？前 30 天会希望我做什么？"
5. "您在 HR 这边接触的几位候选人，您觉得这个岗位最稀缺的能力是什么？"
6. "DeepSeek 不设 KPI，那 PM 是怎样获得反馈和升迁信号的？"

### 给 PM / 产品经理（peer interviewer）

**目的**：了解协作模式、产品哲学、当前痛点

1. "DeepSeek 现在有几位 PM？分线还是平行？跨 PM 之间的协作机制是什么样的？"
2. "如果我加入，第一个 sprint 最有可能和我撞车的同事会是谁？我们的边界怎么划？"
3. "团队过去 3 个月最骄傲的一个产品决策是什么？最遗憾的呢？"
4. "PM 这边对 Harness 的指标体系目前有共识吗？还是各 PM 自定？"
5. "你们怎么看 Cursor / Claude Code / Codex 这三个对标？最想抄的是哪个，最想避开的是哪个？"
6. "DeepSeek 内部用 PM 工具栈（Notion / Linear / Coda）是什么样？这件事会影响我入职后的工作节奏。"

### 给研究员 / 算法（最关键的一轮）

**目的**：证明你和研究员能对话，并真心想协作

> 这一轮的反问比任何回答都重要。研究员能从你的提问判断你是不是"懂行"。

1. "您怎么看 Anthropic 11 月那篇 long-running harness 和 3 月那篇 GAN-style harness 的演进？您觉得 DeepSeek 起步要从两段式还是直接三段式？"
2. "DeepSeek 模型在 tool_use schema 稳定度上和 Claude 比，您观察的差异是什么？这对 Harness 的兜底设计有什么要求？"
3. "如果我作为 PM 给您一份 'trajectory 数据'，您最希望它带什么字段？什么字段是您训练时永远不会用的？"
4. "你们在评估模型时，'sycophancy probe' / 'I don't know rate' 这些 alignment metric 跑得多频繁？是 release 前一次性跑，还是持续跑？"
5. "您觉得 'reward hacking' 在 Harness 用户 trajectory 上有什么典型表现？您希望 PM 怎么帮您识别？"
6. "在您看来，DeepSeek 模型最适合的 agent 用法是什么？最不适合的是什么？" — **这题会让研究员愿意聊很久**

### 给工程师 / 研发

**目的**：摸技术栈、协作方式、当前 pain point

1. "DeepSeek Harness 现在的技术栈是怎样的？TypeScript like Claude Code 还是 Python like LangChain？决策依据是什么？"
2. "你们现在 tool 系统的 schema 用什么定义？Zod、Pydantic 还是自研？"
3. "Sandbox 走 OS-level（macOS sandbox / Linux landlock）还是 container？"
4. "你们怎么测试 Agent Loop？有自动化 trajectory regression 套件吗？"
5. "现在团队有 dogfooding 看板吗？如果没有，PM 进来推一个你们觉得需要吗？"
6. "PM 和你们的协作 ceremony 期望是什么？daily standup / weekly demo / 都没有？"

### 给团队 Leader / 陈德里这种角色（最重要的一轮）

**目的**：测对方的判断、表达你的判断、看双向匹配

> 这一轮的反问要走"高度"——问 leader 才能答的问题。

1. "您之前帖子里说'要做对标 Claude Code 的 DeepSeek Code Harness'——3 年后您希望它是 'Claude Code 的中文版' 还是 '另一种形态'？这两个方向的关键 trade-off 您怎么权衡？"
2. "您招 PM 这个角色，跟招研究员是有意分线的吗？您理想中 PM 给团队的 unique value 是什么？"
3. "假设 6 个月后产品 PMF 没跑出来，您觉得最大的可能原因会是模型不够好、Harness 设计错了，还是 GTM 没找对场景？"
4. "DeepSeek 的开源传统怎么 align 到 Harness？是核心 harness 全开源，还是 hybrid？"
5. "在您看来，PM 在 'model ↔ harness 共同进化' 这件事上，最关键的杠杆是什么？"
6. "您个人对 evaluator agent 在长任务中的角色怎么看？这是 Anthropic 3 月那篇文章的核心论点，但落地复杂度高——您觉得 DeepSeek 现在应该投这件事吗？"
7. **（如果聊得好的话压轴）"如果今天您 26 岁，重新选一份 PM 工作，您会选 Anthropic 还是 DeepSeek Harness？为什么？"** ——这题用得好是金问，用得不好是车祸。**只有面试到最后一刻、对方明显在欣赏你时才用。**

## 二、按目标分

### 目标 1：搜集团队真实状态（决定要不要去）

- "团队最近一次 ship 是什么时候？哪个 release 是你们自己最满意的？"
- "团队加班节奏？每周 deep work 时间占比？"
- "上一位 / 上两位离开团队的人是因为什么离开的？"（这个问题尖锐，但 leader 通常会回答；不答也是信息）

### 目标 2：reveal 自己的产品判断

> 把"问题"变成"我的判断 + 邀请你 verify"

- "我个人判断 v0.1 应该先做 CLI 不做 GUI，理由是 [...]。您怎么看？"
- "我倾向 Skills 走平台 + FS 混合策略，您团队内部有共识吗？"
- "我觉得 DeepSeek-R 系列在 agent loop 里 thinking 应该按场景开关，您们做过实验吗？"

**这种问法的好处**：你既表达了观点（被记住），又把决定权给对方（被尊重）。

### 目标 3：建立长期关系

- "如果今天的 offer 没成，您建议我在哪些方面再积累，下次再聊？"——**这是一个被低估的高分问题**，对方会真诚回答，而且对你有 long-term memory
- "我自己在写一个关于 Harness 的 blog 系列，方便加您 Twitter / 微信吗？您新文章发出来我能第一时间看到？"

### 目标 4：表达成熟度

- "团队招聘节奏听起来很紧，如果我能贡献时间，offer 前我能不能花几天先深度访谈两位你们的现有用户？我把 insight 写成 doc 给你们参考。"——**真正高级 PM 的招——offer 前先创造价值**

## 三、5 个被严重低估的"金问题"

1. **"您觉得我面试中最弱的部分是哪里？"**
   - 在 final round 末尾问，**主动求反馈是高情商动作**；对方答的内容也是你后续 follow-up 的弹药

2. **"如果只能让 Harness 团队保留一个核心信念，您希望是什么？"**
   - 测 leader 真正的产品哲学

3. **"团队过去 3 个月最大的'对自己的发现'是什么？"**
   - leader 必须自我反思才能答这题——好 leader 会答得很好

4. **"在 DeepSeek 内部，Harness 团队是被 protect 的（资源/政治）还是要自己 fight 的？"**
   - 测组织真实生态——决定你来了过得舒不舒服

5. **"假如我入职第一周犯一个错，最不能犯的是什么？"**
   - 测团队 culture 的最重要点；也让对方感受到你有 humility

## 四、绝对不能问的

| 反例问题 | 为什么不能问 |
| --- | --- |
| "我每天几点下班？" | 显得不投入 |
| "工资多少？" | HR 阶段集中问，不在面试官面前问 |
| "你们用的什么模型？参数多少？" | 显得菜，应该已经知道 |
| "你们为什么招我？" | 不是面试官的问题；是你自己要证明的 |
| "竞品 X 在做 Y，你们怎么看？" 单纯一句 | 太空，加上自己判断再问 |
| "升职路径是怎样的？" | 给"求稳"信号，不适合 0→1 团队 |
| 沉默不问 | 给"没思考""不感兴趣"信号 |

---

## 五、按面试场次分配反问额度

假设你有 4~5 轮面试，**每轮反问 5~10 分钟、问 2~4 个问题**。

| 轮次 | 面试官 | 建议反问 |
| --- | --- | --- |
| Round 1·HR | 流程 / 团队 / 团队最稀缺 | HR 第 1, 3, 5 |
| Round 2·PM | 协作 / 哲学 / 对标 | PM 第 1, 3, 5 |
| Round 3·研究员 | 模型行为 / trajectory / sycophancy / DeepSeek 模型 fit | 研究员第 2, 3, 6 |
| Round 4·工程师 | 技术栈 / sandbox / 协作 | 工程师第 1, 3, 5 |
| Round 5·Leader（陈德里） | 路线 / 判断 / culture / 你的弱点 | Leader 第 1, 2, 6 + 金问 1 |

**反问质量的细节**：
- 每个问题问之前先说半句话引出（"基于您刚才提的 X..."），不要突兀
- 问完一个问题要**对答案做一个回应**（"这个回答让我对 [...] 的判断更确定了"），不是机械问下一个
- 最后留 1 个 placeholder 问题，听对面回答得好就深聊，时间紧就跳过

---

## 六、一句应试钥匙

> **反问环节的本质是："如果我们入职，我会怎么和你工作"——你问的问题，就是你将来做事的方式的预演。**

如果你问的全是事实题，对方推测你来了也只会等指令；
如果你问的有判断、有 trade-off、有真诚的好奇——对方就开始想"如果他来了，我们一起干活会很爽"。

**这才是 offer 的来源。**
