# 08 · 反问环节话术库（工程师版）

> 反问环节是工程师候选人区分度最大的 5 分钟。**问得好 = 隐性技术展示**，问得差 = "你来了不会想事"。
>
> PM 版反问话术（[../interview-prep-deepseek-harness-pm/08-反问环节话术库.md](../interview-prep-deepseek-harness-pm/08-反问环节话术库.md)）大部分适用。这里只列工程师特化的差异。

---

## 一、按面试官类型分（工程师特化）

### 给 HR / 人力（first round）

PM 版的所有问题都适用。**工程师特别要问**：

1. "面试有几轮 coding / 几轮 system design / 几轮 behavior？我想为不同轮次准备得更具体。"
2. "Coding 是白板 / 共享屏幕 / take-home assignment？语言能不能选？"
3. "技术栈 / 主要语言 团队偏好是？"
4. "团队对开源 contribution、技术博客、外部演讲的态度？支持还是规避？"
5. "如果我面到 final round，能否安排一次 coding pairing session（不是面试），让我提前感受团队 dev experience？"——**这个问题让 HR 知道你 serious**

### 给工程师同行（peer interview）

**工程师 vs PM 这一轮的差异**：你们说同一种话，可以问深技术。

1. "你们 dev environment 长什么样？monorepo / multi-repo？CI / CD pipeline？语言 / runtime / build tool 选什么？"
2. "你们怎么写测试？unit / integration / E2E 比例？test coverage 目标？"
3. "Code review 文化是什么样？平均 PR review 多久？是不是 maintainer-only merge？"
4. "你们用什么开发 agent 写代码？团队整体使用 Cursor / Claude Code / 别的？"
5. "你们对'AI 写的代码'有什么 review 标准？比'人写的'宽还是严？"
6. "你们最近一次大架构重构 / 重大技术选型是什么？决策过程是怎样的？"
7. "On-call 怎么轮？incident 多吗？事后 retro 文化？"
8. "你最希望 PM 帮你做什么？最希望 PM 不要做什么？"
9. "在 DeepSeek 这一年你学到的最反直觉的事是什么？"
10. "如果我入职，第一周哪件事一旦做错就很难恢复？"

### 给研究员（最关键的一轮，工程师独有）

**这一轮考你能不能成为研究员的 partner**。问题质量直接决定研究员对你的判断。

1. "Anthropic 11 月那篇 long-running harness 用 context reset 解决 Sonnet 4.5 的 context anxiety。你们对 DeepSeek-R / V 系列模型在 long agent loop 里的 anxiety / 收尾倾向有什么内部观察？"
2. "你们在 evaluate 模型 alignment 时 sycophancy probe / I-don't-know calibration 怎么跑？是 release 前一次性，还是持续？"
3. "如果我作为工程师给你一份 'trajectory 数据'，你最希望它带什么字段？什么字段是你永远不会用的？"
4. "DeepSeek-R 系列在 tool_use 上的稳定性 vs Claude，你们的内部数据怎么样？"
5. "你们模型 RL 阶段 reward signal 来自哪里？harness 侧能提供的最有价值反馈是什么形态？"
6. "你们怎么看 reward hacking 在 agent trajectory 上的典型表现？harness 工程师能帮你识别哪些？"
7. "DeepSeek 模型适合的 agent 用法是什么？最不适合的是什么？"
8. "你最近读到的最让你 inspired 的 agent / LLM paper 是？"
9. "如果让你下个季度只解决 harness ↔ model 的一个问题，你会选哪个？"

> **这些问题 = 你在向研究员证明你听得懂他的 daily concerns**——比任何"我会写代码"都有效。

### 给团队 Leader / 陈德里

PM 版的所有问题适用。**工程师特别要问**：

1. "团队目前最大的工程债 / 最棘手的技术挑战是什么？"
2. "你心目中 ideal Harness engineer 是什么 type？强 fundamentals 的 / 强 product instinct 的 / 强 systems 的——哪种最缺？"
3. "你们对 'Eng 写文档 vs Eng 写代码' 的比例期望是怎样的？我观察 Anthropic 工程师写 design doc 的密度很高。"
4. "你们怎么平衡 'ship velocity' 和 'engineering excellence'？小团队这两件事经常冲突。"
5. "如果今天你 26 岁，重新选一份 eng 工作，你会选 DeepSeek 还是 Anthropic？为什么？"——**最高难度金问题，只在面试末段对方很欣赏你时用**

## 二、5 个工程师独有的"金问题"

### 1. "你们 hire-bar 最严的两个评估维度是什么？"

> 让 leader 当面校准 hire bar。回答会告诉你团队真实优先级。

### 2. "团队过去 3 个月改过最大的工程决定是什么？为什么改？"

> 测团队 self-correction 能力。改过的团队 > 不改过的团队。

### 3. "在 DeepSeek，工程师能在多大程度上影响产品方向？"

> 测 'eng 是 PM 执行者还是平等 partner'——这对长期满意度影响巨大。

### 4. "如果让你给一年前的自己一个建议，你会说什么？"

> 让 leader 自我反思，回答的内容比 '团队怎么样' 更真实。

### 5. "你们对 'AI 写代码' 这件事的工程文化怎么处理？比如 review 标准、test 覆盖、attribution。"

> 直击未来工作日常 + 显示你思考过 AI-native engineering 的 nuance。

## 三、绝对不能问的（工程师特化）

| 反例 | 为什么不能问 |
| --- | --- |
| "你们 stack 是什么？我会不会用？" | 这是 HR 阶段问题，到工程师面问显得没准备 |
| "工资多少？" | 给 HR 集中问 |
| "你们用什么 cloud？AWS / GCP / 阿里？" | 太琐碎；工程师不在意你云的偏好 |
| "996 吗？" | 给负面信号——找一份工作不是为了不加班 |
| "我能不能 remote / 居家办公？" | 工程师轮先不问；HR 决定 |
| 沉默不问 | 给"零思考"信号 |

## 四、按面试场次分配反问额度（工程师版）

假设 4~6 轮：

| 轮次 | 面试官 | 建议问 |
| --- | --- | --- |
| Round 1·HR | 流程 / 团队 / 招聘节奏 | HR 第 1-3 |
| Round 2·工程师 A（peer） | 技术栈 / 测试 / code review | 工程师 1, 3, 5 |
| Round 3·Coding round | 简短反问 1~2 个 | 工程师 6, 8 |
| Round 4·System design | 简短反问 2 个 | 工程师 2, 4 |
| Round 5·研究员 | 深度技术 + 模型协作 | 研究员 1, 3, 7 |
| Round 6·Leader（陈德里） | 战略 / 文化 / 决策 | Leader + 金问题 1 / 2 / 5 |

---

## 五、反问质量的细节（同 PM 版铁律）

- 每个问题问之前**先说半句话引出**：'你刚才提到 X，让我想问...'
- 问完一个问题要**对答案做一个回应**：'你这个回答让我对 Y 的判断更准了'，不是机械问下一个
- 最后留 1 个 **placeholder 问题**：听对面回答得好就深聊，时间紧就跳过

---

## 六、一句应试钥匙

> **反问环节本质是预演：你将来在 1:1 / design review / RFC 评审里和这个人怎么对话。**
>
> 反问的质量 = 你将来工作的质量预告片。
>
> **让面试官感觉到 "我希望未来和这个人 sync"——offer 就来了。**
