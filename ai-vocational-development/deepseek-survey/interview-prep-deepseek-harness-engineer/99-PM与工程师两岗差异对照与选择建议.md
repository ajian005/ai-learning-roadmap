# 99 · PM 与工程师两岗差异对照与选择建议

> 这份文档假设你**可能在 DeepSeek Harness 团队的 PM 和工程师两个 JD 之间犹豫**——或者想知道用哪份准备包。
>
> 两份 JD 是**同一个团队**（陈德里牵头）、**同一款产品**（DeepSeek 桌面端 Code Harness）、**几乎同一批面试官**——但角色定位、考察方式、加分项**截然不同**。
>
> 投递前花 30 分钟读完，可能改变你这次面试结果。

| 维度 | Agent Harness **产品经理** | Agent Harness **研发工程师** |
| --- | --- | --- |

## 二、谁该投哪个？

### 投 PM 的典型画像

- 主要经验是**产品经理**（无论 AI / SaaS / 工具类）
- 写代码主要靠 Cursor / Claude Code "vibe"，不算系统训练
- 有 **2~5 年定义产品 + 推动跨职能** 经验
- 喜欢"想清楚再 ship"而不是"边 ship 边想"
- 对**用户研究、A/B 测试、统计学**有系统训练
- 能用英文写得体的开源社区帖子 / RFC
- 喜欢 **stakeholder 协调**而不是孤独写代码

**典型 background**：互联网 PM 转 AI / 上一份是 Notion AI / 字节 AI / Cursor PM / 小公司 founding PM

### 投工程师的典型画像

- 主要经验是**软件工程**（无论后端 / 前端 / infra / 系统）
- 能独立 ship **production grade 代码 + 工程债清理 + 性能优化**
- 写过技术博客 / 贡献过开源 / 有 GitHub 痕迹
- 享受 **"和研究员 1:1 讨论 paper"** 多过 "和 stakeholder 对齐 roadmap"
- 对 **系统设计 / 架构选型** 有 opinion
- 学新技术栈很快，AI 辅助下能 1 周 ramp up 陌生语言
- 喜欢深聊 KV cache / agent loop 实现细节

**典型 background**：infra eng / framework author / AI infra eng / open-source maintainer / quant developer / 算法工程师转工程

## 三、自我评估表（给自己打分）

- [ ] 我享受跨职能协调 + 上下沟通
- [ ] 我能在 stakeholder 意见冲突时主导 alignment

**A 组总分**：__ / 35

### B 组（工程师倾向）

- [ ] 我能在 45 分钟里白板写一个 minimal agent loop
- [ ] 我能在 30 分钟里设计一个 LLM inference gateway 系统
- [ ] 我读过 ≥ 2 个开源 agent harness 源码 + 能讲设计思路
- [ ] 我有 ≥ 1 个 100+ star 的 GitHub repo / 有质量 PR
- [ ] 我能用 Cursor 在 1 周内学会一门新语言到 production 程度

**B 组总分**：__ / 35

### 解读

- A 组 ≥ B 组 + 7：**投 PM**
- B 组 ≥ A 组 + 7：**投工程师**
- A 组 vs B 组差 ≤ 7：**两个都投或选你更想做的**
- A 组 < 20 + B 组 < 20：考虑先**补能力**再投这个团队

## 四、两岗"日常一周"模拟

### PM 的典型一周

```
周一 上午  Sprint planning + research 同步
        下午 用户访谈（2 个）
        晚上 整理 insight + 同步研究员

周二 上午  指标看板 review + A/B 实验设计
        下午 写 PRD / spec
        晚上 dogfood 用 Harness 写一个 case study

周三 上午  和工程师 1:1 (2 人) review trade-off
        下午 和研究员 1:1 review 模型行为新观察
        晚上 GitHub issue / Discord 答疑

周四 全天 用 vibe coding 帮工程师做一个 prototype demo

周五 上午 周会 + 数据 review
        下午 路线图更新 + 上 leader 1:1
        晚上 看 paper / 试 competitor
```

**心理模式**：80% 时间在"对齐 + 决策 + 反馈"；20% 时间在"自己 ship"

### 工程师的典型一周

```
周一 上午  Sprint planning + 选 task
        下午 设计 doc 写 / RFC 评审
        晚上 实现核心模块

周二 全天 写代码 + test + code review 同事 PR

周三 上午  和研究员深聊新 feature 接口
        下午 调试 + perf benchmark
        晚上 看 paper / 看源码 (Claude Code / OpenCode)

周四 上午  和模型团队 sync 新 ckpt 的 harness 适配
        下午 实现 + ship
        晚上 GitHub 个人 repo 维护

周五 上午 周会 + 看 dogfood telemetry
        下午 处理 user feedback / debug 用户报的 bug
        晚上 写技术博客 / 开源贡献
```

**心理模式**：70% 时间在"写代码 + 调试 + design"；30% 时间在"对齐 + 反馈"

## 六、两岗"被拒后的转化"

## 七、混合策略：两个都投？

**优点**：
- 增加进面试概率
- 看哪个 HR / 面试官先回应
- 同时投显示 commitment（"我对这个团队，不是这个岗位")

**风险**：
- 团队内部可能不喜欢 "投两个岗"（显示定位不清）
- 占用 hire budget
- 两边都准备摊薄精力

**操作建议**：
- 时间间隔 1~2 周，先投你更确定的那个
- 第一个拒了再投另一个（不是同时）
- 内推时跟内推人沟通，确认"是否能 cross-apply"
- 准备包**优先做你更倾向的那个**，另一个轻量准备

## 九、终极选择问题（5 问）

如果还犹豫，问自己这 5 个问题，**多数答案靠 A 选 PM，靠 B 选工程师**：

1. **过去 1 年里我花更多时间在……**
   - A) 写文档 / spec / proposal / report
   - B) 写代码 / debug / code review

2. **我觉得 work 最有 flow 的时刻是……**
   - A) 协调 5 个 stakeholder 把一件事 ship
   - B) 一个人安静 4 小时写代码

3. **被问到"我擅长什么"时第一反应是……**
   - A) 沟通 / 判断 / 优先级
   - B) 编码 / 系统 / 调试

4. **5 年后我希望被人 introduce 为……**
   - A) "AI Product Leader at DeepSeek"
   - B) "Agent Engineer who built X / wrote Y / contributed to Z"

5. **如果一个机会让我每天写 8 小时代码 + 2 小时开会，我感觉……**
   - A) 太多代码了，我希望反过来
   - B) 完美

## 十、最后的话

> 这两个岗位都是 DeepSeek **少有的真正前沿 + 长期主义 + 研究密度极高** 的位置。
>
> 选哪个不是"哪个机会大"，是"哪个我能持续燃烧热情 3 年"。
>
> 工程师能转 PM（如果你想），PM 转工程师较难。但更重要的不是"路径可逆性"，是"你做哪个会自己跑得最快"。
>
> 选下来一份 JD，配上对应的准备包，全力以赴 7 天——offer 不取决于"你天赋多高"，取决于"你这 7 天有多 specific 地准备"。
>
> **祝顺利。**
