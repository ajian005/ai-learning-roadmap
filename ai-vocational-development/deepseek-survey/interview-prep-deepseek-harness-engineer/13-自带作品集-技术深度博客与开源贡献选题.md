> 这是 Minimal Harness（文档 12）的"姊妹篇"。文档 12 是"看我能写代码"；本文档是"看我能写文字 / 能在社区被记住"。
>

## 一、路线 A · 技术深度博客（高 leverage）

### 1.1 为什么写博客

- **被搜到**：陈德里这种 leader 会用 Google + Twitter 搜你的名字
- **被记住**：面试官可能在面后 search "你的名字 + harness"
- **建立 identity**：一篇真正深度的文章 = 1 年的 LinkedIn 经验
- **是简历的"代码"**：可以贴到简历

### 1.2 推荐 10 个选题（按难度排序）

#### 入门难度（1~2 天写完）

**1. 《Claude Code 源码学习笔记：从 entry 到 agent loop》**
- 受众：中文 AI 开发者
- 长度：2500 字
- 重点：QueryEngine + diminishing-returns + concurrency-safe dispatch
- 配图：架构图 + 调用链图

**2. 《Anthropic 11 月那篇 long-running harness 我是这么复刻的》**
- 长度：1800 字
- 重点：Initializer + Coding Agent 两段式；handoff schema 设计
- 配 code：自己写的 ~150 行复刻

**3. 《为什么 Claude Code 选 Bun 而不是 Node：从启动延迟到 feature gates》**
- 长度：1500 字
- 重点：实测 cold start + bundler dead code elimination + tree-shaking
- 配数据：3 张 benchmark 图

#### 中级难度（3~5 天）

**4. 《Agent 长任务里的 context anxiety：观察、原因、缓解》**
- 长度：3000 字
- 重点：Sonnet 4.5 context anxiety 实测 + 三种缓解方案对比
- 配实验：3~5 个 prompt 跑出来的对比 chart

**5. 《用 500 行实现一个 Minimal Harness：我学到的 10 件事》**
- 长度：3000 字
- 重点：把你的 Minimal Harness（文档 12）经验沉淀
- 配 repo：放 GitHub URL

**6. 《Tool Use 在 OpenAI / Anthropic / Gemini 三家的协议差异：一个 harness 工程师的踩坑笔记》**
- 长度：2500 字
- 重点：JSON string vs object / streaming partial / id 协议
- 配 code：3 个 adapter 代码片段

#### 高难度（5~10 天，硬核）

**7. 《复刻 GAN-style 三 Agent（Anthropic 2026-03）：跑 6 小时的实测》**
- 长度：4000 字
- 重点：完整 reproduce + 自己变体 + cost 对比
- 配 repo：单独 repo + readme

**8. 《对照 KV Cache 友好性：4 种 system prompt 装配方案的 token 经济实验》**
- 长度：3500 字
- 重点：实测不同 prompt structure 对 cache hit rate 影响 + 数据
- 配数据：成本曲线 + cache hit %

**9. 《如何评估 LLM Agent：从 SWE-Bench 到自建 100-task benchmark》**
- 长度：4000 字
- 重点：metric 选择 + LLM-as-judge calibration + 跑 5 个开源 model 横评
- 配数据：leaderboard 表

**10. 《为 DeepSeek-R 系列设计 Agent Loop 的 5 个工程考量》**
- 长度：3000 字
- 重点：thinking budget / tool_use stability / 长 trace / 中文 instruction following
- **特别杀招**：这篇直接对 DeepSeek 团队招呼

### 1.3 博客发表渠道（中英双轨）

| 渠道 | 适合 |
| --- | --- |
| **个人博客 / Github Pages** | 长期 owned，简历可放 URL |
| **知乎** | 中文流量最大，技术受众强 |
| **公众号** | 中文 viral 潜力 |
| **LinkedIn** | 国际 + 招聘官能搜到 |
| **Medium / Substack** | 英文，海外社区 |
| **Hacker News / Reddit r/MachineLearning** | 提交链接，国际曝光 |
| **Twitter / X thread** | 短链路高曝光，发完贴文章 link |
| **小红书** | 国内 AI 圈年轻人 |

**建议组合**：知乎 + Github 双发 + 一条 Twitter thread 引流。

### 1.4 一篇高质量博客的 7 个 checklist

- [ ] **标题 ≤ 12 字 中文 / ≤ 8 词 英文**，含一个具体名词
- [ ] **开头 1 段话回答 "Why should I read this?"**
- [ ] **至少 1 张架构图 / 数据图**
- [ ] **至少 1 段可运行 code**
- [ ] **至少 1 个反直觉 finding**
- [ ] **结尾给 link / next-step**
- [ ] **结尾署名 + 联系方式（Twitter / GitHub）**

### 1.5 博客快速完成 SOP（如果只有 3 天）

```
Day 1·早上    选题 + outline（30 min）
Day 1·下午    跑 demo / 实验（2~4h）—— 让 Cursor / Claude Code 帮
Day 1·晚上    第一稿（2h）—— 让 Claude 帮 draft
Day 2·早上    精修 + 补图（3h）
Day 2·下午    朋友 review + 改（1h）
Day 2·晚上    发布到 1 个主渠道
Day 3·全天    分发：Twitter / 知乎 / Reddit / LinkedIn
```

## 二、路线 B · 开源贡献（中 leverage）

### 2.1 为什么贡献开源

- JD 直接列加分项："**个人开源作品及开源社区的深度贡献**"
- 1 个合并的 PR 在 GitHub profile 上永久可见
- maintainer 可能给你写推荐信
- 学习高水平 codebase

### 2.2 推荐 10 个 candidate repo（按贡献门槛排序）

#### 低门槛（适合 docs / typo / 小 fix）

**1. [OpenCode (sst/opencode)](https://github.com/sst/opencode)**
- 活跃；中文社区少；issue 多
- 推荐方向：docs 中文化 / examples / 小 bug fix
- PR 难度：低-中

**2. [Aider (paul-gauthier/aider)](https://github.com/paul-gauthier/aider)**
- 老牌 coding agent
- 推荐方向：DeepSeek model 适配 / 新 LLM provider
- PR 难度：中

**3. [liteagent (DrChrisLevy/liteagent)](https://github.com/drchrislevy/liteagent)**
- 小项目，维护者一人，PR 容易合
- 推荐方向：example / docs / 加 tool
- PR 难度：低

**4. [tinyAgent-TS (alchemiststudiosDOTai/tinyagent-ts)](https://github.com/alchemiststudiosDOTai/tinyagent-ts)**
- TS 版极简
- 推荐方向：example / typescript types / docs
- PR 难度：低

**5. [Model Context Protocol (anthropics/mcp)](https://github.com/anthropics/mcp)**
- Anthropic 官方仓库
- 推荐方向：example MCP server / docs / 新 transport
- PR 难度：中-高（review 严）

#### 中门槛（feature contribution）

**6. [Continue (continuedev/continue)](https://github.com/continuedev/continue)**
- 开源 Cursor 替代
- 推荐方向：DeepSeek provider / 中文 system prompt
- PR 难度：中

**7. [LangChain (langchain-ai/langchain)](https://github.com/langchain-ai/langchain)**
- 老牌但 PR 池大；最近不再火
- 推荐方向：DeepSeek integration / 中文 example
- PR 难度：低-中

**8. [Pydantic AI (pydantic/pydantic-ai)](https://github.com/pydantic/pydantic-ai)**
- 新 agent framework
- 推荐方向：DeepSeek provider / example
- PR 难度：中

#### 高门槛（核心代码）

**9. [vLLM (vllm-project/vllm)](https://github.com/vllm-project/vllm)**
- LLM inference 主流引擎
- 推荐方向：性能优化 / 新 model 支持
- PR 难度：高，但合一个就很重

**10. [SGLang (sgl-project/sglang)](https://github.com/sgl-project/sglang)**
- 新 inference 引擎，DeepSeek 团队熟
- 推荐方向：tool calling 改进 / agent loop pattern
- PR 难度：高
- **特别 leverage**：DeepSeek 团队可能熟悉 SGLang 贡献者

### 2.3 1 周内贡献 1 个合并 PR 的 SOP

```
Day 1·早上    选 repo（建议 OpenCode / Aider / liteagent 三选一）
Day 1·下午    fork + clone + setup dev env
Day 1·晚上    读 CONTRIBUTING.md + 浏览 issue 标 'good first issue'

Day 2·全天    选 1 个 issue，在 issue 下留言 "I want to take this，my approach is X"

Day 3-4       实现 + 测试

Day 5         submit PR，写清 PR description：
              - what changed
              - why
              - how tested
              - any open questions

Day 5-7       响应 review，迭代直到 merge
```

### 2.4 PR 礼仪 5 条（提升 merge 率）

1. **小 + 单一职责**：1 PR 1 事，不要一锅炖
2. **测试同步加**：每个 PR 至少 1 个新 test
3. **PR description 详细**：what / why / how，附 screenshot 如适用
4. **review 必回复**：哪怕拒绝也要解释为什么不接受 review feedback
5. **respect maintainer's call**：他们的 repo 他们的规则

## 三、路线 C · Paper-style writeup（高 leverage 但费时）

### 3.1 适合什么人

- 已经有学术背景 / 写过 paper
- 时间充足（10 天+）
- 想申请研究员风格 role

### 3.2 推荐 5 个 writeup 选题

**1. "Reproducing Anthropic's 3-Agent Harness: A 30-Day Engineering Diary"**
- 类型：实验复刻
- 长度：3000 英文 / 4000 中文
- 数据：实测 cost / quality / latency 对比

**2. "How Tool Calling Format Affects Agent Loop Robustness: A Cross-Model Empirical Study"**
- 类型：对比研究
- 实验：OpenAI / Anthropic / Gemini / DeepSeek 4 家 tool use schema drift rate

**3. "Cache-Aware Prompt Design for Multi-Turn Agents: A Cost-Effectiveness Analysis"**
- 类型：实验研究
- 数据：4 种 prompt structure × 30-turn session × 4 种模型 cost matrix

**4. "Evaluating Agent Harness Quality: Beyond SWE-Bench"**
- 类型：方法论
- 内容：propose 4-dim metric + 100-task benchmark

**5. "Context Reset vs Compaction in Long Agent Sessions: When Each Wins"**
- 类型：实证比较
- 实验：3 种长任务 × 2 种 context management × 2 种模型

### 3.3 Paper-style writeup 的格式

```
# Title

## Abstract (200 words)
## 1. Introduction
## 2. Related Work
## 3. Method
## 4. Experiments
## 5. Results
## 6. Discussion
## 7. Limitations
## 8. Conclusion
## References
```

**关键**：不需要发 venue（除非你想），传到 arXiv / 个人博客 就行。
