### 题 1·实现一个 Minimal Agent Loop（30~45 分钟）

**题面**：白板用任意语言实现一个最小 agent loop，支持 tool use。约束：
- 接 OpenAI / Anthropic Messages API
- 至少 2 个 tool（read_file / list_files）
- 支持 max_turns + tool error 自动喂回
- 加分：streaming / abort signal

**评分点**（面试官心中表）：
- ✅ messages 序列正确（user/assistant/user 交替，tool_use 必须配 tool_result）
- ✅ stop_reason 处理完整（end_turn / tool_use / max_tokens）
- ✅ tool 错误不抛异常，作为 tool_result 注入
- ✅ 有 max_turns 上限
- ✅ 至少识别 isConcurrencySafe 概念（即使没实现）
- ✅ 显式提到 "我会加 streaming，但先给阻塞版本，时间够再加"

**参考实现**（Python，~60 行）：

```python
from anthropic import Anthropic
import json
import os

client = Anthropic()

TOOLS = {
    "read_file": {
        "fn": lambda path: open(path).read()[:10000],
        "schema": {
            "name": "read_file",
            "description": "Read content of a file",
            "input_schema": {
                "type": "object",
                "properties": {"path": {"type": "string"}},
                "required": ["path"],
            },
        },
    },
    "list_files": {
        "fn": lambda directory: "\n".join(os.listdir(directory)),
        "schema": {
            "name": "list_files",
            "description": "List files in a directory",
            "input_schema": {
                "type": "object",
                "properties": {"directory": {"type": "string"}},
                "required": ["directory"],
            },
        },
    },
}


def run(user_input: str, max_turns: int = 20):
    messages = [{"role": "user", "content": user_input}]
    tool_schemas = [t["schema"] for t in TOOLS.values()]

    for turn in range(max_turns):
        resp = client.messages.create(
            model="claude-sonnet-4-5",
            max_tokens=4096,
            tools=tool_schemas,
            messages=messages,
        )
        messages.append({"role": "assistant", "content": resp.content})

        if resp.stop_reason == "end_turn":
            return resp
        if resp.stop_reason != "tool_use":
            raise RuntimeError(f"Unexpected stop: {resp.stop_reason}")

        tool_results = []
        for blk in resp.content:
            if blk.type != "tool_use":
                continue
            try:
                output = TOOLS[blk.name]["fn"](**blk.input)
            except Exception as e:
                output = f"ERROR {type(e).__name__}: {e}"
            tool_results.append({
                "type": "tool_result",
                "tool_use_id": blk.id,
                "content": str(output)[:10000],
            })
        messages.append({"role": "user", "content": tool_results})

    raise RuntimeError("Exceeded max_turns")


if __name__ == "__main__":
    result = run("List my home directory and read .zshrc if exists")
    print(result.content)
```

**面试中口头补充**：
> "工业版我会加 5 件事：(1) AbortController 让 Esc 即时停；(2) streaming via SSE；(3) concurrency-safe 并发 tool；(4) 每轮 budget 检查；(5) diminishing-returns detector。这 60 行只展示骨架。"

### 题 2·实现一个 Tool Schema Validator（20~30 分钟）

**题面**：实现一个 function 接受 `tool_schema` (JSON schema) 和 `tool_input` (any)，返回 valid / invalid + 详细错误。不让用第三方库。

**评分点**：
- ✅ 处理 type / required / properties
- ✅ 嵌套对象 / 数组递归
- ✅ 友好错误（path + 期望 vs 实际）
- ✅ 主动提到："工业版会用 Ajv / Zod / jsonschema 库；这里只是手撕骨架"

**参考实现**（Python）：

```python
def validate(schema, value, path="$"):
    """Returns (is_valid, errors)"""
    errors = []
    t = schema.get("type")

    if t == "object":
        if not isinstance(value, dict):
            errors.append(f"{path}: expected object, got {type(value).__name__}")
            return False, errors
        for req_key in schema.get("required", []):
            if req_key not in value:
                errors.append(f"{path}: missing required field '{req_key}'")
        for key, sub_schema in schema.get("properties", {}).items():
            if key in value:
                ok, sub_errs = validate(sub_schema, value[key], f"{path}.{key}")
                errors.extend(sub_errs)

    elif t == "array":
        if not isinstance(value, list):
            errors.append(f"{path}: expected array, got {type(value).__name__}")
            return False, errors
        items_schema = schema.get("items", {})
        for i, item in enumerate(value):
            _, sub_errs = validate(items_schema, item, f"{path}[{i}]")
            errors.extend(sub_errs)

    elif t == "string":
        if not isinstance(value, str):
            errors.append(f"{path}: expected string, got {type(value).__name__}")
        elif "enum" in schema and value not in schema["enum"]:
            errors.append(f"{path}: value '{value}' not in enum {schema['enum']}")

    elif t == "integer":
        if not isinstance(value, int) or isinstance(value, bool):
            errors.append(f"{path}: expected integer, got {type(value).__name__}")

    elif t == "number":
        if not isinstance(value, (int, float)) or isinstance(value, bool):
            errors.append(f"{path}: expected number, got {type(value).__name__}")

    elif t == "boolean":
        if not isinstance(value, bool):
            errors.append(f"{path}: expected boolean, got {type(value).__name__}")

    return len(errors) == 0, errors
```

**追问 + 应对**：
- "如果用户给了 OpenAI 风格的 `arguments` 是 JSON 字符串，你怎么改？" → 加一层 `try: value = json.loads(value)` if str + schema expects object
- "Schema 演进怎么办？" → 加 `$schema` version 字段 + adapter

### 题 3·实现一个 Concurrency-Safe Tool Dispatcher（30 分钟）

**题面**：你有一批 tool calls，每个 tool 有 `isConcurrencySafe: bool`。设计一个调度器：
- 连续 concurrency-safe 的可并发执行
- 一个 non-safe 必须独占执行
- 默认 max concurrency = 10

**评分点**：
- ✅ 正确按 safe boundary 分批
- ✅ 用 `Promise.all` / `asyncio.gather` 并发
- ✅ 单 non-safe 串行
- ✅ 默认 + 可配置 concurrency
- ✅ 提到 error handling（一个 fail 怎么处理）

**参考实现**（Python asyncio）：

```python
import asyncio
from typing import Callable, Any

class ToolCall:
    def __init__(self, name: str, fn: Callable, is_concurrency_safe: bool, input: dict):
        self.name = name
        self.fn = fn
        self.is_concurrency_safe = is_concurrency_safe
        self.input = input

async def dispatch(tool_calls: list[ToolCall], max_concurrency: int = 10):
    """
    按 concurrency-safe 边界分批并发。
    一个不安全的 tool 单独跑。
    """
    results = [None] * len(tool_calls)
    semaphore = asyncio.Semaphore(max_concurrency)

    async def run_one(idx: int, tc: ToolCall):
        async with semaphore:
            try:
                results[idx] = await asyncio.get_event_loop().run_in_executor(
                    None, tc.fn, **tc.input
                )
            except Exception as e:
                results[idx] = f"ERROR: {e}"

    # 切批
    batches: list[list[tuple[int, ToolCall]]] = []
    current_batch: list[tuple[int, ToolCall]] = []
    for i, tc in enumerate(tool_calls):
        if tc.is_concurrency_safe:
            current_batch.append((i, tc))
        else:
            if current_batch:
                batches.append(current_batch)
                current_batch = []
            batches.append([(i, tc)])  # non-safe 独占批
    if current_batch:
        batches.append(current_batch)

    # 串行执行批，批内并发
    for batch in batches:
        await asyncio.gather(*[run_one(i, tc) for i, tc in batch])

    return results
```

**追问 + 应对**：
- "如果某个 tool 卡死怎么办？" → 加 `asyncio.wait_for(...timeout=N)`
- "如果用户在中间 abort？" → AbortSignal / asyncio.Event + 各 task 检查
- "如果 5 个安全 tool 都依赖同一个文件 read？" → 在 read 层加 cache（不在调度层）

### 题 4·实现一个 Streaming SSE Parser（30 分钟）

**题面**：解析 Anthropic Streaming Messages 的 SSE chunks，把 `text_delta` 和 `tool_use` 的 `input_json_delta` 累积成完整 message。

**Anthropic SSE chunk 示例**：

```
event: message_start
data: {"type":"message_start","message":{"id":"...","content":[],...}}

event: content_block_start
data: {"type":"content_block_start","index":0,"content_block":{"type":"text","text":""}}

event: content_block_delta
data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","text":"Hello"}}

event: content_block_delta
data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","text":" world"}}

event: content_block_stop
data: {"type":"content_block_stop","index":0}

event: content_block_start
data: {"type":"content_block_start","index":1,"content_block":{"type":"tool_use","id":"toolu_xxx","name":"read_file","input":{}}}

event: content_block_delta
data: {"type":"content_block_delta","index":1,"delta":{"type":"input_json_delta","partial_json":"{\"path\":\""}}

event: content_block_delta
data: {"type":"content_block_delta","index":1,"delta":{"type":"input_json_delta","partial_json":"foo.txt\"}"}}

event: content_block_stop
data: {"type":"content_block_stop","index":1}

event: message_delta
data: {"type":"message_delta","delta":{"stop_reason":"tool_use"}}

event: message_stop
data: {"type":"message_stop"}
```

**评分点**：
- ✅ 处理 SSE 格式（event: + data: + 空行分隔）
- ✅ 按 index 重建多个 content block
- ✅ accumulate text_delta 和 input_json_delta 分别处理
- ✅ 最后 JSON.parse 完整的 partial_json
- ✅ 处理 partial chunks（一行 data: 可能跨多个 read）

**参考实现**：

```python
import json
from typing import Iterator

class SSEParser:
    def __init__(self):
        self.blocks: dict[int, dict] = {}  # index -> block
        self.stop_reason = None
        self.text_accumulators: dict[int, str] = {}
        self.json_accumulators: dict[int, str] = {}

    def feed(self, chunk: str) -> Iterator[dict]:
        """把 chunk 切成 event，逐个处理，yield 高层事件"""
        for line in chunk.split("\n"):
            if not line.startswith("data: "):
                continue
            data = json.loads(line[6:])
            t = data["type"]

            if t == "content_block_start":
                idx = data["index"]
                blk = data["content_block"]
                self.blocks[idx] = blk
                if blk["type"] == "text":
                    self.text_accumulators[idx] = ""
                elif blk["type"] == "tool_use":
                    self.json_accumulators[idx] = ""

            elif t == "content_block_delta":
                idx = data["index"]
                delta = data["delta"]
                if delta["type"] == "text_delta":
                    self.text_accumulators[idx] += delta["text"]
                    yield {"type": "text", "content": delta["text"]}
                elif delta["type"] == "input_json_delta":
                    self.json_accumulators[idx] += delta["partial_json"]

            elif t == "content_block_stop":
                idx = data["index"]
                blk = self.blocks[idx]
                if blk["type"] == "text":
                    blk["text"] = self.text_accumulators[idx]
                elif blk["type"] == "tool_use":
                    blk["input"] = json.loads(self.json_accumulators[idx])
                    yield {"type": "tool_use", "block": blk}

            elif t == "message_delta":
                self.stop_reason = data["delta"]["stop_reason"]

            elif t == "message_stop":
                yield {"type": "done", "stop_reason": self.stop_reason}

    def result(self) -> dict:
        return {
            "content": [self.blocks[i] for i in sorted(self.blocks.keys())],
            "stop_reason": self.stop_reason,
        }
```

**追问 + 应对**：
- "如果 chunk 在 JSON 中间断开（partial UTF-8 byte）怎么办？" → buffer 未完成的 chunk，等下一次 feed
- "如果同时有多个 tool_use block？" → 用 index 隔离（上面已经支持）
- "如果 input_json_delta 累积起来不是合法 JSON？" → 流式 stale state，可以用 `json5` lenient parser 兜底

### 题 5·实现一个 Context Compactor（30 分钟）

**题面**：messages 太长，超过 budget。写一个 compact 函数：
- 保留 system + 最近 N 轮
- 中间历史用 LLM summarize 成 1 段
- 保留所有 `tool_use_id` 引用关系（不能让 tool_result 失去对应的 tool_use）

**评分点**：
- ✅ 识别 messages 必须 user/assistant 交替
- ✅ 保留最近 N 轮原文（"最近" 用 turn count 不是字数）
- ✅ tool_use ↔ tool_result 配对保护（要么同删要么同留）
- ✅ summary 占一个 user message
- ✅ token 估算（tiktoken / 估算 1 token ≈ 4 char）

**参考实现**：

```python
def compact(messages: list[dict], budget_tokens: int, keep_recent_turns: int = 5,
            summarize_fn=None) -> list[dict]:
    """
    保留 system + 最近 N 轮，中间用 summary 替换。
    保持 tool_use ↔ tool_result 配对。
    """
    # 1. 提取 system（如果用 system 字段就跳过；如果在 messages 里第一条）
    system_msg = None
    body = messages
    if messages and messages[0].get("role") == "system":
        system_msg = messages[0]
        body = messages[1:]

    # 2. 找 keep_recent_turns 的起点
    # 一轮 = user → assistant pair（含 tool_use/tool_result 来回）
    turn_starts = []
    for i, m in enumerate(body):
        if m["role"] == "user" and not _is_tool_result_only(m):
            turn_starts.append(i)

    if len(turn_starts) <= keep_recent_turns:
        return messages  # 不需要 compact

    cutoff = turn_starts[-keep_recent_turns]
    to_summarize = body[:cutoff]
    to_keep = body[cutoff:]

    # 3. summarize 中间部分
    if summarize_fn:
        summary = summarize_fn(to_summarize)
    else:
        summary = f"[Summary of {len(to_summarize)} earlier messages]"

    summary_msg = {
        "role": "user",
        "content": f"[Earlier conversation summarized]:\n{summary}"
    }

    # 4. 拼回
    result = []
    if system_msg:
        result.append(system_msg)
    result.append(summary_msg)
    result.extend(to_keep)

    return result

def _is_tool_result_only(msg: dict) -> bool:
    """判断这个 user message 是不是只有 tool_result 而没有真实 user 输入"""
    if isinstance(msg["content"], str):
        return False
    return all(
        isinstance(b, dict) and b.get("type") == "tool_result"
        for b in msg["content"]
    )
```

**追问 + 应对**：
- "Tool_use 在被 summarize 的部分，但 tool_result 在保留部分怎么办？" → 配对必须同删同留，或在 summary 里加 "tool X returned Y"，但 tool_result 也必须 drop
- "Summary 也很长怎么办？" → 多 stage compact（summary of summary）
- "如果 cache 被破坏？" → compact 后立即在 summary_msg 加 cache_control breakpoint
