# 第二部分·面试中临场策略

> 同 [Eng 包 10](../interview-prep-deepseek-harness-engineer/10-面试中临场策略与话术.md) 通用部分。本文档只列**全栈特化**。

## 开场 60 秒（全栈版）

> "您好 [面试官]，谢谢您的时间。
>
> 我是 [姓名]，过去 [N] 年做 [后端 / 前端 / Infra 偏哪]，最近 [N] 个月把大部分时间花在 Agent infra + 评测平台方向。
>
> **我面试前用 ~500 行做了一个 mini Eval Platform——前端 React + 后端 FastAPI + Dockerfile + 一个 LangChain agent adapter——repo 在 [URL]。如果今天聊到 trajectory 可视化 / scaffold 集成，我们可以一起 review；如果先聊背景或 coding 也都没问题。您想从哪里开始？**"

## 全栈面试的"切栈"话术

如果面试官问你前端时你后端更强（反之亦然），用：

```
"我前端 / 后端 / Infra 这三块强弱有别。
我后端 / 前端 是主力，能扛 production；
另一块我能 own 中等复杂度，但深度未必到 production grade。
我会在 [具体动作] 上持续补——比如 [例子]。"
```

> 主动 surface 强弱比 inflate 装的更有可信度。

## Live Coding 时的"全栈分组"信号

每写一段代码主动 label "这是哪一层"：

```
"现在我写的是 frontend 拿数据的部分..."
"切到后端 endpoint..."
"接下来 Dockerize 这个..."
```

让面试官知道你**意识到全栈跨层**，不是单层思维。

## 系统设计 4 阶段（全栈题加深）

全栈题（如"设计 eval platform"）和纯后端不同：必须**主动覆盖前后端 + Infra 三层**。

```
Stage 2·High-level 时画分层架构（UI / API / Pipeline / Storage）
Stage 3·Deep Dive 时选 1 个组件覆盖三栈：
  - 前端：组件状态管理 / 性能
  - 后端：DB schema / API design
  - Infra：部署 / scale
```

## 全栈面试常见"陷阱场景"

| 场景 | 应对 |
| --- | --- |
| 让你白板写 Dockerfile + manifest | 别背模板，主动讲"为什么 multi-stage / 为什么 distroless / 为什么 readiness probe" |
| 问 "你 React 经验如何" 但同时考 K8s | 不假装精通；主动列实际经验深度 + 例子 |
| 问 "如果让你做 LangChain 二次开发"  | 主动讨论 contract test、API stability、版本管理 |
| 问"用什么 LLM observability 工具" | 不要说"我没用过"——讲你看过的开源（LangFuse / Phoenix） + 你会怎么 evaluate |

## 5 个邮件原则

1. **24h 内发**
2. **不超过 200 字 + 附 1 个 commit URL**
3. **必须提到面试中 1 个具体话题**
4. **commit message production-ready**（不要 'wip'）
5. **抄送 HR**

## 全栈面试"心理建设三句话"

1. **"我不是来证明我每块都强的，是证明我能 ship + 能跨栈 + 能学。"**
2. **"代码写错很 OK，debug 思路展示比代码完美更值钱。"**
3. **"题不会很 OK——我答'我不会但 mental model 是这样'比硬装更高分。"**
