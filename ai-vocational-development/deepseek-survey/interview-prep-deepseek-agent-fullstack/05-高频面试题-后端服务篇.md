### 题 1·设计 Trajectory DB Schema + 写 GET endpoint（30 分钟）

**题面**：DB 存 trajectory（10K/天，每个最多 1000 步）。设计 schema + 写一个 `GET /trajectory/{id}` 和 `GET /trajectory?model=X&task_id=Y&limit=20`。

**评分点**：
- ✅ 区分 trajectory metadata 表 vs steps 表
- ✅ 用 JSONB / Parquet 存大的 step content
- ✅ 索引设计（model / task_id / created_at）
- ✅ pagination（cursor 优于 offset）
- ✅ 主动讨论 partition / sharding

**参考 schema（PostgreSQL）**：

```sql
-- 主表：trajectory metadata
CREATE TABLE trajectories (
  id           UUID PRIMARY KEY,
  model        TEXT NOT NULL,
  harness_ver  TEXT NOT NULL,
  task_id      TEXT NOT NULL,
  outcome      TEXT NOT NULL CHECK (outcome IN ('success', 'fail', 'abandoned')),
  step_count   INT  NOT NULL,
  total_tokens INT  NOT NULL,
  duration_ms  INT  NOT NULL,
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  metadata     JSONB DEFAULT '{}'::jsonb
);
CREATE INDEX trajectories_model_created_idx ON trajectories(model, created_at DESC);
CREATE INDEX trajectories_task_idx          ON trajectories(task_id);
CREATE INDEX trajectories_metadata_gin      ON trajectories USING gin(metadata);

-- 步骤详情：可选 JSONB 或单独表
-- 方案 A：JSONB 列（简单 + 一次性读取）
ALTER TABLE trajectories ADD COLUMN steps JSONB NOT NULL;
-- 缺点：单行可达 MB 级，UPDATE 重写全行

-- 方案 B：steps 子表（高频 partial 读时优）
CREATE TABLE trajectory_steps (
  trajectory_id UUID NOT NULL REFERENCES trajectories(id) ON DELETE CASCADE,
  step_index    INT  NOT NULL,
  role          TEXT NOT NULL,
  content       TEXT,
  tool_name     TEXT,
  tool_input    JSONB,
  tool_output   TEXT,
  PRIMARY KEY (trajectory_id, step_index)
);
-- 缺点：JOIN 多

-- 方案 C：Parquet on S3 + 元数据在 DB
-- 大规模用，本题非必需
```

**参考 endpoint（FastAPI）**：

```python
from fastapi import FastAPI, HTTPException, Query
from pydantic import BaseModel
from typing import Optional
import asyncpg

app = FastAPI()
pool: asyncpg.Pool = None  # init at startup

@app.on_event("startup")
async def startup():
    global pool
    pool = await asyncpg.create_pool(dsn=..., min_size=2, max_size=10)

class TrajectoryMeta(BaseModel):
    id: str
    model: str
    task_id: str
    outcome: str
    step_count: int
    duration_ms: int
    created_at: str

@app.get("/trajectory/{tid}")
async def get_trajectory(tid: str):
    async with pool.acquire() as conn:
        meta = await conn.fetchrow("SELECT * FROM trajectories WHERE id = $1", tid)
        if not meta:
            raise HTTPException(404)
        steps = await conn.fetch(
            "SELECT * FROM trajectory_steps WHERE trajectory_id = $1 ORDER BY step_index",
            tid,
        )
        return {**dict(meta), "steps": [dict(s) for s in steps]}

@app.get("/trajectory")
async def list_trajectory(
    model: Optional[str] = None,
    task_id: Optional[str] = None,
    cursor: Optional[str] = None,  # base64-encoded created_at
    limit: int = Query(default=20, le=100),
):
    where = []
    params = []
    if model:
        params.append(model); where.append(f"model = ${len(params)}")
    if task_id:
        params.append(task_id); where.append(f"task_id = ${len(params)}")
    if cursor:
        import base64
        ts = base64.b64decode(cursor).decode()
        params.append(ts); where.append(f"created_at < ${len(params)}")
    where_sql = "WHERE " + " AND ".join(where) if where else ""

    params.append(limit + 1)  # +1 to detect next
    sql = f"""
      SELECT id, model, task_id, outcome, step_count, duration_ms, created_at
      FROM trajectories {where_sql}
      ORDER BY created_at DESC
      LIMIT ${len(params)}
    """
    async with pool.acquire() as conn:
        rows = await conn.fetch(sql, *params)
    has_next = len(rows) > limit
    items = [dict(r) for r in rows[:limit]]
    next_cursor = None
    if has_next:
        import base64
        next_cursor = base64.b64encode(
            items[-1]["created_at"].isoformat().encode()
        ).decode()
    return {"items": items, "next_cursor": next_cursor}
```

**追问 + 应对**：
- "10K/天 6 个月 = 1.8M 行，单表性能" → 单表撑得住；可按月 partition
- "什么时候用 ClickHouse / DuckDB" → 当 OLAP 多于 OLTP；做聚合 dashboard 用 DuckDB on parquet
- "怎么处理超大 steps 列" → 移到 S3，DB 存 URL + checksum
- "并发请求 query 同 trajectory 怎么 cache" → Redis cache aside + ETag

### 题 2·实现 SSE Stream Endpoint（25 分钟）

**题面**：实现一个 `/trajectory/{id}/stream` 把 trajectory 步骤一步一步以 SSE 推送给前端，模拟真实回放速度。

**评分点**：
- ✅ SSE 格式正确（`event:` + `data:` + `\n\n`）
- ✅ 异步生成器
- ✅ 客户端断开时优雅停止
- ✅ heartbeat（每 15s 一个空 comment 防代理超时）
- ✅ 错误处理（trajectory 不存在 emit error event）

**参考实现**：

```python
from fastapi import FastAPI, Request
from sse_starlette.sse import EventSourceResponse
import asyncio
import json

@app.get("/trajectory/{tid}/stream")
async def stream(tid: str, request: Request):
    async def event_gen():
        traj = await fetch_trajectory(tid)
        if traj is None:
            yield {"event": "error", "data": json.dumps({"msg": "not found"})}
            return

        last_heartbeat = asyncio.get_event_loop().time()
        for step in traj["steps"]:
            if await request.is_disconnected():
                return  # client 走了，停
            yield {"event": "step", "data": json.dumps(step)}
            await asyncio.sleep(0.1)  # 模拟节奏
            now = asyncio.get_event_loop().time()
            if now - last_heartbeat > 15:
                yield {"event": "heartbeat", "data": ""}
                last_heartbeat = now

        yield {"event": "done", "data": "ok"}

    return EventSourceResponse(event_gen())
```

**追问**：
- "WebSocket vs SSE 怎么选" → SSE 单向 + 简单；WebSocket 双向（如允许前端发 pause/seek）
- "Backpressure（前端慢消费）" → SSE 内置不显式 backpressure，server 端 buffer 满会 block；考虑 WebSocket
- "反向代理（nginx）超时" → 必须 heartbeat + nginx `proxy_read_timeout` 调大

### 题 3·LLM Tool Call 解析器（30 分钟）

**题面**：写一个 function 接收任意 LLM API 的 raw response（OpenAI / Anthropic / DeepSeek），返回 normalized `ToolCall[]`。

**评分点**：
- ✅ 识别三家不同 format
- ✅ OpenAI 的 `arguments` 是 string，要 JSON.parse + error handling
- ✅ 返回统一 schema
- ✅ 主动谈 "stream 时怎么 incremental 拼"

**参考实现**：

```python
from typing import Literal, Union
from pydantic import BaseModel
import json

class ToolCall(BaseModel):
    id: str
    name: str
    input: dict

def parse_tool_calls(raw: dict, provider: Literal['openai', 'anthropic', 'deepseek']) -> list[ToolCall]:
    if provider == 'anthropic':
        # content 是 list of blocks
        blocks = raw.get('content', [])
        return [
            ToolCall(id=b['id'], name=b['name'], input=b['input'])
            for b in blocks if b.get('type') == 'tool_use'
        ]
    if provider in ('openai', 'deepseek'):
        # message.tool_calls
        calls = raw.get('choices', [{}])[0].get('message', {}).get('tool_calls', [])
        result = []
        for c in calls:
            fn = c.get('function', {})
            args_str = fn.get('arguments', '{}')
            try:
                args = json.loads(args_str) if isinstance(args_str, str) else args_str
            except json.JSONDecodeError:
                # lenient fallback
                try:
                    import json5
                    args = json5.loads(args_str)
                except Exception:
                    args = {'__raw__': args_str, '__error__': 'invalid json'}
            result.append(ToolCall(id=c['id'], name=fn['name'], input=args))
        return result
    raise ValueError(f"Unknown provider {provider}")
```

### 题 4·Rust axum 写一个 trajectory upload endpoint（30 分钟）

**题面**：用 axum 写 `POST /trajectory` 接收 multipart upload（包含 metadata.json + steps.jsonl），落 DB。

**评分点**：
- ✅ multipart 解析
- ✅ 流式（不全 load 到 memory）
- ✅ 错误处理（用 thiserror + Into Response）
- ✅ 主动谈 max body size + content-type validation

**参考实现**：

```rust
use axum::{
    Router, routing::post,
    extract::{Multipart, State},
    http::StatusCode,
    response::IntoResponse,
};
use serde::{Serialize, Deserialize};
use sqlx::{PgPool, postgres::PgPoolOptions};
use thiserror::Error;

#[derive(Debug, Deserialize)]
struct Metadata {
    id: String, model: String, task_id: String,
    outcome: String, step_count: i32, total_tokens: i32, duration_ms: i32,
}

#[derive(Error, Debug)]
enum AppError {
    #[error("missing field {0}")] MissingField(&'static str),
    #[error("invalid json: {0}")] InvalidJson(#[from] serde_json::Error),
    #[error("db error: {0}")] Db(#[from] sqlx::Error),
    #[error("multipart error: {0}")] Multipart(#[from] axum::extract::multipart::MultipartError),
}

impl IntoResponse for AppError {
    fn into_response(self) -> axum::response::Response {
        let code = match self {
            AppError::MissingField(_) | AppError::InvalidJson(_) => StatusCode::BAD_REQUEST,
            _ => StatusCode::INTERNAL_SERVER_ERROR,
        };
        (code, self.to_string()).into_response()
    }
}

async fn upload(
    State(pool): State<PgPool>,
    mut multipart: Multipart,
) -> Result<impl IntoResponse, AppError> {
    let mut meta: Option<Metadata> = None;
    let mut steps_raw: Option<String> = None;

    while let Some(field) = multipart.next_field().await? {
        match field.name() {
            Some("metadata") => {
                let bytes = field.bytes().await?;
                meta = Some(serde_json::from_slice(&bytes)?);
            }
            Some("steps") => {
                let bytes = field.bytes().await?;
                steps_raw = Some(String::from_utf8_lossy(&bytes).into_owned());
            }
            _ => {}
        }
    }

    let meta = meta.ok_or(AppError::MissingField("metadata"))?;
    let steps_raw = steps_raw.ok_or(AppError::MissingField("steps"))?;

    let mut tx = pool.begin().await?;
    sqlx::query(
        "INSERT INTO trajectories (id, model, task_id, outcome, step_count, total_tokens, duration_ms)
         VALUES ($1, $2, $3, $4, $5, $6, $7)"
    )
    .bind(&meta.id).bind(&meta.model).bind(&meta.task_id)
    .bind(&meta.outcome).bind(meta.step_count)
    .bind(meta.total_tokens).bind(meta.duration_ms)
    .execute(&mut *tx).await?;

    for (i, line) in steps_raw.lines().enumerate() {
        let step: serde_json::Value = serde_json::from_str(line)?;
        sqlx::query(
            "INSERT INTO trajectory_steps (trajectory_id, step_index, role, content, tool_name, tool_input, tool_output)
             VALUES ($1, $2, $3, $4, $5, $6, $7)"
        )
        .bind(&meta.id).bind(i as i32)
        .bind(step["role"].as_str())
        .bind(step["content"].as_str())
        .bind(step["tool_name"].as_str())
        .bind(&step["tool_input"]).bind(step["tool_output"].as_str())
        .execute(&mut *tx).await?;
    }
    tx.commit().await?;
    Ok(StatusCode::CREATED)
}

#[tokio::main]
async fn main() {
    let pool = PgPoolOptions::new().connect(&std::env::var("DATABASE_URL").unwrap())
        .await.unwrap();
    let app = Router::new()
        .route("/trajectory", post(upload))
        .with_state(pool)
        .layer(axum::extract::DefaultBodyLimit::max(100 * 1024 * 1024));  // 100MB
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

### 题 5·LLM-as-Judge Scoring Pipeline（30 分钟）

**题面**：写一个 batch scoring service：拿 100 个 trajectory + scoring prompt，并发调 judge LLM，结果落表，能重跑失败的。

**评分点**：
- ✅ 并发 + 限速（rate limit）
- ✅ 错误隔离（一条 fail 不拖全部）
- ✅ 幂等（重跑跳过已成功的）
- ✅ Progress reporting

**参考实现**：

```python
import asyncio, json
from anthropic import AsyncAnthropic
from dataclasses import dataclass
from typing import Callable

@dataclass
class ScoreTask:
    trajectory_id: str
    prompt: str

@dataclass
class ScoreResult:
    trajectory_id: str
    score: float | None
    reason: str
    error: str | None

async def score_one(
    task: ScoreTask, client: AsyncAnthropic, sem: asyncio.Semaphore
) -> ScoreResult:
    async with sem:
        try:
            resp = await client.messages.create(
                model="claude-opus-4-5",
                max_tokens=512,
                messages=[{"role": "user", "content": task.prompt}],
            )
            text = resp.content[0].text
            data = json.loads(text)  # judge 必须返回 JSON
            return ScoreResult(task.trajectory_id, data["score"], data["reason"], None)
        except Exception as e:
            return ScoreResult(task.trajectory_id, None, "", str(e))

async def batch_score(
    tasks: list[ScoreTask], max_concurrency: int = 8,
    save: Callable[[ScoreResult], None] = lambda r: None,
    on_progress: Callable[[int, int], None] = lambda d, t: None,
) -> list[ScoreResult]:
    client = AsyncAnthropic()
    sem = asyncio.Semaphore(max_concurrency)

    async def wrap(idx: int, t: ScoreTask):
        r = await score_one(t, client, sem)
        save(r)
        on_progress(idx + 1, len(tasks))
        return r

    return await asyncio.gather(*[wrap(i, t) for i, t in enumerate(tasks)])

# 幂等 wrapper：跳过已经评过的
async def batch_score_resumable(tasks: list[ScoreTask], conn):
    existing = await conn.fetch(
        "SELECT trajectory_id FROM scores WHERE trajectory_id = ANY($1)",
        [t.trajectory_id for t in tasks]
    )
    done = {r['trajectory_id'] for r in existing}
    remaining = [t for t in tasks if t.trajectory_id not in done]

    async def save(r: ScoreResult):
        if r.error:
            return  # 错误的不存，下次重跑
        await conn.execute(
            "INSERT INTO scores (trajectory_id, score, reason) VALUES ($1, $2, $3) "
            "ON CONFLICT (trajectory_id) DO NOTHING",
            r.trajectory_id, r.score, r.reason
        )

    return await batch_score(remaining, save=save)
```

**追问**：
- "judge LLM 偶尔输出非 JSON 怎么办" → JSON mode / structured output；fallback：retry 1 次 + lenient parse
- "并发太高把 LLM 限流" → 看 429 header `retry-after` + token bucket 自适应
- "如何保 audit 重现" → save 时同时存原始 response

## 二、必背 5 个后端"系统问题"答案

### Q1·从写完到上 prod 一次完整 release 流程是？

> "Branch → PR → CI（lint + type check + test + build）→ peer review → merge to main → CD build image push registry → ArgoCD / Helm sync 到 staging → smoke test → manual promote 到 prod with canary 5% → 全量 → 观察 metrics/log 24h → 如有异常 rollback。
> 全链关键 controls：每个 commit 都能 rollback；image 用 sha 而不是 latest tag；DB migration 必须可向前兼容 + 两步部署。"

### Q2·Postgres 慢查询定位流程？

> "1. `pg_stat_statements` 看 top N； 2. `EXPLAIN ANALYZE` 看执行计划； 3. 看 seq scan / nested loop / sort； 4. 加 index / 改写 query / 优化 join 顺序； 5. 多列复合索引看 leftmost prefix； 6. 长事务用 `pg_stat_activity` 找； 7. 如果数据量太大 → partition / 移到 OLAP（DuckDB / ClickHouse）。"

### Q3·Python OOM 怎么 debug？

> "1. tracemalloc 看 top allocations； 2. 检查是否大对象长生命周期（cache / module-level list 一直加）； 3. pandas 一次 load 整 file → 改流式 chunksize； 4. generator 化所有 list comprehension on 大数据； 5. 用 memory_profiler 标可疑函数； 6. 容器 OOM：K8s 看 memory usage curve + 把 limit/request 写对； 7. 极端：Python 是 GIL 单进程，可以拆成 worker pool 让单进程小。"

### Q4·Rust async runtime 选 tokio 还是 async-std？

> "默认 tokio——生态最完整（axum / sqlx / reqwest 都基于它）；async-std 已经基本停更。
> tokio 内部 runtime 选 `multi_thread`（默认）支持 work-stealing；CPU 密集任务用 `spawn_blocking` 移出 worker，别堵 reactor。"

### Q5·gRPC vs REST vs GraphQL 怎么选？

> "REST：通用，前端友好，cache 好；
> gRPC：内部服务，强类型，性能高，streaming 好；
> GraphQL：前端按需取，复杂查询少 round trip，但 cache / 复杂度控制难。
>
> 我们这个评测平台主对前端：REST + OpenAPI 最稳；trajectory 推送用 SSE。如果内部 RL trainer ↔ Agent 容器 ↔ scheduler 三者通信，gRPC + protobuf 合适——schema 演进强。"
