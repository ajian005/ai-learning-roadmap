## 二、每个岗位的"画像"一句话

### PM 岗
> **"想清楚 + 协调 5 方 + 推动 ship 的 generalist——不写 production 代码但写好的 spec。"**

### Harness Eng 岗
> **"造给真实用户用的 Code Agent 产品本身——和研究员对话级深技术 + 工程美感 + 0→1 ship 力。"**

### 全栈 Eng 岗（本目录）
> **"造给内部 50 个算法每天用的评测平台 + 容器服务 + framework 适配层——前端 + 后端 + Infra 三栈都能扛 + 集成业界主流 Agent SDK。"**

## 三、谁该投哪个？

### 投 PM 的典型画像

- 主要经验是 **产品经理**（无论 AI / SaaS / 工具类）
- 写代码靠 vibe coding，不算系统训练
- 2~5 年定义产品 + 推动跨职能经验
- 对**用户研究、A/B 测试、统计学**有系统训练
- 能用英文写得体的 RFC
- 喜欢 stakeholder 协调

### 投 Harness Eng 的典型画像

- 主要经验是**软件工程**（后端 / 前端 / infra / 系统）
- 能独立 ship production grade 代码
- 写过技术博客 / 贡献过开源
- 享受**和研究员讨论 paper** 多过对齐 PRD
- 学新技术栈很快
- 喜欢深聊 KV cache / agent loop 实现细节
- 有比赛 / 论文 / 开源信号

### 投 全栈 Eng 的典型画像（本岗）

- 主要经验是**全栈工程**（同时做过前端 + 后端 + 至少接触过 K8s）
- 喜欢造**内部工具**多过造对外产品
- 能容忍"维护别人的依赖升级"这种活
- 能享受**算法找你催"我的 trajectory 看不到"** 的紧迫感
- 对 **LangChain / OAI SDK 二次开发** 有兴趣或经验
- 安全 / CTF / 红队思维是 bonus
- 学历不是顶尖 / 没 paper 也能进——能力比标签重要

## 五、三岗"日常一周"模拟

### PM 的一周

```
Mon: sprint planning + research 同步 + 用户访谈
Tue: 数据看板 + A/B 实验 + 写 PRD
Wed: 1:1 工程师 / 研究员对齐
Thu: dogfood + vibe coding demo
Fri: 周会 + 数据 review + paper / competitor 跟进
```

模式：80% 对齐决策反馈；20% 自己 ship

### Harness Eng 的一周

```
Mon: sprint planning + 选 task + design doc
Tue: 写代码 + 测试 + code review
Wed: 和研究员深聊新 feature 接口
Thu: 调试 + perf + 实现 + ship
Fri: 周会 + dogfood telemetry + 看 paper / 源码
```

模式：70% 写代码；30% 对齐 + 反馈

### 全栈 Eng 的一周（本岗）

```
Mon: sprint planning + 看 user request（算法 backlog）
Tue: 前端 component / 后端 endpoint 实现
Wed: K8s manifest / Helm chart / CI tweak
Thu: LangChain 升级适配 + 容器依赖管理
Fri: 算法 1:1 收 feedback + on-call shadow（如果有）
```

模式：50% 写新东西；30% 维护现有；20% 跨界沟通

## 七、混合策略：投多个吗？

### 一次投一个（推荐）

- 团队内部清楚你的定位
- 显示 commitment + 思考清晰
- 缺点：拒了等几个月再投

### 间隔 1~2 周投两个

- 优点：增加进面试概率
- 缺点：HR 可能问"你为什么也投另一个"——必须能讲清差异

### 三个一起投（不推荐）

- 显示"any job will do"——对小团队反感

> **建议**：选你 self-evaluation 最高的那个先投；拒了再投第二个；不要同时投三个。

## 十、最后建议

> 三个岗位都在 DeepSeek 同一个"前沿 + 长期主义 + 研究密度高"的 talent dense 区域。
>
> 选哪个不是"哪个机会大"，是"哪个我能持续燃烧热情 3 年"。
>
> - **全栈 Eng** 是这三个里**最容易被低估的**——很多候选人觉得"全栈 = generalist = 廉价"，事实相反：能造内部工具 + 跨栈 + 集成第三方的工程师在 AI infra 时代极稀缺
> - **如果你已经"会写前端 + 会 K8s + 用过 LangChain" 这三件事**——本岗对你 friction 最低
> - **如果你 70% 时间在写后端 / 系统 / 偏 Harness 工程深度**——优先考虑 Harness Eng
> - **如果你 70% 时间在做产品判断 / 用户研究**——投 PM
>
> 选下来后配套准备包全力 7 天——offer 不取决于天赋，取决于这 7 天有多 specific 准备。
>
> **祝顺利。**
