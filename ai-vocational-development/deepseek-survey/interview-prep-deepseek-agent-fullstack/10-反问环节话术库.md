# 10 · 反问环节话术库（全栈版）

> PM 版（[../interview-prep-deepseek-harness-pm/08-反问环节话术库.md](../interview-prep-deepseek-harness-pm/08-反问环节话术库.md)）和 Eng 版（[../interview-prep-deepseek-harness-engineer/08-反问环节话术库.md](../interview-prep-deepseek-harness-engineer/08-反问环节话术库.md)）大部分通用，本文档只列**全栈岗特化的差异**。

## 一、给不同面试官的全栈特化反问

### 给 HR（first round）

通用问题 + **全栈特化**：

1. "面试有几轮 coding（前端 / 后端 / Infra）？是混合一轮还是分开？"
2. "Coding 是白板手写、共享屏幕 IDE、还是 take-home？前端用什么环境？"
3. "我能否提前知道大概的技术选型？这帮我准备得更具体（比如前端是 React 还是 Vue / 后端 Python 还是 Rust 更多）"
4. "这个岗位是 IC 走深还是希望我开始带 mini team？"
5. "团队 onboarding 周期是？有 buddy 系统吗？"

### 给同 team 工程师（peer interview）

**全栈岗最关键的一轮**——你的未来同事在 evaluate "他会不会成为我的 partner"。

1. "你们前端栈具体是？React / Vue / Solid？build tool / 测试栈是？"
2. "你们后端 Python / Rust 比例大概是？什么场景用 Rust？"
3. "K8s 集群规模是？你们用 Helm 还是 Kustomize 还是 ArgoCD？"
4. "你们 CI/CD 是 GitHub Actions / GitLab CI / Jenkins / Spinnaker？"
5. "Code review 文化是什么样？PR 平均合多久？是否 1 票通过？"
6. "你们怎么决定写测试？unit coverage 目标？E2E 用什么？"
7. "你们用什么 LangChain / OAI SDK / 自研框架更多？为什么？"
8. "现在最让你头疼的工程债是什么？"
9. "如果我入职，第一周做什么 task 你觉得最有 ramp 价值？"
10. "你最希望前任 / 候选人具备但缺的能力是什么？"

### 给算法 / 研究员（最重要的"差异判断" round）

> 同 Eng 包 Q3、Q5、Q7。**全栈岗特化补**：

1. "目前评测平台 / 轨迹查看器 / 容器服务 哪一块你最不满意？"
2. "如果我入职，第一个月你最希望我修哪个 pain point？"
3. "你 daily debug 一个 fail trajectory 大概要多久？瓶颈在哪？"
4. "你最希望 platform team 帮你做的'最低延迟反馈'是什么形态？"
5. "你怎么看 DeepSeek 内部 RL infra 这一年的演进？"

### 给 SRE / DevOps（如果有这一轮）

1. "你们集群 K8s 版本是？升级节奏？"
2. "On-call 怎么轮？平均一周多少 page？"
3. "你们 observability stack 是？Prometheus / Grafana / Loki / Jaeger / Datadog？"
4. "Storage 用什么？S3 / Ceph / 自建？"
5. "你期望全栈工程师在 Infra 这块做多少 vs SRE 做多少？"
6. "团队最近最严重的一次 incident 是什么？怎么解决？事后改了什么？"
7. "你们 SLO 怎么定？错误预算 burn rate 用吗？"

### 给 Leader / 团队 Lead

通用问题 + 全栈特化：

1. "这个岗位 1 年后理想结果是什么样？怎么衡量？"
2. "你看 5 年内 Agent infra / 评测平台 这个方向的演进会怎样？"
3. "团队目前在前端 / 后端 / Infra 哪一块最缺人？我可能弥补哪个 gap？"
4. "你最希望 platform team 文化里有什么 traits？最讨厌什么？"
5. "如果我入职 3 个月做不出大东西，你的评估机制是什么？"
6. "你会不会希望我业余持续做开源 / 写技术博客？团队对外部 visibility 怎么看？"

## 二、5 个全栈"金问题"

### 1·"团队过去 6 个月修过最大的工程债是什么？"

> 测团队 self-improvement 能力。能改的团队 > 永不改的团队。

### 2·"你心目中 ideal 全栈工程师是 70/20/10（前/后/Infra） 还是 33/33/33？"

> 让对方告诉你团队对"全栈"的真实定义——避免入职后期望不匹配。

### 3·"如果我入职后想推动从 Python 全栈到 Rust 重构核心 service，团队的接受度大概是？"

> 测团队的技术债务/演进自由度——给你的"主权空间"信号。

### 4·"现在 LangChain / OAI Agents SDK 之外，团队最想加但还没加的 Agent framework 是什么？"

> 让 leader 告诉你 roadmap——他能给具体回答说明 strategy 清晰。

### 5·"你团队最近一次 PR fail merge 的根因是什么？"

> 测 code review 文化深度——好团队会拒绝技术债 PR。

## 三、按面试场次分配反问额度（全栈版）

假设 5~7 轮：

| 轮次 | 面试官 | 推荐问 |
| --- | --- | --- |
| Round 1·HR | 流程 / 团队 / 节奏 | HR 第 1-3 |
| Round 2·前端 coding | 简短反问 1~2 | 工程师 1, 2 |
| Round 3·后端 coding | 简短反问 1~2 | 工程师 3, 4 |
| Round 4·System Design | 简短反问 2 | 工程师 9, 10 |
| Round 5·算法 / 研究员 | 深技术 + 真实痛点 | 算法 1, 2, 3 |
| Round 6·SRE（如有） | 运维 + observability | SRE 1, 2, 6 |
| Round 7·Leader | 战略 + 文化 + 金问题 | Leader 1, 2, 6 + 金问题 1, 2 |

## 四、反问的细节（同 PM 版铁律）

- 每个问题前**先说半句话引出**："你刚才提到 X，让我想到..."
- 问完一个对答案做回应："你这个回答让我对 Y 的判断更准了"
- 最后留 1 个 **placeholder 问题**：listening 好就深聊，时间紧就跳

---

## 五、绝对不能问的（全栈特化）

| 反例 | 为什么 |
| --- | --- |
| "你们前端 / 后端 / Infra 哪个我以后做更多？" | 像在挑活；显得没主动性 |
| "996 吗？" | 给负面信号 |
| "工资多少？" | 给 HR 集中谈 |
| "可以远程吗？" | 工程师轮先不问 |
| "你们栈是？" | HR 阶段问过；到工程师面再问显得没准备 |
| "你们用 Cursor 吗？" | 当然用——这是 silly 问题 |

---

## 六、一句应试钥匙

> **反问环节本质是：你将来在 1:1 / design review / on-call sync 里会问的问题的彩排。**
>
> **让面试官感觉到 "如果他来了，我每周愿意和他一起 sync 30 分钟"——这才是 offer 的 signal。**
