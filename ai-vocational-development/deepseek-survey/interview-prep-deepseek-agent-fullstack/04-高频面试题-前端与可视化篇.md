### 题 1·实现 Trajectory Timeline 组件（30~45 分钟）

**题面**：用 React + TS 写一个组件，展示一个 agent trajectory 的时间线。需求：
- 输入：`steps: Step[]`，每个 step 有 `index / role / content / tool_name / tool_input / ts`
- 显示：时间线 + 每步可展开看详情
- filter：按 role 过滤 / 按 tool 过滤
- search：输入框 substring 搜
- 加分：当前选中 step 高亮 + 可键盘上下切换

**评分点**：
- ✅ TS types 写得对（discriminated union 处理 step kind）
- ✅ filter / search 用 useMemo 缓存
- ✅ 至少基本的 a11y（aria-label / button role）
- ✅ 性能意识（10K steps 时怎么办：virtual list / pagination）
- ✅ 主动谈"if I have time, I'd add diff view / sticky header / URL state"

**参考实现骨架**：

```tsx
import { useState, useMemo, useDeferredValue } from 'react';

type Role = 'user' | 'assistant' | 'tool';

interface BaseStep { index: number; ts: number; }
interface UserStep extends BaseStep { role: 'user'; content: string; }
interface AssistantStep extends BaseStep {
  role: 'assistant';
  content: string;
  tool_calls?: { id: string; name: string; input: unknown }[];
}
interface ToolStep extends BaseStep {
  role: 'tool';
  tool_use_id: string;
  tool_name: string;
  output: string;
}
type Step = UserStep | AssistantStep | ToolStep;

interface Props {
  steps: Step[];
  selectedIndex?: number;
  onSelect?: (i: number) => void;
}

export function TrajectoryTimeline({ steps, selectedIndex, onSelect }: Props) {
  const [search, setSearch] = useState('');
  const [roleFilter, setRoleFilter] = useState<Set<Role>>(
    new Set(['user', 'assistant', 'tool'])
  );
  const deferredSearch = useDeferredValue(search);

  const filtered = useMemo(() => {
    const q = deferredSearch.toLowerCase();
    return steps.filter(s => {
      if (!roleFilter.has(s.role)) return false;
      if (!q) return true;
      const text = JSON.stringify(s).toLowerCase();
      return text.includes(q);
    });
  }, [steps, roleFilter, deferredSearch]);

  return (
    <div className="flex flex-col h-full">
      <header className="flex gap-2 p-2 border-b">
        <input
          aria-label="search trajectory"
          placeholder="search..."
          value={search}
          onChange={e => setSearch(e.target.value)}
          className="flex-1 px-2 py-1 border rounded"
        />
        {(['user', 'assistant', 'tool'] as Role[]).map(r => (
          <label key={r} className="flex items-center gap-1">
            <input
              type="checkbox"
              checked={roleFilter.has(r)}
              onChange={e => {
                const next = new Set(roleFilter);
                e.target.checked ? next.add(r) : next.delete(r);
                setRoleFilter(next);
              }}
            />
            {r}
          </label>
        ))}
      </header>

      <ol className="flex-1 overflow-auto" role="list">
        {filtered.map(step => (
          <StepRow
            key={step.index}
            step={step}
            selected={step.index === selectedIndex}
            onClick={() => onSelect?.(step.index)}
          />
        ))}
      </ol>

      <footer className="text-xs p-1 border-t">
        Showing {filtered.length} / {steps.length} steps
      </footer>
    </div>
  );
}

function StepRow({ step, selected, onClick }:
  { step: Step; selected: boolean; onClick: () => void }) {
  const [open, setOpen] = useState(false);
  const bg = selected ? 'bg-blue-100' : 'hover:bg-gray-50';
  return (
    <li className={`border-b ${bg}`}>
      <button
        className="w-full text-left p-2 flex items-baseline gap-3"
        onClick={() => { onClick(); setOpen(o => !o); }}
        aria-expanded={open}
      >
        <span className="text-xs text-gray-500 w-12">#{step.index}</span>
        <RoleBadge role={step.role} />
        <span className="truncate flex-1">{summarize(step)}</span>
      </button>
      {open && (
        <pre className="text-xs bg-gray-50 p-2 overflow-x-auto">
          {JSON.stringify(step, null, 2)}
        </pre>
      )}
    </li>
  );
}

function RoleBadge({ role }: { role: Role }) {
  const map = { user: 'bg-green-200', assistant: 'bg-blue-200', tool: 'bg-purple-200' };
  return <span className={`text-xs px-2 rounded ${map[role]}`}>{role}</span>;
}

function summarize(step: Step): string {
  if (step.role === 'user') return step.content.slice(0, 80);
  if (step.role === 'assistant')
    return step.content?.slice(0, 80) ||
      `[${step.tool_calls?.length ?? 0} tool calls]`;
  return `[${step.tool_name}] ${step.output.slice(0, 60)}`;
}
```

**追问 + 应对**：
- "10K steps scroll 卡顿怎么办？" → TanStack Virtual / react-window；filter 也要 incremental
- "URL 状态怎么同步？" → 用 search params / TanStack Router useSearch
- "想 share 当前 filter 给同事怎么办？" → 把状态序列化到 URL 即可
- "增量加载呢？" → server pagination + virtual list + intersection observer

### 题 2·实现两个 Trajectory Diff View（30 分钟）

**题面**：左右对比两个 trajectory，按 step 对齐高亮差异（model A 在第 3 步多调了一次 tool）。

**评分点**：
- ✅ 知道这是 sequence alignment 问题（类比 git diff）
- ✅ 用 LCS 算法对齐 step
- ✅ UI 三栏（A / 对齐符 / B）
- ✅ 主动谈"steps 不能完全 1:1 对齐时怎么 fallback"

**参考思路**：

```ts
// 1. 对齐：动态规划 LCS（O(N×M)）
function alignSteps(a: Step[], b: Step[]) {
  // dp[i][j] = LCS length matching a[0..i], b[0..j]
  const n = a.length, m = b.length;
  const dp: number[][] = Array.from({ length: n + 1 }, () =>
    Array(m + 1).fill(0)
  );
  for (let i = 1; i <= n; i++)
    for (let j = 1; j <= m; j++)
      if (stepEqual(a[i - 1], b[j - 1]))
        dp[i][j] = dp[i - 1][j - 1] + 1;
      else
        dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);

  // backtrack to produce aligned pairs
  const pairs: [Step | null, Step | null][] = [];
  let i = n, j = m;
  while (i > 0 && j > 0) {
    if (stepEqual(a[i - 1], b[j - 1])) {
      pairs.push([a[i - 1], b[j - 1]]); i--; j--;
    } else if (dp[i - 1][j] >= dp[i][j - 1]) {
      pairs.push([a[i - 1], null]); i--;
    } else {
      pairs.push([null, b[j - 1]]); j--;
    }
  }
  while (i > 0) { pairs.push([a[--i], null]); }
  while (j > 0) { pairs.push([null, b[--j]]); }
  return pairs.reverse();
}

function stepEqual(a: Step, b: Step): boolean {
  if (a.role !== b.role) return false;
  if (a.role === 'assistant' && b.role === 'assistant') {
    // tool 相同算同；content 不同打 partial
    return JSON.stringify(a.tool_calls) === JSON.stringify(b.tool_calls);
  }
  return JSON.stringify(a) === JSON.stringify(b);
}

// 2. 渲染三栏（伪代码）
function DiffView({ a, b }: { a: Step[]; b: Step[] }) {
  const aligned = useMemo(() => alignSteps(a, b), [a, b]);
  return (
    <table>
      {aligned.map((pair, i) => (
        <tr key={i}>
          <td className={pair[0] && pair[1] ? '' : 'bg-red-100'}>
            {pair[0] && <StepCell step={pair[0]} />}
          </td>
          <td>{pair[0] && pair[1] ? '=' : pair[0] ? '<' : '>'}</td>
          <td className={pair[0] && pair[1] ? '' : 'bg-green-100'}>
            {pair[1] && <StepCell step={pair[1]} />}
          </td>
        </tr>
      ))}
    </table>
  );
}
```

**追问**：
- "step 不能 1:1 比较时（如 tool input 不同算同还是异）怎么办" → 给 equality function 多个 mode（strict / structural / semantic）
- "trajectory 5000 step LCS O(N²) 慢吗" → 5000² = 25M，浏览器能跑但要 worker；或用 Myers diff O((N+M)D)

### 题 3·实现 Streaming JSON Renderer（20 分钟）

**题面**：后端 SSE 推 partial JSON chunk（tool_use 的 input_json_delta），前端**实时**渲染最新 best-effort 的对象。

**评分点**：
- ✅ 知道 partial JSON 不能 JSON.parse
- ✅ 用 lenient parser（partial-json / json5）兜底
- ✅ debounce 渲染（不要每 chunk 重 render）
- ✅ stream 结束后用 strict parse 校验

**参考实现**：

```tsx
import { useState, useEffect } from 'react';
import { parse as partialParse } from 'partial-json';  // npm i partial-json

function useStreamingJSON(url: string) {
  const [json, setJson] = useState<unknown>(null);
  const [error, setError] = useState<Error | null>(null);
  const [done, setDone] = useState(false);

  useEffect(() => {
    const es = new EventSource(url);
    let buffer = '';
    es.addEventListener('input_json_delta', (e) => {
      buffer += (e as MessageEvent).data;
      try { setJson(partialParse(buffer)); } catch {}
    });
    es.addEventListener('done', () => {
      try { setJson(JSON.parse(buffer)); }
      catch (e) { setError(e as Error); }
      setDone(true);
      es.close();
    });
    es.onerror = (e) => { setError(new Error('stream error')); es.close(); };
    return () => es.close();
  }, [url]);

  return { json, error, done };
}
```

### 题 4·实现 Leaderboard 多模型横评 Table（30 分钟）

**题面**：展示一个模型 × benchmark 二维表，每格是分数；支持按列排序 / 隐藏列 / 高亮最大值 / sticky header / 数据 5000 行。

**评分点**：
- ✅ 用 TanStack Table（不要手撸）
- ✅ 列定义 + 排序 + filter
- ✅ 性能：virtual row
- ✅ 主动谈"sparse data 怎么显示（缺值）/ 颜色尺度"

**关键代码**：

```tsx
import { useReactTable, getCoreRowModel, getSortedRowModel,
         flexRender, ColumnDef, SortingState } from '@tanstack/react-table';
import { useState, useMemo } from 'react';

interface Row { model: string; [bench: string]: number | string; }

function Leaderboard({ data, benchmarks }: { data: Row[]; benchmarks: string[] }) {
  const [sorting, setSorting] = useState<SortingState>([]);

  const columns = useMemo<ColumnDef<Row>[]>(() => [
    { accessorKey: 'model', header: 'Model' },
    ...benchmarks.map(b => ({
      accessorKey: b,
      header: b,
      cell: ({ getValue, column }: any) => {
        const v = getValue() as number | undefined;
        if (v === undefined) return <span className="text-gray-300">—</span>;
        // highlight max
        const max = Math.max(...data.map(r => Number(r[b] ?? 0)));
        return <span className={v === max ? 'font-bold' : ''}>{v.toFixed(2)}</span>;
      },
    })),
  ], [benchmarks, data]);

  const table = useReactTable({
    data, columns, state: { sorting }, onSortingChange: setSorting,
    getCoreRowModel: getCoreRowModel(), getSortedRowModel: getSortedRowModel(),
  });

  return (
    <table className="w-full text-sm">
      <thead className="sticky top-0 bg-white">
        {table.getHeaderGroups().map(hg => (
          <tr key={hg.id}>
            {hg.headers.map(h => (
              <th
                key={h.id}
                onClick={h.column.getToggleSortingHandler()}
                className="cursor-pointer p-1 border-b text-left"
              >
                {flexRender(h.column.columnDef.header, h.getContext())}
                {{ asc: ' ▲', desc: ' ▼' }[h.column.getIsSorted() as string] ?? null}
              </th>
            ))}
          </tr>
        ))}
      </thead>
      <tbody>
        {table.getRowModel().rows.map(row => (
          <tr key={row.id} className="hover:bg-gray-50">
            {row.getVisibleCells().map(cell => (
              <td key={cell.id} className="p-1 border-b">
                {flexRender(cell.column.columnDef.cell, cell.getContext())}
              </td>
            ))}
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

### 题 5·debounced search + cancelable fetch（20 分钟）

**题面**：用户输入快速变化时不能每个字 trigger fetch；并且新请求应取消旧请求。

**评分点**：
- ✅ debounce
- ✅ AbortController
- ✅ TanStack Query / SWR 优雅做法

**参考**：

```tsx
import { useState, useEffect } from 'react';
import { useQuery } from '@tanstack/react-query';

function useDebounced<T>(value: T, ms: number) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const t = setTimeout(() => setDebounced(value), ms);
    return () => clearTimeout(t);
  }, [value, ms]);
  return debounced;
}

function SearchBox() {
  const [q, setQ] = useState('');
  const debouncedQ = useDebounced(q, 300);

  const { data, isFetching } = useQuery({
    queryKey: ['search', debouncedQ],
    queryFn: ({ signal }) =>
      fetch(`/api/search?q=${encodeURIComponent(debouncedQ)}`, { signal })
        .then(r => r.json()),
    enabled: debouncedQ.length >= 2,
    staleTime: 30_000,
  });

  return (
    <div>
      <input value={q} onChange={e => setQ(e.target.value)} />
      {isFetching && <span>...</span>}
      <ul>{(data ?? []).map((d: any) => <li key={d.id}>{d.title}</li>)}</ul>
    </div>
  );
}
```

**口头补充**：TanStack Query 默认会用 abort signal cancel 上一个未完成的 query，不需要手动 AbortController。

## 二、可视化深问 Q&A

### Q1·渲染一个 1000 节点的 agent 决策树用什么？

> "节点数 1K 量级，svg 直接 render；> 10K 上 canvas。
> 用 d3-hierarchy 算 layout（tree / cluster / pack 三选一）；交互层用 React state 控展开收起。
> 如果要 force-directed graph 用 d3-force 但要 Web Worker 避免主线程卡。
> 如果是 trajectory 时间线一类的 left-to-right 严格顺序 → 不用图，用 list + indent 视觉化更快。"

### Q2·实时 dashboard 推送 100 metrics/sec 怎么不卡 chart？

> "三层 throttle：
> 1. WebSocket 接收时 buffer 1 秒
> 2. 用 requestAnimationFrame 合并 render
> 3. Chart 库选支持 incremental update 的（recharts 不行，uplot / echarts streaming 可以）。
> 数据保留滑窗（如最近 5min），老的扔掉。"

### Q3·前端怎么实现 trajectory 的 "step playback"（点击播放从头跑一遍）？

> "状态机：currentStep / playing / speed。
> setInterval 按 speed (e.g. 1 step / 200ms) 增 currentStep。
> 渲染时只 mark 当前 step 高亮 + scroll into view。
> 加 keyboard: Space play/pause, ←→ 单步。
> 用 useRef 避 stale closure；clear interval 必须 cleanup。"

## 三、3 个常见 React 陷阱（必背）

### 陷阱 1·stale closure

```tsx
// ❌ Bug: count 永远是 0
useEffect(() => {
  const id = setInterval(() => setCount(count + 1), 1000);
  return () => clearInterval(id);
}, []);

// ✅ 用 functional updater
useEffect(() => {
  const id = setInterval(() => setCount(c => c + 1), 1000);
  return () => clearInterval(id);
}, []);
```

### 陷阱 2·useEffect 依赖错

```tsx
// ❌ 依赖 user object，每次父 render 都触发
useEffect(() => fetchData(user), [user]);

// ✅ 依赖 user.id 字符串
useEffect(() => fetchData(user), [user.id]);
```

### 陷阱 3·条件渲染数据丢失

```tsx
// ❌ 卡片每次切换都 unmount，输入丢失
{tab === 'a' ? <TabA /> : <TabB />}

// ✅ 都挂着 + 用 hidden / opacity 隐藏
<div hidden={tab !== 'a'}><TabA /></div>
<div hidden={tab !== 'b'}><TabB /></div>
```
