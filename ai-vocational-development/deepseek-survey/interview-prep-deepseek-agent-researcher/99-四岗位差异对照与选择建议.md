## 二、Org 中的位置 + 协作关系

```
         ┌──────────────────────────────────┐
         │       研究员 (Researcher)         │
         │   训模型 + 设计算法 + 写 paper    │
         └──────────────────────────────────┘
                     ▲           │
              评测/训练需求   产出 model weights
                     │           ▼
       ┌─────────────────────────────────────┐
       │     全栈工程师 (Fullstack Eng)        │
       │  评测平台 + 容器 + RL infra + adapter │
       └─────────────────────────────────────┘
                     ▲           │
                model + eval   infra 服务
                     │           ▼
         ┌──────────────────────────────────┐
         │     Harness 工程师 (Harness Eng)   │
         │  封装 Code Agent: tool / context  │
         │  / sandbox / hooks / UI           │
         └──────────────────────────────────┘
                     ▲           │
              产品需求/PRD   用户产品 ship
                     │           ▼
         ┌──────────────────────────────────┐
         │       产品经理 (PM)                │
         │  定义产品 + 路线图 + 指标        │
         └──────────────────────────────────┘
                     ▲
                 用户/市场
```

**核心洞察**：四岗位形成一个 **闭环**——研究员"造大脑"，全栈"造神经系统/医院"，Harness Eng "造身体"，PM "定义身体长什么样、做什么"。

## 四、自检：你适合哪一个？

### 决策树

```
Q1: 你能在 30 分钟内白板推导 DPO loss + GRPO advantage 吗？
├── 能 → Q2
└── 不能 → Q5

Q2: 你跑过 SFT → RM → RLHF / DPO 训练吗？
├── 跑过且懂 → Q3
└── 没跑过 → Q5

Q3: 你有顶会 paper 吗？
├── 有 → 👑 Researcher 是首选
└── 没有 → Q4

Q4: 你愿意花 1~2 周做 mini-DPO/GRPO 复现 吗？
├── 愿意 → 仍可冲 Researcher，加 [12 文档] 项目代偿
└── 不愿意 → 转 Fullstack Eng / Harness Eng

Q5: 你享受写产品代码 + 用户思考吗？
├── 是 → Q6
└── 不 → Q7

Q6: 偏前端 + 内部工具？还是偏面向终端用户的产品？
├── 内部工具 / 评测平台 → 🛠️ Fullstack Eng
└── 终端用户 / Code Agent 产品 → 🤖 Harness Eng

Q7: 你享受写 PRD + 用户访谈 + 数据分析吗？
├── 是 → 📋 PM
└── 不 → 重新审视职业方向
```

### 自检 Checklist (打分每岗)

每岗满分 10：

#### PM (10 分)
- [ ] 我能流畅写一份 PRD (2 分)
- [ ] 我做过用户访谈 (1 分)
- [ ] 我能用 SQL 拉数据分析 (1 分)
- [ ] 我熟 Claude Code/Cursor 等产品 (2 分)
- [ ] 我对 LLM Agent 产品形态有 thesis (2 分)
- [ ] 我享受跨团队沟通 / 协调 (2 分)

#### Harness Eng (10 分)
- [ ] 我能在白板写 mini Agent loop (2 分)
- [ ] 我熟 TypeScript / Python (2 分)
- [ ] 我做过 sandbox / permission 系统 (1 分)
- [ ] 我读过 Claude Code / Cursor 等 OSS 源码 (2 分)
- [ ] 我能 system design 一个完整 Harness (2 分)
- [ ] 我有 1+ 开源项目 / 大厂研发经验 (1 分)

#### Fullstack Eng (10 分)
- [ ] 我熟 React + FastAPI/Python + Postgres (2 分)
- [ ] 我熟 Docker / K8s (2 分)
- [ ] 我做过 internal tool / dashboard (1 分)
- [ ] 我用过 LangChain / OpenAI Agents SDK (1 分)
- [ ] 我熟 CI/CD + 监控 (2 分)
- [ ] 我能 system design 评测平台 (2 分)

#### Researcher (10 分)
- [ ] 我能默写 PPO/DPO/GRPO loss (2 分)
- [ ] 我跑过 SFT/RM/RLHF/DPO 中至少一项 (2 分)
- [ ] 我读过 DeepSeek-R1/V3 + 5 篇 alignment paper (2 分)
- [ ] 我有顶会 paper 或活跃开源 (1 分)
- [ ] 我熟 PyTorch + 至少能讲 CUDA/Triton (1 分)
- [ ] 我有 mini RL 复现项目（GitHub）(2 分)

**得分最高的就是你的最佳投递**。

## 五、如果一个岗没拿到，转投策略

### Researcher 没拿到？

**可转投：Fullstack Eng**

理由：
- 研究员 background 转 RL infra / 评测平台技术上 OK
- 一年内积累 infra 经验 → 内部转岗回 Researcher
- 已经准备的 PyTorch / RL 知识 + 实验经验都用得上

**避免转投：PM**

理由：背景不 match，PM 看重产品/用户经验，研究员转 PM 跨度大。

### Fullstack Eng 没拿到？

**可转投：Harness Eng**

理由：技术栈相近（TS+Python），都是 LLM 产品方向。

### Harness Eng 没拿到？

**可转投：Fullstack Eng / PM (如果有产品经验)**

### PM 没拿到？

**可转投：可能要换公司或岗位**

理由：PM 岗 fungibility 较弱，DeepSeek 内部 PM 转工程难。

## 七、长期 ROI 视角

```
PM:        快速上手 → 5 年后 → 产品负责人 / 创业 CEO
Harness:   系统能力 → 5 年后 → Tech Lead / 创业 CTO
Fullstack: 平台能力 → 5 年后 → Staff Eng / Platform Lead
Researcher: 算法 + paper → 5 年后 → 首席科学家 / 学术 / 创业 CTO
```

**关键问题**：你 5 年后想做什么？反推现在的选择。

- 想做 **公司**: Harness Eng / PM
- 想做 **infra 大将**: Fullstack Eng
- 想做 **科学家 / 学者 / AGI 推动者**: Researcher

## 八、特殊情况

### 我应届/在校生，应投哪个？

- **有 paper + 跑过 RL**：Researcher 首选
- **CS 背景 + 写过开源 Agent**：Harness Eng
- **会前端 + 熟 ops**：Fullstack Eng
- **MBA / 非技术背景**：PM 或考虑别的公司

### 我在大厂做应用 LLM，想转 frontier research？

- 投 Researcher，但要补 paper 阅读 + mini RL 项目
- 准备解释 "为什么从 applied 转 frontier"

### 我是 SRE / DevOps 想做 AI 相关？

- 投 Fullstack Eng，强调 infra + container 经验
- 用 LangChain / OpenAI SDK 等做几个 demo

## 九、回到本包：研究员准备的 Top-5 reminder

如果你最终决定投 **Researcher**，记住 5 件事：

```
1. paper 准备 > 算法手撕 > 系统设计
   (顺序 ≠ 难度顺序，是面试 ROI 顺序)

2. 一个跑通的 mini RL 项目 = 比一打 paper summary 重要

3. 反问环节问 R1/V3 的具体技术细节 → 显示真读过

4. 行为题里的"0→1 push"故事比技术答案更有 weighting

5. 不要假装会——不会的大方说 "我了解到这层, 推测应是..."
   研究员社区 reward honesty, punish bullshit
```
