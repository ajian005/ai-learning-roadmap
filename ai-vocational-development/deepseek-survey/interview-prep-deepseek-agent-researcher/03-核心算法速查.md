## 一、RLHF 全景与术语对齐

```
SFT (supervised) ───→ RM (reward model) ───→ RL alignment ───→ deployed model
   ↑                       ↑                         ↑
demonstration data    preference data            online sampling
```

| 术语 | 含义 |
| --- | --- |
| **π_θ** | policy（被训的 LLM） |
| **π_ref** | reference model（通常 = SFT 后的模型，冻结） |
| **r(x,y)** | reward function (RM output 或 rule-based) |
| **β** | KL 系数（DPO / PPO 控制 policy 偏离 reference 的强度） |
| **A(s,a)** | advantage |
| **V(s)** | value function (critic) |

## 二、PPO (Proximal Policy Optimization) ⭐

### 目标函数

```
L^CLIP(θ) = E_t [ min(
    r_t(θ) * A_t,
    clip(r_t(θ), 1-ε, 1+ε) * A_t
) ]

其中 r_t(θ) = π_θ(a_t|s_t) / π_old(a_t|s_t)
```

完整 RLHF PPO objective：

```
L_total = L^CLIP - c1 * L^VF + c2 * L^Entropy - β * D_KL(π_θ || π_ref)
```

- `L^VF`：value head 的 MSE loss
- `L^Entropy`：鼓励探索
- KL penalty：防止 policy 偏离 reference 太远（reward hacking）

### 伪代码（RLHF 场景）

```python
for epoch in range(N):
    # 1. Rollout
    prompts = sample_batch()
    responses = policy.generate(prompts)  # on-policy
    rewards = reward_model(prompts, responses)
    
    # 2. KL penalty
    logp_new = policy.log_prob(prompts, responses)
    logp_ref = ref_model.log_prob(prompts, responses)
    kl = logp_new - logp_ref
    adjusted_rewards = rewards - beta * kl
    
    # 3. Compute advantages (GAE)
    values = critic(prompts, responses)
    advantages = gae(adjusted_rewards, values)
    
    # 4. PPO update (multi-epoch on same batch)
    for _ in range(ppo_epochs):
        ratio = exp(logp_new - logp_old)
        surr1 = ratio * advantages
        surr2 = clip(ratio, 1-eps, 1+eps) * advantages
        policy_loss = -min(surr1, surr2).mean()
        value_loss = (values - returns).pow(2).mean()
        loss = policy_loss + 0.5 * value_loss - 0.01 * entropy
        loss.backward()
```

### 关键超参

| 超参 | 典型值 | 备注 |
| --- | --- | --- |
| `lr` | 1e-6 ~ 5e-6 | 比 SFT 小 10x |
| `clip_eps` | 0.2 | 防止 update 过大 |
| `kl_coef` β | 0.01 ~ 0.1 | 太小→reward hacking；太大→不学 |
| `ppo_epochs` | 2~4 | 在 same rollout 上跑几轮 |
| `gamma` | 0.99 / 1.0 | token-level usually 1.0 |
| `gae_lambda` | 0.95 | bias-variance trade-off |

### 失败模式

1. **Reward hacking**：policy 学会"刷分"——产生看似高 reward 但实际质量差的 response。Fix: 调大 KL、更鲁棒 RM、用 rule-based 替代
2. **Mode collapse**：所有 response 越来越像。Fix: 加 entropy bonus、用更高 temperature
3. **KL drift**：训练后期 KL 不受控暴涨。Fix: KL adaptive controller
4. **Value function 不稳定**：critic 跟不上 policy 变化。Fix: warmup critic 再放 policy

### 高频面试

| Q | A |
| --- | --- |
| 为什么用 ratio clip 而不是 trust region? | 计算便宜，效果接近 TRPO |
| 为什么 KL 既加在 reward 又加在 loss? | InstructGPT 默认在 reward 里减 KL（per-token），让 advantage 直接吸收；有些实现用 loss 里加，效果差不多 |
| GAE 的 λ 是什么? | bias-variance trade-off：λ=0 (TD)，λ=1 (Monte Carlo) |

## 三、DPO (Direct Preference Optimization) ⭐⭐⭐

### 推导思路（必背）

RLHF 的 KL-regularized objective 有**闭式解**：

```
π*(y|x) ∝ π_ref(y|x) * exp(r(x,y) / β)
```

反解出 reward：

```
r(x,y) = β * log[π*(y|x) / π_ref(y|x)] + β * log Z(x)
```

代入 Bradley-Terry preference 模型：

```
P(y_w > y_l | x) = σ(r(x, y_w) - r(x, y_l))
```

`log Z(x)` 自动消掉，得到 **DPO loss**：

```
L_DPO = -E_{(x, y_w, y_l)} [ log σ(
    β * log[π_θ(y_w|x) / π_ref(y_w|x)]
  - β * log[π_θ(y_l|x) / π_ref(y_l|x)]
) ]
```

> **核心 insight**：不需要显式训 reward model + 不需要 RL sampling，把整个 RLHF 退化成一个 classification loss。

### 伪代码

```python
for batch in preference_data:  # (x, y_w, y_l)
    logp_w_theta = policy.log_prob(x, y_w)
    logp_l_theta = policy.log_prob(x, y_l)
    logp_w_ref   = ref.log_prob(x, y_w)
    logp_l_ref   = ref.log_prob(x, y_l)
    
    log_ratio_w = logp_w_theta - logp_w_ref
    log_ratio_l = logp_l_theta - logp_l_ref
    
    loss = -F.logsigmoid(beta * (log_ratio_w - log_ratio_l)).mean()
    loss.backward()
```

### 关键超参

| 超参 | 典型值 |
| --- | --- |
| `β` | 0.1 ~ 0.5（越大 → KL 越紧） |
| `lr` | 5e-7 ~ 5e-6 |
| `epochs` | 1~3（容易 overfit） |

### 失败模式

1. **Likelihood displacement**：训完后 chosen 和 rejected 的 prob **都降低**——只是 chosen 降得少（IPO 主要解决这个）
2. **Distribution shift**：preference data 离 reference policy 太远，π_ref(y_w) 太低 → 梯度爆炸
3. **Reference model dependence**：换 reference 整个 dynamic 变
4. **No exploration**：纯 offline，无法发现 ref policy 外的新模式

### DPO 变体地图

| 变体 | 改动 | 解决 |
| --- | --- | --- |
| **IPO** | 用 squared loss 替 sigmoid | likelihood displacement |
| **SimPO** | 去掉 reference + 长度归一 | 显存 + 长度 bias |
| **KTO** | 不需 pair，single sample binary feedback | preference 数据稀缺 |
| **ORPO** | SFT loss + odds-ratio penalty | SFT + alignment 一步训完 |
| **CPO** | 加 SFT term 当 anchor | 避免 likelihood collapse |

### 高频面试

| Q | A |
| --- | --- |
| DPO 为什么不需要 reward model？ | 用 Bradley-Terry + KL-reg 闭式解把 reward 隐式吸收到 policy ratio |
| DPO 为什么不需要 sampling？ | offline——pair 是预先标好的 |
| β 调大调小区别？ | 大 β → policy 离 reference 更近，更保守；小 β → 学得快但更易 reward hacking |
| DPO vs PPO 谁好？ | style/tone DPO 够；math/code/agent 用 verifier-based RL (PPO/GRPO) 更好（因有 ground truth） |

## 四、GRPO (Group Relative Policy Optimization) ⭐⭐⭐

### 设计动机

PPO 需要：1) policy 2) ref 3) **critic** 4) reward → 4 个模型，对 LLM 巨大显存压力。GRPO 去掉 critic，用**组内归一化**当 baseline。

### 目标函数

```
对每个 prompt x，sample G 个 response: {y_1, ..., y_G}
计算每个 y_i 的 reward: r_i
组内归一化：A_i = (r_i - mean(r)) / std(r)   ← advantage

L_GRPO = E_{x, y_i} [ min(
    ρ_i * A_i,
    clip(ρ_i, 1-ε, 1+ε) * A_i
) ] - β * D_KL(π_θ || π_ref)

其中 ρ_i = π_θ(y_i|x) / π_old(y_i|x)
```

### 伪代码

```python
for prompt in batch:
    # Sample G responses
    responses = policy.sample(prompt, n=G)  # e.g., G=64
    rewards = [reward_fn(prompt, r) for r in responses]
    
    # Group-relative advantage
    mean_r = mean(rewards)
    std_r = std(rewards) + 1e-8
    advantages = [(r - mean_r) / std_r for r in rewards]
    
    # Per-token GRPO loss
    for resp, adv in zip(responses, advantages):
        logp_new = policy.log_prob(prompt, resp)
        logp_old = policy_old.log_prob(prompt, resp)
        logp_ref = ref.log_prob(prompt, resp)
        
        ratio = (logp_new - logp_old).exp()
        surr1 = ratio * adv
        surr2 = clip(ratio, 1-eps, 1+eps) * adv
        policy_loss = -min(surr1, surr2).sum()
        
        kl = compute_kl(logp_new, logp_ref)  # exact or estimator
        loss = policy_loss + beta * kl
        loss.backward()
```

### 关键 trade-off vs PPO

| 维度 | PPO | GRPO |
| --- | --- | --- |
| critic | 需要 | 不需要 |
| advantage | GAE | 组内归一化 |
| 显存 | 4 models | 3 models |
| sample 效率 | 高（每 sample 都用） | 略低（需 G 个 sample 才能算 advantage） |
| reward scale 适应 | 需手动归一 | 自动（除以 std） |
| 适合场景 | 通用 | reward 易计算 + sample 便宜（如 math/code） |

### 高频面试

| Q | A |
| --- | --- |
| 为什么 R1 用 GRPO 而不是 PPO？ | 省显存 + reward 是 rule-based 可大量并行 sample + 不需要复杂 critic warmup |
| GRPO 的 advantage 一定无 bias 吗？ | 不一定——group baseline 是 sample mean，有限 G 时是 estimator |
| 如果 G=1 GRPO 退化成什么？ | 退化成 advantage=0 → 等于 SFT (无信号) |
| GRPO 缺点？ | 1) 需 G 大才稳定（典型 G=8~64）；2) 无 step-level credit assignment——所有 token 共享同 advantage |
| 怎么改进 GRPO？ | 1) 加 PRM 做 step-level advantage 2) DAPO/Dr.GRPO 等改 clip / normalization |

## 五、RLAIF (RL from AI Feedback) + Constitutional AI

### 核心思想

把 RLHF 里的"人工偏好"换成"AI 偏好"——用一个 LLM 当 judge 来给 preference label。

### Constitutional AI（Anthropic）

两阶段：
1. **SL-CAI**：写一组"宪法"原则 → 让 model 自我 critique + revise → 用 revise 后数据 SFT
2. **RL-CAI**：用另一个 model 比较两个 response 哪个更符合宪法 → 生成 AI preference → 跑 RL（PPO）

### Pros / Cons

| | Pros | Cons |
| --- | --- | --- |
| RLAIF | 1) 标注便宜 100x 2) 可 scale 3) 一致性高 | 1) judge model 的 bias 会被强化 2) reward hacking 风险更大 3) "model collapse" 风险 |

### 高频面试

- "RLAIF 和 RLHF 哪个更好？" → 现在主流：**helpfulness 用 RLHF，harmlessness/format 用 RLAIF**（混用）
- "怎么避免 judge model 的 bias？" → multi-judge / multi-criteria / 加少量 human data 做校准

## 六、PRM (Process Reward Model) vs ORM (Outcome Reward Model)

### 区别

| | ORM | PRM |
| --- | --- | --- |
| 输入 | (prompt, full response) | (prompt, prefix up to step k) |
| 输出 | 1 个 reward | 每个 step 1 个 reward |
| 训练数据 | response + 0/1 标签 | response + per-step 0/1 标签 |
| 推理 | best-of-N + 比较 final reward | 每步 reward → 早停坏分支 / search 引导 |

### PRM 为什么对 reasoning 重要

- ORM 是 **sparse**——错一步整个 response 都 0 分
- PRM 是 **dense**——能告诉 model "你在第 3 步开始错了"
- 对 reasoning（多步推理）的 credit assignment 更准

### PRM 数据获取

1. **人工标注**（OpenAI PRM800K）——贵但金标
2. **MCTS rollout**（Math-Shepherd）——从 prefix 起 N 个 rollout，看多少能到对 → 估 step value
3. **二分搜索**（OmegaPRM）——更高效定位错误 step
4. **Implicit**（PRIME）——直接用 reward model 当 logp，不显式训 PRM

### 高频面试

- "PRM 训完怎么用？" → 1) inference-time best-of-N + PRM 重排 2) 训练时用 step reward 做更细的 advantage 3) MCTS 引导
- "PRM 容易过拟合吗？" → 是。step reward 信号弱 → 易过拟合到训练分布
- "PRM 和 RM 的实战权衡？" → 通用聊天用 ORM/RM；reasoning/math/code 强烈推荐 PRM

## 八、面试必背 5 公式（默写级）

### 1. PPO clipped objective
```
L^CLIP = E[ min( r(θ) * A, clip(r(θ), 1-ε, 1+ε) * A ) ]
```

### 2. DPO loss
```
L_DPO = -E[ log σ( β * (log π/π_ref(y_w) - log π/π_ref(y_l)) ) ]
```

### 3. GRPO advantage
```
A_i = (r_i - mean({r_1,...,r_G})) / std({r_1,...,r_G})
```

### 4. KL penalty in RLHF reward
```
r̃(x,y) = r_RM(x,y) - β * KL[π_θ || π_ref]
```

### 5. Bradley-Terry preference model
```
P(y_w > y_l | x) = exp(r(y_w)) / (exp(r(y_w)) + exp(r(y_l))) = σ(r(y_w) - r(y_l))
```
