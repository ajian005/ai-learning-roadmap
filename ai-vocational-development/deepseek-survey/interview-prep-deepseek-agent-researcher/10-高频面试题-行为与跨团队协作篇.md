# 10 · 高频面试题·行为与跨团队协作篇（研究员特化）

> JD 任职要求 5：**"善于跨团队沟通，能够与数据标注、工程团队高效配合"**——这一份覆盖研究员特化的行为题：和数据团队 collab、和工程团队 align、研究 0→1 故事、抗压 / 失败复盘。
>
> 答题统一使用 **STAR-C 框架**：Situation → Task → Action → Result → **Critical reflection**（研究员特别要 calibrate 自己的判断）。
>
> 8 道题分 4 组：
> - A 组：Research 0→1 故事（2 题）
> - B 组：跨团队协作（数据 / 工程）（3 题）
> - C 组：失败 / 抗压（2 题）
> - D 组：DeepSeek 文化匹配（1 题）

---

## 一、STAR-C 框架（研究员版）

| Part | 普通 STAR | STAR-C 研究员版 |
| --- | --- | --- |
| **S** | 背景 | 背景 + **当时的不确定性 / 未知** |
| **T** | 任务 | 任务 + **我提出的 hypothesis** |
| **A** | 行动 | 行动 + **设计的 ablation / 实验** |
| **R** | 结果 | 结果 + **数字 + 统计意义** |
| **C** | （无） | **Critical reflection**：confound / 我现在不同意自己之前的什么 / 学到什么 meta-lesson |

> 研究员 vs 工程师 STAR 的核心差异：**研究员故事必有 "我事后发现的局限 / 我会怎么改进设计"** —— 这个 C 是 senior researcher 的 signal。

## 二、A 组·Research 0→1 故事

### Q1·讲一个你独立 propose idea 并推动到 result 的故事

**JD 击中**：要求 1 "独立分析提 idea + 0→1 推动研究能力"。

**示范故事（请改成你自己的）**：

> **S** (Situation)：
> 去年在 X 实验室做 LLM RLHF 实验时，我注意到一个 anomaly——同样的 reward model，PPO 训练 6K step 后 reward 降而 helpfulness eval 升。这违反我们对 RLHF 的预期。当时没人深究，因为 reward 下降但 eval 升是 "好结果"。
>
> **T** (Task)：
> 我决定自己 investigate。我的初步 hypothesis 是：**RM 在训练后期 OOD shift**——policy 已经 generate 了 RM 训练集里没有的样本类型，RM 估值不准。这只是一个 hunch，导师没指派我做这个。
>
> **A** (Action)：
> 1. 收集了 12 个 ckpt 的 generation sample（每个 200 条）
> 2. 计算每个 ckpt generation 在 RM training data 上的 nearest neighbor distance（embedding cosine）
> 3. 同时让 GPT-4 评 holdout helpfulness
> 4. 设计了一个 mini ablation：每 1K step refresh RM training data with current policy generation
>
> **R** (Result)：
> 1. 证实了 OOD shift——distance 在 step 4K+ 明显↑
> 2. RM refresh 让 reward-eval correlation 从 0.4 → 0.78
> 3. 写成 internal tech report，被纳入下次 RLHF pipeline
>
> **C** (Critical reflection)：
> 复盘有 3 个 caveat 我事后才意识到：
> 1. 我的 OOD 度量用 cosine distance，可能 underestimate semantic shift（如果用 BLEU/Rouge 会更全面）
> 2. RM refresh frequency 1K step 是 heuristic，没系统 ablation
> 3. 后来看 Anthropic 一篇 paper 提到类似现象，他们叫 "RM drift"，比我的 framing 更准
>
> 学到 meta-lesson：**anomaly 是 free signal——别人忽视的 small irregularity 经常是 publishable insight 的入口**。

**为什么这个故事强**：
- 有具体数字（0.4 → 0.78）
- 有 anomaly-driven curiosity
- 有 critical 反思（3 个 caveat）
- 学到 meta-lesson 不只是 specific finding
- "Anomaly 是 free signal" —— researcher 思维

### Q2·讲一个你从论文复刻到拓展的故事

**示范**：

> **S**：2024-Q4 我读了 DeepSeek-R1 paper，特别想验证 GRPO 在 small model 上是否 work。Paper 主要 demo 在 32B+，没明确 1B 以下数据。
>
> **T**：我设了 goal：在 Qwen-0.5B 上用 GRPO 训 GSM8K，看是否能复出 paper 提到的 self-correction 模式。
>
> **A**：
> 1. 用 TRL 实现 GRPO baseline，跑 baseline = 35% acc on GSM8K
> 2. Group size sweep: G ∈ {4, 8, 16, 32}
> 3. KL coef sweep: β ∈ {0.01, 0.04, 0.1}
> 4. 加 warm-up SFT 5K step (5K math 题)
> 5. 训 5K RL step
>
> **R**：
> - acc 从 35% → 60%
> - 找到 G=8 是 sweet spot（小 model 不需太大 G）
> - 观察到 step 3K 后 "let me check..." 开始 emerge——比 R1 paper 的 6K step early，可能小模型 reasoning 简单更早涌现
> - 写成 blog 发 Twitter，得到 ~200 star repo
>
> **C**：
> 1. 我的 reward 是 binary correct——没用 partial credit，前期 advantage variance 太小 train 不动（debug 1 周才发现）
> 2. 没控变量：warm-up SFT 5K step 同时改变了起点 + 数据 distribution，没拆开
> 3. 0.5B 上观察到的"emergence" 可能只是 prompt format 学会，不是 reasoning 涌现——我后来不那么 confident 这个 claim

**signal 体现**：
- 复刻 + 拓展（不只是复出）
- 实验设计严谨（sweep）
- 自我批判（3 个 caveat）
- Action：写 blog + open source

## 三、B 组·跨团队协作

### Q3·讲一个你和"数据标注团队"协作的故事

**JD 击中**：JD 第 2 条 "**与数据标注团队紧密协作，设计标注方案与质量标准**"。

**示范**：

> **S**：去年我做 instruction following 提升，发现 baseline 在 multi-step instruction (e.g., "summarize then translate") 上明显 fail。我推断是 training data 缺这种 compound instruction，需要 1K~5K 高质量标注 data 补。
>
> **T**：和数据标注团队 PM 合作，设计 label spec + 启动 1K 条标注 + 在 2 周内 close loop。
>
> **A**：
> 1. **我先自己标 50 条**——发现 boundary case：什么算"step 2 用 step 1 输出"？写第一版 rubric (1 页 PDF)
> 2. **和 senior labeler 5 人共标 50 条 calibration set**——算 Cohen's kappa = 0.62（中等 agreement）
> 3. **Disagreement review**：识别 2 个模糊 case → 更新 rubric v2 + 3 个新 example
> 4. **Re-calibration**：kappa = 0.84
> 5. **Pilot run 200 条**：spot check 20% 自己抽
> 6. **大规模 1000 条**：3 周完成
> 7. **加入 SFT data 重训 → eval IFEval +6pp**
>
> **R**：
> - IFEval multi-step subset 从 67% → 78%（+11pp）
> - 数据团队也 reuse 了我的 rubric template 给其他 task
> - 团队开始有 "researcher 必须自己标 50 条" 文化
>
> **C**：
> 1. 我一开始 underestimate rubric iteration 时间——以为 1 个 sprint 搞定，实际 calibration 来回 2 次
> 2. 200 条 pilot 时漏了多语言 case，后来 1000 条标到一半才发现 → patch fix
> 3. 没和 labeler 1-on-1 face-time，全 async——后来意识到一个 30 分钟 calibration 会议比 10 轮邮件高效

**signal 体现**：
- 主动 dogfood 标 50 条（researcher 不嫌 manual）
- Rubric calibration 严谨（kappa 数字）
- 推动 process improvement（"researcher 必须标 50 条"文化）
- Critical 自省（3 个 lesson）

### Q4·讲一个你和"工程团队"协作的故事

**示范**：

> **S**：去年 Q3 我提了个 RL training infra 改进——用 group sample batch 方式，让 RM forward 共享 KV cache 节省 40% inference cost。但这需要工程团队改 inference scheduler。
>
> **T**：说服工程团队这个改进 priority 够高，并 collaborate 1 个 sprint 落地。
>
> **A**：
> 1. **先做 proof-of-concept**——用 toy 1B model + 简化 scheduler，自己 hack 一个版本，showcase 38% latency 降
> 2. **写 1 页 design doc**——把 trade-off / 边界条件 / 测试方案写清
> 3. **和工程 tech lead 1-on-1**——理解他们当前 backlog / 担心 (我的改动会不会 break 其他用户)
> 4. **拆 PR**——主 PR 由我和工程同学结对；小 PR (eval / monitor) 我独立 ship
> 5. **A/B test**——上线后 24h 验证；监控指标我提供
>
> **R**：
> - GRPO training time -32%
> - 工程团队后来 generalize 到 PPO / DPO pipeline 都受益
> - 我学到了 "scheduler" 这部分代码细节
>
> **C**：
> 1. 第一次没写 design doc 直接讲，工程不愿 prioritize——后来才意识到 written communication 比 verbal pitch 强
> 2. POC 用 1B model，后来 13B 才发现 scheduler 有 subtle race condition——应该 medium-scale 也 test
> 3. 应该早邀工程同学 review 我的 POC（他们 5 分钟能指出我没想到的边界条件）

**signal 体现**：
- 不是"研究员只管算法工程负责实现"心态——主动 hack POC
- Design doc → 工程合作 hygiene
- 复盘自我 calibration

### Q5·讲一个你和"其他研究员"意见不一致的故事

**示范**：

> **S**：组里讨论新一轮 RLHF preference data 怎么收。Senior researcher A 主张用 GPT-4 大规模合成 100K 条；我 disagree——觉得 quality 比 quantity 重要，应该人标 10K。
>
> **T**：在团队 review 时 surface 我的不同意见，但又不能 burn 关系。
>
> **A**：
> 1. **先 ask question 而非 push back**："你之前用合成 data 训过的 best case 是什么？踩过的坑是什么？"——理解 A 的 perspective
> 2. **承认 A 的 point**：合成 quick + cheap，确实优势
> 3. **提自己 evidence**：引用 Anthropic 一篇 paper 指出合成 preference data 在 hard task 上 hack 严重；提供我自己跑过的 sycophancy data 实验
> 4. **提 third option**：先 1K 人标 (calibration set) + 30K GPT-4 合成 (scale)，然后人 spot check 100 条调 mix——hybrid approach
> 5. **结果**：A 同意 hybrid；后来 ablation 证明 hybrid > 单一方案
>
> **R**：
> - 团队采用 hybrid recipe
> - 我和 A 后来在另一项目深度合作
>
> **C**：
> 1. 我一开始 ask question 是 strategic（buy time），但发现真心 listen 让我自己也学到 A 的 valid concern
> 2. 如果直接 pushback "我不同意你"，A 可能 defensive；hybrid 让两人都有 ownership
> 3. 学 meta：disagreement 不是 win/lose，是 information aggregation

**signal**：
- 不 ego 战
- 用 data / 引用 evidence
- Third option mindset
- 协作 > 正确

## 四、C 组·失败 / 抗压

### Q6·讲一个你 research 上"失败"的故事

**示范**：

> **S**：去年我花 6 周 implement 一个新 RL alignment loss（类似 KTO 但 weight scheme 不同），早期 toy task 看似 work——baseline 加 2pp。
>
> **T**：scale 到 7B model，期望复出 toy 优势。
>
> **A**：
> 1. 7B 训了 2 周（compute 不少）
> 2. 结果 baseline 没显著差异（在 noise 内）
> 3. 多次 hyperparameter sweep——仍无明显提升
> 4. 重审 toy task ablation——发现 toy task 一个 confound：我用了 deterministic env，scale up 后真实 env stochastic 多了，weight scheme 优势消失
>
> **R**：
> - 没出 paper / 没产品落地
> - 写了一个 internal "negative result" doc，详细解释 why fail
> - 学到一个新 insight：**deterministic vs stochastic env 对 RL method evaluation 是 critical confound**
>
> **C**：
> 1. 早期 toy task 太"toy"——没 stochasticity，提前看不出
> 2. 我 sunk cost——3 周后该 cut，但情绪上不甘心又跑 3 周
> 3. 没及早和 senior 讨论，自己 silo
> 4. 但 negative result doc 后来被另一个 project 引用，提醒他们 test env 选择——所以 "失败" 也 contribute
>
> **Meta-lesson**：**Cut your losses early。Sunk cost is the most expensive RL trap for researchers themselves。**

**signal**：
- 真承认失败（不假装是"学习经历"）
- 拆 root cause（confound）
- 仍 contribute（negative result doc）
- Meta-lesson

### Q7·讲一个你在"deadline 紧张 + 实验 fail"双重压力下的应对

**示范**：

> **S**：去年 NeurIPS deadline 前 10 天，主实验 result 比预期低 30%——我没法 ship。
>
> **T**：决定 submit / 不 submit / 改 scope。
>
> **A**：
> 1. **24h 内 root cause**——和 1 个 senior 通话 30 分钟，识别是 data preprocessing bug（少了 augmentation pass）
> 2. **修 bug 重跑** 主实验——4 天跑完
> 3. **同步 ablation 简化**——把原计划 12 个 cell 砍到 6 个 critical
> 4. **改 paper framing**——从 "X 比 baseline 好 5pp" 改成 "X 在 specific regime 下与 baseline trade-off 不同"——更 honest，less hype
> 5. **submit**——被 reject（reviewer 觉得 contribution 弱）
>
> **R**：
> - NeurIPS reject
> - 但 paper 投了 ICLR + revision，accept as workshop
> - 学到怎么在压力下 cut scope + reframe
>
> **C**：
> 1. 我应该早 4 周和 senior sanity check——不应等 deadline 才 root cause
> 2. 改 framing 是对的，但应该一开始就避 hype
> 3. NeurIPS reject 当时很 painful，但 reviewer 意见后来在 ICLR 改稿很有帮助——peer review 是 free feedback

**signal**：
- 在压力下 retain calm + execute
- 不 lie（reject 就说 reject）
- 失败转化 (workshop)
- 复盘 process 改进

## 五、D 组·DeepSeek 文化匹配

### Q8·"为什么选 DeepSeek？你怎么看 DeepSeek 的 research culture？"

**金句模板**：

> "三个原因：
>
> **1. Open weight + paper-first 文化**：DeepSeek 是中国少数 frontier 模型公司里同时开源 + 写 detailed technical report 的——R1 / V3 paper 都极详尽。我读了 R1 paper 第 5 节 GRPO 部分，对 'aha moment' figure 印象深——这种 honest about 'we tried SFT first then realized pure RL works' 的写法，是顶级 lab 风格。这种环境 incubate research integrity。
>
> **2. 无 KPI / 无赛马 / 长期主义**：公开报道反复提。意味着我可以做 risky bet——比如花 3 个月探一个 50% 概率 fail 的方向。短期 KPI 文化下这种 bet 没法做。
>
> **3. Agent capability 是 frontier**：我相信 reasoning + agent + RL 是 next 2 年最 active 方向。DeepSeek 在 R1 已经 demonstrate world-class——这是少数能在这个方向 contribute 而不只 follow 的中国公司。
>
> Caveat: 我也意识到 DeepSeek bar 很高（接近 Anthropic / OpenAI），同时强度也高（不是 9-5 工作）。我对 long hour + 高强度 ok，前提是我能学 + 能做出真正的 contribution——这个 deal 我接受。"

**signal**：
- 引用具体 paper / figure（证明真读过）
- 提"长期主义"和"无 KPI"（show 你做了 culture 调研）
- 不光 positive，也提 trade-off（"strong but not naive"）
- "I accept the deal" 框架——professional buy-in

## 六、行为题"反诘陷阱"5 个

> **面试官追问最高频的 5 个陷阱，预先准备**：

1. **"如果重做这件事，最想改什么？"**——必须有 calibrated regret
2. **"你最大的 weakness 是什么？"**——真实但 actionable improving 的
3. **"为什么离开上家？"**——never bad-mouth；growth opportunity framing
4. **"如果 manager 不同意你 idea 怎么办？"**——示范 pushback + ultimately respect decision
5. **"你 5 年规划？"**——research excellence + specific area depth + 不必 obsess title

## 八、最后忠告

> **行为题不是测"你是好人吗"——测 "你 think like a researcher 吗"**。
>
> Researcher 思维的核心 markers：
> - **calibrated** (我 confidence 是 70% 不是 100%)
> - **honest about caveat** (我事后看 confound)
> - **critically self-aware** (我会怎么改进)
> - **collaborative** (用 data + 第三 option)
> - **anomaly-driven** (irregularity 是 free signal)

> 把这 5 个 marker 植入每个 STAR-C 故事，研究员面试官会自然 cluster 你为 "peer"。
