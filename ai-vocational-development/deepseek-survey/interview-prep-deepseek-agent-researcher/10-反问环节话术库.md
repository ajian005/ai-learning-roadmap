# 10｜反问环节话术库（研究员向）

> 反问 = 你的"软实力筛选器"。研究员面试的反问，需要展示：**你已经在思考 DeepSeek 内部的 research dynamics + 你想要的环境是什么样**。
>
> 按"面试官类型"分类——不同人问不同问题。

---

## 一、反问的"金科玉律"

1. **每个面试官至少 2~3 个问题**——少于 1 个 = 不感兴趣
2. **避免可以 Google 的问题**（"DeepSeek 多少人？" → bad）
3. **避免敏感问题**（薪资 / 加班 / 裁员 / 内卷 → 留给 HR/Offer 谈判）
4. **问题要表现"你已经做了功课"**（"我注意到 R1 用 rule-based reward, ... 是不是有计划 extend?"）
5. **问题暴露你想要的环境**——研究员特别看这个："你想要 fast iteration 还是 deep think"
6. **听完答案后追问 1 次** —— 显示真在听，不是按 list 走

---

## 二、按"面试官身份"分类

### A. 对 HR / Recruiter

#### 面试节奏 + Offer 流程
- "这个岗位整个面试流程大概几轮？每轮主要 focus 什么？"
- "从 final round 到 offer 一般多久？"
- "实习生 / 应届 / 社招的录用比例大概？"

#### 团队 + 文化
- "Agent 算法研究 team 现在有多少人？背景大致是？"
- "新人 onboarding 一般多久？有 buddy / mentor 制吗？"
- "Team 是 office-only 还是 hybrid？"

#### 评价机制
- "研究员的 promotion 主要看什么？paper / 模型 perf / team contribution 的权重？"
- "DeepSeek 是否鼓励发 paper / 参加学术会议？"

#### Compensation（晚一点问）
- "Compensation package 大致由几部分构成？有 sign-on 吗？股权 vest 节奏？"
- 这一类一般 offer round 才聊，前期面试可以浅问

---

### B. 对 同级 Engineer / Researcher（一面/二面）

#### 技术深度
- "你最近在 own 什么 sub-direction？最难的部分是什么？"
- "Team 现在最 active 的 research thread 是哪几个？有没有 weekly reading group？"
- "DeepSeek-R1 之后, 你认为下一步最 promising 的方向是？"

#### Day-to-day life
- "你一天大致怎么 spend 时间？coding / 跑实验 / 读 paper / 开会 各占多少？"
- "实验 turnaround 周期一般多久？跑一个 ablation 多久能出结果？"
- "Infra 怎样？训练 cluster 资源紧不紧？需要排队吗？"

#### 工作风格
- "Code review / paper review 文化怎么样？"
- "失败的实验你们怎么处理？有 'negative results doc' 吗？"
- "你和 senior 研究员的 1on1 频率？"

#### Tooling
- "DeepSeek 内部有没有像 Weights & Biases 的实验追踪系统？"
- "Eval pipeline 怎么 organize？有 standardized harness 吗？"
- "Training framework 是自研的吗？基于 Megatron / DeepSpeed / verl？"

---

### C. 对 Senior Researcher / Team Lead (二面/三面)

#### Research strategy
- "Team 现在 vision 是什么？6 个月 / 1 年 / 3 年的 north star metric 是？"
- "你认为 LLM Agent 这个领域接下来 12 个月最 critical 的 challenge 是什么？"
- "DeepSeek 在 capability 之外还想 differentiate 什么——alignment / safety / efficiency？"

#### Hiring philosophy
- "你 hire 一个 researcher 最看重什么品质？最不能容忍什么？"
- "在 team 里, 你期待新人 6 个月内有什么 deliverable？"
- "你怎么判断一个 research direction 'work or not'？什么时候 kill?"

#### Their own work
- "你自己最近最 exciting 的工作是什么？"
- "你之前最 proud 的 research idea, 是怎么想到的？"

#### Cross-team dynamics
- "Algorithm team 和 product / engineering team 怎么协作？谁主导 model release？"
- "你们和 Harness team / Fullstack team 关系如何？eval 平台是谁 own？"

---

### D. 对 Director / Lab Head (终面)

#### 公司战略
- "DeepSeek 的中长期战略 priority 是？（基础模型 / 应用 / 对外开源）"
- "在中美 LLM 竞争格局中, 你觉得 DeepSeek 的 unique position 是什么？"
- "对于研究员的 hiring growth, 接下来 12 个月预期？"

#### 文化 + 价值观
- "你最 proud DeepSeek 文化里哪一点？最想改的是哪一点？"
- "梁文锋 / 公司层面给 research team 的 autonomy 实际有多大？"

#### 资源 + 投入
- "公司给 research 团队的算力 / 数据 / 人力投入怎么决策？是 top-down 还是 bottom-up？"
- "如果 researcher 提出'我要试一个看起来 risky 但 promising 的方向', team 怎么评估？"

#### 长期愿景
- "DeepSeek 想成为 OpenAI 还是 Anthropic 还是 ...？怎么看？"
- "AGI / superintelligence 的 timeline 你的判断？"
- （如果敢问）"对 open source vs closed source 的 stance？"

## 三、研究员特化的"杀手锏"反问

这些问题展示你"已经在为加入做准备"：

### 1. 关于 R1 / V3 的具体追问

> **"R1 paper 里 rule-based reward 在 math/code 上 work 得很好, 但在 helpfulness / open-domain 上未必。Team 是否在探索 RLAIF + rule-based hybrid？或者 process reward 在 open domain 的应用？"**

→ 展示你**真读过 paper + 想过 limitation + 准备贡献**

### 2. 关于 agent training 的具体追问

> **"R1 主要训 math reasoning, 但 agent task (如 SWE-Bench) 是 long-horizon + sparse reward。Team 有没有探索 step-level credit assignment 或 PRM-guided RL 在 agent training 上的应用？我个人对 Agent-RLVR 那条思路很感兴趣。"**

→ 展示你**关注 cutting edge + 有自己 taste**

### 3. 关于 infra 的具体追问

> **"你们 RL training 用什么 framework？我看过 verl, OpenRLHF, TRL 等开源 stack, 也踩过坑——比如 critic warmup / KL controller / reward scale。DeepSeek 内部有没有自研的 RL trainer？"**

→ 展示你**有实战经验 + 关心 production reality**

### 4. 关于评测 / iteration 的追问

> **"Team 怎么平衡 'capability benchmark' (MMLU / MATH) 和 'open-ended quality' (Arena / human judge)？我注意到 R1 在 reasoning 上很强但 IFEval 提升有限——这是 trade-off 选择还是 work in progress？"**

→ 展示你**理解 alignment tax + 关心整体而非 single metric**

### 5. 关于"我能贡献什么"

> **"如果我 join, 我之前的 background 是 [X]。短期看, 我可以 own [一个具体可贡献的方向, 如 'agent RL 的 reward design 实验']。Team 现在最缺这种 capacity 吗？"**

→ 展示你**思考 fit + 不是 entitled**

## 四、不要问的问题（红线）

| 问题 | 为什么不问 |
| --- | --- |
| "加班严重吗?" | 显得不投入 / 怕辛苦 |
| "DeepSeek 比 Anthropic 怎么样?" | 显得在 shopping |
| "你们 996 吗?" | 同上 |
| "薪资 package 大概多少?" | 留给 offer round |
| "我能 work from home 吗?" | 等 offer 谈 |
| "你们裁员吗?" | 不 appropriate |
| 任何能 Google 的（如"DeepSeek 几个 office?"） | 显得没准备 |
| 太大的 ("AGI 什么时候来?") | 显得 cliché |
| 给面试官出难题（"我考你一道 GRPO 公式"） | 大忌 |

## 五、3 个反问模板（可直接用）

### 模板 A：技术面后的标准 3 问

```
1. "你最近在 own 的项目最 exciting 的是哪部分? 最难的是哪部分?"

2. "你们 [team] 内部对 [R1 next step / agent RL / 某具体方向] 怎么看?"

3. "如果我 join, 6 个月内你期待我达成什么? 12 个月呢?"
```

### 模板 B：和 senior researcher 的 4 问

```
1. "你 hire 一个新人, 最看重的 3 个 quality 是什么?"

2. "Team 里你最 proud 的研究文化 / 习惯是什么?"

3. "你最近读到一篇 paper 最 exciting 的是哪一篇?"

4. "对于 [一个具体 research thread, 如 'long-horizon agent'], team 现在的 thesis 是?"
```

### 模板 C：和 Director 的 5 问

```
1. "DeepSeek 中长期 strategy priority?"

2. "Research team 在公司里的 autonomy 实际有多大?"

3. "你怎么看接下来 12 个月 LLM Agent 领域的 critical challenge?"

4. "DeepSeek 的 culture 你最 proud / 最想改的?"

5. "如果我 6 个月做出 X 成果, 1 年呢? 你的 expectation 是?"
```

## 六、反问的"听 + 追问"技巧

```
面试官回答你的问题时:

Step 1: 真听 (不是脑里准备下个问题)
Step 2: take notes (笔在桌上 / 记忆)
Step 3: 找一个有意思的点 follow up

"你刚提到 X 这一点很有意思——能展开讲讲 [具体 follow up] 吗?"

→ 显示你真的关心, 不是走 checklist

→ 也给面试官一个"我很 engaged"信号
```

---

## 七、面试结尾的"closing"

```
"非常感谢你今天的时间, 我从你的分享里学到很多 [具体某点].

我对这个机会非常 excited——尤其是 [你最 hooked 的方面].

接下来的 process 我可以做什么准备?

我有几个问题想后续邮件确认 [如果有], 可以发邮件给你或 HR 吗?

非常期待下一步."
```

→ 礼貌 + 具体 + 主动

---

## 八、研究员反问的"反向用途"

反问环节其实是你的**信息收集环节**——决定要不要去：

| 你问的 | 你听到的 | 你应该判断的 |
| --- | --- | --- |
| "team 多少人？" | "5 个 senior + 8 个 junior" | scale 合不合预期 |
| "实验 turnaround?" | "通常 1~2 天" vs "1~2 周" | infra 好不好 |
| "失败实验怎么处理?" | "都写 doc" vs "不太追究" | 是不是 research-driven |
| "1on1 频率?" | "每周" vs "每月" | mentor 投入度 |
| "promotion 看什么?" | "paper + model perf" vs "OKR" | 是不是 research culture |
| "和 product team 谁主导?" | "research 主导" vs "product 主导" | autonomy |

**反问时认真听——这一环节比技术面更 inform 你是否要去**。
