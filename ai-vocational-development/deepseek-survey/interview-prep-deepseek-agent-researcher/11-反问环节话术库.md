# 11 · 反问环节话术库（研究员向）

> 研究员面试反问 = **"我和 senior researcher 在喝咖啡聊真问题"**。问题要：
> - 体现你看了 paper / 了解 lab
> - 体现你想清楚自己来这干什么
> - 避免 HR-able 信息（薪资 / 假期 / OKR）—— 那些 HR round 问
>
> 这一份按 5 类面试官（**peer researcher / senior researcher / 数据团队 / 工程团队 / leader / HR**）分组，每类备 5~8 个高质量问题。
>
> **金句原则**：每个问题尾带一句"我这么问是因为我之前 X 经验观察到 Y"，瞬间从 "面试问问题"变成 "two researchers talking shop"。

---

## 一、Peer Researcher 面试官（最常见，1~2 轮）

### Research 内容深度

1. **"你们 team 当前最重要的 research thread 是什么？最近 6 个月最 surprised 你的 finding 是什么？"**
   > *我这么问是因为我读 R1 paper 时被 'aha moment' 那个 figure 震到——想知道你们内部还有什么没公开的 finding。*

2. **"R1 paper 里的 GRPO 在你们日常 RL pipeline 用得多吗？相比 PPO / RLOO 你们的实际 trade-off 观察是什么？"**
   > *我自己在 0.5B 上跑过 mini GRPO，发现 group size 是关键 hyper——想了解 scale 到 70B+ 你们调过什么。*

3. **"你们怎么处理 reward hacking？有特别的 adversarial probe set 吗？"**
   > *我之前实验观察到 step 4500 后 RM ↑ 但 helpfulness eval ↓，怀疑是 OOD shift。*

4. **"你们对 Reasoning model 的 over-thinking 问题怎么看？budget-aware reasoning 是 active direction 吗？"**
   > *用 R1 时感觉简单题 thinking 太长 cost 浪费，好奇这是 research 优先级还是 product 优先级。*

5. **"PRM800K 这类 process reward data 你们自己有 build 吗？还是用 rule-based reward？"**
   > *PRM 标注成本高，我看 MathShepherd 的 Monte Carlo 自动标蛮 elegant。*

### 工作方式

6. **"研究员的 typical week 是什么样？读 paper / 跑实验 / 写 doc / 开会的大概比例？"**
   > *我习惯 35/45/15/5，但 DeepSeek 可能不一样。*

7. **"你们 idea 是怎么从 0 → push 到 ckpt 的？是 PM 给方向，还是 researcher 自下而上提？"**
   > *我观察 Anthropic 偏 bottom-up，OpenAI 偏 top-down，想了解 DeepSeek 是哪种。*

8. **"compute 申请是怎么 process 的？小实验 vs 7B 训练 vs 70B 训练 各自的流程？"**
   > *这直接影响我能 take risk 的 frequency。*

## 二、Senior Researcher / 团队 lead 面试官

### Vision / Strategy

1. **"DeepSeek 的 Agent capability research roadmap 在 next 12 月 priority 是什么？reasoning / tool use / coding 哪个最 active？"**

2. **"在 reasoning model 越来越普及后，你们认为下一个 capability frontier 是什么？multi-modal? embodied? long horizon agent?"**

3. **"DeepSeek 怎么看 open vs closed weight 的长期 trade-off？R1 完全开放对 future research 是 enable 还是 burden？"**
   > *Llama / DeepSeek 共同 push 开源，但 OpenAI / Anthropic 越来越 closed——你们 long term thinking 是什么。*

4. **"R1 paper 里 GRPO 是 DeepSeekMath 已有，R1 把它 push 到 reasoning。下一个'已有方法 push 到新场景' 的候选 idea 你们看好哪个？"**

### Hiring / Team

5. **"你们对'好 researcher'的 mental model 是什么？哪 2~3 个 trait 你们最看重？"**
   > *我可以 calibrate 自己的 development priority。*

6. **"我加入后，你期望我前 90 天 / 180 天分别 deliver 什么？"**
   > *经典但有效——表明我想 ramp up 快。*

7. **"团队里 PhD vs 工业经验 vs 自学 GitHub-known 的比例如何？我属于 X 背景，能融入吗？"**
   > *具体讲你自己背景，自然 segue。*

8. **"DeepSeek 不发 KPI 和 bonus 的文化，对 researcher 的 motivation 是怎么 sustain 的？"**
   > *公开 statement 听过，想了解 internal mechanism。*

## 三、数据团队面试官（可能 cross-functional 出现）

1. **"你们的 preference / SFT 标注数据 typical 量级和 throughput 是多少？"**
   > *我之前协作过 1K~10K rubric task，想了解 DeepSeek scale。*

2. **"研究员和数据标注的 collab cycle 是什么？rubric design 是 researcher draft 还是 collab？"**

3. **"calibration 流程怎么做？inter-annotator agreement 一般目标是多少？"**
   > *我自己实验观察 kappa 0.85+ 才放心 large scale。*

4. **"数据 quality issue 怎么 surface？有 spot check / audit pipeline 吗？"**

5. **"AI-assisted labeling (Constitutional AI / RLAIF) 你们 adoption 程度如何？人 vs AI mix 比例？"**
   > *Anthropic 走 RLAIF 重，OpenAI 仍偏人——想了解 DeepSeek stance。*

---

## 四、工程团队面试官（infra / RL framework）

1. **"你们 RL training framework 是基于 TRL / OpenRLHF / verl / 自研？"**

2. **"研究员需要改 framework 时的 process 是什么？需要 PR review 还是 fork?"**
   > *我有过 hack inference scheduler 的经历，想了解 DeepSeek 工程边界。*

3. **"7B / 70B 训一个 epoch 的 typical wall-clock？bottleneck 在 GPU / network / data？"**

4. **"RL rollout 是 in-process 还是 distributed inference server？长 horizon agent task 怎么 manage rollout latency？"**

5. **"研究员实验 ckpt 的 storage / versioning 是怎么做的？MLflow / W&B / 自研？"**
   > *我习惯用 W&B + Hugging Face hub，能 transfer 经验吗。*

## 五、Leader / VP / 创始团队（hidden round）

1. **"DeepSeek 的长期 vision 5 年内你认为什么是 best case / worst case？"**

2. **"你怎么看 Anthropic / OpenAI / DeepMind 三家的差异？DeepSeek positioning 在这个 landscape 里是什么？"**

3. **"R1 之后，DeepSeek 在 Agent / Coding Agent 方向，哪个 product 是接下来重点？Harness 团队的 strategic 重要性如何？"**

4. **"如果让你 hire 1 个 staff researcher，你会希望他/她 first principle 强还是 execution 强？"**

5. **"你自己作为 leader，希望团队文化 5 年后变什么样？什么是你不希望改变的？"**

## 六、HR 面试官（薪资 / package / 行政）

> 这部分**只在 HR round 问**，技术面别问。

1. 薪资结构 + 是否签字奖金 + 股权（DeepSeek 比较少 stock 风格，可问 long-term）
2. relocate 支持（外地 → 北京 / 杭州）
3. 入职时间 + onboarding 流程
4. 远程 / hybrid policy
5. 假期 + 法定福利
6. Visa / 户口（如适用）

---

## 七、"我不能问"的 anti-question 5 个

> ❌ 千万别问，会扣分：

1. ❌ **"你们每天工作几小时？996 吗？"** — implies you're afraid of work
2. ❌ **"OKR / KPI 怎么定？年底怎么打分？"** — DeepSeek 明确无 KPI，问 = 你没做研究
3. ❌ **"R1 是怎么训出来的？"** — 太大问题，看起来你没读 paper
4. ❌ **"你们用 PyTorch 还是 JAX？"** — 太基础
5. ❌ **"我能 work from home 吗？"** — 上来就要弹性，signal weak commitment

## 八、Top-3 反问"杀手锏"（任何 round 都能用）

如果只能问 3 个，按下面 priority：

### 1·"你们当前 research 最让你 personally inspired 的 ongoing thread 是什么？"

> **为什么强**：迫使面试官分享 internal excitement；他/她的答案给你 valuable signal 团队真在乎什么；同时表明你想加入"team's mission"不只是"a job"。

### 2·"我加入后，你最希望我前 90 天 ship 出什么 specific contribution？"

> **为什么强**：迫使面试官想清你 fit 哪个 gap；如果他答不上 / 模糊，说明 hire intent 不强；如果具体，你立刻知道 ramp-up 路线。

### 3·"R1 之后，你认为下一个最 critical 的 alignment / capability 突破会是什么？为什么不是 X？"（X 替换为 hot topic 如 multimodal）

> **为什么强**：peer-level research conversation；testing 面试官 thinking depth 同时也 demonstrate 你的 thinking depth。

## 九、反问环节"3-2-1 节奏"

```
3 个核心问题（research / team / vision）—— 必问
2 个 backup 问题（compute / hiring）—— 时间够再问
1 个 personal 收尾（"对你个人，DeepSeek 工作 1 年里最 unexpected 的 surprise 是什么？"）—— show human curiosity
```

---

## 十、最后忠告

> **反问环节是 50% 的面试体验——好反问能挽回前面平庸答题，烂反问能毁掉前面精彩答题。**

> 准备 10~15 个 ready-to-fire 问题；按面试官 role 抽 3~5 个；现场 listen 再选最 relevant 的；**每个问题尾带一句"我这么问是因为…"** 让它成为对话不是面试。

> **强反问的本质：你不是 candidate 求 offer——你是 senior researcher 在 evaluate 这家公司值不值得加入。这个 mindset shift 是 senior signal。**
