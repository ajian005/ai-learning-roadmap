# 12 · 面试前-中-后全流程行动手册（研究员版）

> 把"面试前 / 中 / 后"3 阶段合在 1 份的实战手册。研究员准备**比工程岗周期长**（paper 阅读 + RL 数学不能临时抱佛脚），推荐 T-14 ~ T+0。
>
> 3 部分：
> - **PART A·T-14 ~ T-0**：14 天 daily 行动 + 项目里程碑
> - **PART B·面试当天**：开场 60 秒模板 + 各类型题临场策略 + 卡壳话术
> - **PART C·T+0 ~ T+14**：复盘 framework + 感谢邮件 + V0.2 commit + 后续

## PART B·面试当天·临场策略

### 一、开场 60 秒模板（必背）

> *面试官问 "请简单自我介绍" 时用。*

```
"您好，我是 [姓名]。

我目前在 [公司/学校] 做 [research direction]——
主要做过 3 件 relevant 工作：
  1. [项目 A 1 句话] 比如：在 X 实验室做 RLHF 实验，pioneer 了 RM drift 现象的诊断
  2. [项目 B 1 句话] 比如：复刻了 R1 的 GRPO 在 7B 模型上，写了 200-star blog
  3. [项目 C 1 句话] 比如：在 ACL 2024 发了一作 paper 关于 instruction tuning

我对 DeepSeek 的 Agent capability 方向特别感兴趣——
特别是 R1 paper 的 GRPO 和 emergent reasoning 'aha moment'，
非常希望能在 reasoning + tool use 这个 frontier 一起 push。

今天希望能多了解 team 当前 most exciting research thread，
也希望证明我能成为你们的 peer。"
```

**signal**：
- 3 个 specific 项目（避免泛泛）
- 引用 DeepSeek 具体 paper / figure
- "peer" 词——明确 positioning

### 二、各类题型临场策略

#### 1·数学深问题（[05 文档]）

```
1. 第一反应（5 秒）：心里走"公式 → 直觉 → trade-off"3 步
2. 复述题目（10 秒）："您问的是 PPO 的 clipped objective 推导对吧？"
3. 边写边讲（3~5 分钟）：白板/iPad 上推
4. 主动提 trade-off（30 秒）："这里 epsilon 选 0.2 vs 0.1 trade-off 是..."
5. 收尾加实战（15 秒）："我自己在 7B 上跑过 epsilon=0.1 比 0.2 更稳"
```

**关键**：**不要直接给答案——展示推理过程**。

#### 2·Paper 讨论（[07 文档]）

用 4-part 模板：WHAT → WHY → CRITIQUE → ACTION。

**关键追问处理**：
- "具体 figure N 想说什么"：必须真记得；不记得直接说 "我重点看 figure 1+方法部分，figure N 我没细看"
- "你能 derive main equation 吗"：能则 derive；不能则 "我记得直觉是 X，公式具体推导我需要查"——别 bluff

#### 3·Open Research 题（[09 文档]）

7-step framework：CLARIFY → SCOPE → HYPOTHESIS → EXPERIMENT → EXPECTED → FAILURE MODE → NEXT STEP

**避免雷区**：
- ❌ 直接 jump to solution
- ❌ 只给 1 个 hypothesis
- ❌ 没考虑 failure mode

**关键 phrase**：
- "我看到至少 3 个 sub-problem"
- "Hypothesis 1 我倾向最 likely，因为..."
- "Minimal yet decisive ablation 我会..."
- "Failure mode 我最担心 confound 是..."

#### 4·Coding 手撕（[08 文档]）

5-step：CLARIFY → SKETCH → IMPLEMENT → DEBUG → EXTEND

**关键 phrase**：
- "我假设 shape 是 (B, T)，对吗？"
- "我先 sketch plan 在 comment 里"
- "这一行是关键 trick——advantage normalization 防止..."
- "Production 我会加 mixed precision / mask"

#### 5·行为题（[10 文档]）

STAR-C：S → T → A → R → **Critical reflection**

**关键 phrase**：
- "复盘有 3 个 caveat 我事后才意识到..."
- "如果重做，我最想改..."
- "学到 meta-lesson 是..."

### 三、卡壳话术（万能 5 句）

```
1. "让我想 30 秒，我想 collect 思路。"

2. "我现在不确定 [X]——可以让我 walk through 我的 reasoning，
    您可以纠正我吗？"

3. "我 30 分钟内大概答 X；如果给我 1 周深思考，我会..."

4. "这个问题我读过相关 paper 但具体数字不记得——
    let me give 直觉，您可以 fact check。"

5. "我可以 think out loud 吗？我现在的 partial 想法是..."
```

**绝对不要**：
- ❌ 长时间沉默（>30 秒）
- ❌ 直接说"不知道"然后 next
- ❌ 编造 paper 名 / 编造数字
- ❌ 装懂；研究员闻得出 BS

### 四、面试中"加分动作"5 个

1. **主动反向连接**："您之前提的 X，让我想到 [related paper Y]——这两者是 same insight 吗？"
2. **calibrated confidence**："我 70% 确定 X，30% 确定 Y——主要 uncertainty 在 Z"
3. **提 alternative**："这是一种解法；另一种是 X，trade-off 是 Y"
4. **承认 limitation**："我的回答有 2 个 caveat——caveat 1...caveat 2..."
5. **主动 surface idea**："顺便提一下，我之前在想 [related idea]，可能值得一聊"

### 五、最后 5 分钟反问

```
1. (Research thread) "你们当前 research 最 exciting 的 ongoing thread 是？"
2. (My fit) "我加入后前 90 天 most valuable 的 contribution 是？"
3. (Vision) "R1 之后下一个 alignment / capability frontier 你认为是？"
4. (Personal) "你个人在 DeepSeek 工作 1 年里最 unexpected 的 surprise 是？"

收尾：
"今天聊得很 inspired。我之前还没问的 X / Y / Z，
我整理后可以邮件再 follow up 吗？"
```

## PART C·T+0 ~ T+14 复盘 + 后续

### T+0 当晚（30 分钟内 必做）

**1. 立刻 dump memory** (15 min)：

```markdown
# Interview Dump - YYYY-MM-DD

## Interviewers (姓名 / role / 印象)

## Questions Asked (顺序 + 我答的 summary)
1. Q: ...
   A: ...
   Self-rating: A/B/C
2. ...

## My Bombs (答不好的 / 卡壳的 / 后悔的)
- Q3 "PPO epsilon trade-off"：我没提 LLM 场景特殊性，扣分
- ...

## Their Signal (面试官给我的 hints)
- "你提到的 RM drift 我们也有过类似 finding" → 拉近 peer
- "我们 H100 比较紧" → resource constraint signal
- ...

## My Follow-up Ideas (我事后想到的更好答案)
- Q3 我应该说 "0.2 在 LLM 上易 reward hack，因为 vocab size 大..."
- ...

## My Next-Round Concerns
- 数学题 OK
- 系统设计可能弱
- ...
```

**2. 暂时不评分**：让大脑沉淀 12h，明天再 calibrate。

### T+1 (24h 内 必做)

**1. 发感谢邮件 (HR + senior interviewer)** — 1 封 / 人

模板：

```
Subject: 感谢今天面试 - [JD 名 - 你的名字]

[Hi 面试官名],

非常感谢您今天和我深入讨论了 [PPO clipped objective / R1 GRPO / our research direction 等具体话题]。
您提到 "[他们一句 specific 见解]" 给我很多启发——我后来回想 [related idea]。

特别地，您问的 [某具体题] 我事后又想了一下：
[1~2 句 better answer 或 follow-up 思考]。

也很期待 next round。如果您觉得我之前哪些回答还不充分，
非常欢迎您 follow up 让我补充。

再次感谢，
[你的名字]
```

**signal**：
- 引用 specific 话题（证明真在听）
- Follow-up better answer（demonstrate continued thinking）
- 邀请 negative feedback（mature mindset）

**2. 项目 V0.2 commit (重要 differentiator)**：

在 mini RLHF 项目 commit 1 个 V0.2，commit message：

```
Add [improvement related to interview question]
Inspired by question from DeepSeek interview about [topic].
```

发邮件附上 GitHub link 给面试官。

**这一动作的价值**：99% candidate 面试结束就停；你在 24h 内 ship V0.2 == "我 take 这个 conversation seriously" 极强 signal。

### T+2 ~ T+7 等待期

**等待 next round 前 7 天**：

- T+2: 第一轮完整复盘（看自己录音 / 笔记）
- T+3: 针对弱点补：如系统设计弱 → 看 [05 文档] / 如 paper 弱 → 再读 1 篇
- T+4: 准备 next round 可能 deeper 问题
- T+5: mini 项目 V0.3（如时间）
- T+6: rest
- T+7: 等回复 / 必要时 follow up HR

**Follow up HR 模板** (T+7 还无回复)：

```
Hi [HR],

想 follow up [日期] 面试的进展。
我对 DeepSeek 兴趣浓厚 —— 也很期待 next steps。

如有 update 麻烦告知；如需要 additional material 也欢迎告诉我。

谢谢！
```

### T+0 复盘 framework（重要题型）

针对每个"考砸"的题，按下面 5 列复盘：

| 题 | 我的答 | 理想答 | Gap 原因 | 行动 |
| --- | --- | --- | --- | --- |
| PPO epsilon | "0.2 / 0.1" | + LLM 场景 + 我跑过的数字 | 没准备到 LLM 特定 | 加到 ready phrase 库 |
| ... | ... | ... | ... | ... |

**3-轮迭代**：每个题至少 3 次 review (1d / 7d / 30d) 直到 muscle memory。

### Offer / Reject 应对

**若 offer**：
- 不要 100% rush accept；要 talk team / 上 Glassdoor
- Negotiate package 时 specific 数字（base / sign-on / equity）
- 30~60 days notice 给上家

**若 reject**：
- 礼貌回复 + 请求 feedback (50% 概率 HR 会给)
- 复盘 feedback → 改 [00 文档] strategy → 准备下 1 家
- 把 mini 项目继续推进 — 1~2 month 内 mature 可以 reapply

### 长期价值

**这两周的真正 ROI 不只是这个 offer——是**：
- 你 RL 数学功底升一档
- 你有了 GitHub mini 项目展示
- 你 paper 阅读速度 + critical thinking 升级
- 你下次任何 LLM/RL researcher 面试都 8 折准备时间

**这些都是 transferable + permanent。即使这次 fail，下次必 stronger。**

## 最后忠告

> **面试不是 single-shot performance——是 process。**
>
> - 准备 process 的质量 >> 临场发挥
> - 复盘 process 的诚实 >> "我 perfect 了"
> - Follow-up process 的 mindset >> "等结果"

> **研究员 hiring loop 通常 4~6 周；你的 process > 别人，offer 才会 trump 别人的 hype。**
