## 第二部分：临场策略（面试中）

### 一、面试前 30 分钟

```
Checklist:
- [ ] 简历 + 作品集 GitHub 链接 + 1 页关键 summary 打开
- [ ] 提前 15 分钟登 Zoom / 会议系统
- [ ] 桌面整洁: 笔, 白纸, 水
- [ ] 关 Slack / 微信通知
- [ ] 试一次 webcam + mic + 屏幕共享
- [ ] 喝水 + 深呼吸 5 次
```

### 二、Self-intro (1 分钟)

模板：

```
"Hi I'm [名字]. 

我目前在 [机构 / 公司] 做 [岗位], 主要研究 [方向, 如 "LLM alignment with focus on preference learning"].

最近 6~12 个月最值得一提的工作是 [一句话 + 一个数字结果], 这个项目让我对 [DeepSeek 相关方向] 产生了兴趣.

我特别 excited DeepSeek 这个岗位是因为 [1~2 sentence], 也希望今天能多和你讨论."
```

**60 秒——多了 boring 少了 vague**。

### 三、面试中的"节奏控制"

#### 算法手撕

```
1. 听完题, 1 句话 paraphrase 确认理解
   "OK 所以你想让我实现 X, 输入 Y, 输出 Z, 对吗?"

2. 5 秒思考 → 讲思路（不上来就写）
   "我先讲思路: 大致分三步 ...你觉得方向对吗?"

3. 写代码 + think aloud
   "我先写 forward, 再写 backward, 注意这里 shape ..."

4. 写完自己 review 一遍
   "Let me trace through with a small example ..."

5. 主动指出 edge case
   "注意 batch_size=1 时这里要 ...另外 padding mask ..."

6. 主动讨论 trade-off / optimization
   "如果要 efficiency, 可以 ..."
```

#### Paper 讨论

```
1. 用 [08 文档] 100 秒模板
2. 不超 2 分钟独白, 留 hook
3. 等面试官追问, 不要 dump 所有细节
4. 追问到不会的, 大方说: 
   "这一层我没读到——我会按 X 推测应该是 Y, 你能 confirm 吗?"
```

#### 行为面

```
1. STAR 严格执行
2. 不超过 90 秒 / 题
3. 量化结果——至少一个数字
4. 失败题选真失败 + lessons
5. 不抱怨, 不甩锅
```

#### 反问环节

```
1. 至少准备 3 个问题
2. 听完答案追问 1 次
3. 结尾感谢 + 重申 excited
```

### 四、临场遇到不会的题

**绝对不要**：
- 沉默 30 秒以上
- 装会 / 编答案
- 自暴自弃 "我不行"

**应该**：
```
1. 先 paraphrase 确认听懂

2. Think aloud 暴露思路:
   "我没直接做过 X, 但我知道相关的 Y——可能 Z 思路适用..."

3. 主动 ask for hint:
   "我能问一个 clarifying question 吗——这个 X 是 ... 还是 ...?"

4. 给一个 partial answer + 标记 boundary:
   "Part A 我清楚, 是 ...; Part B 我不确定, 我会查 ..."

5. 结束时:
   "这一题暴露了我对 X 的知识缺口, 我回去会补 ..."
```

**面试官重视的是 thinking process, 不是 perfect answer.**

### 五、面试中的"情绪管理"

```
紧张时:
- 深呼吸 3 次
- 拿水杯, 喝一口（合理停顿）
- 不要让紧张外溢——节奏放慢比加快好

被尖锐挑战时:
- 不要立刻 defensive
- "这是个好问题——让我想想"（赢得 5 秒）
- "你说得对, 我之前没考虑到这一点, 如果是这样的话 ..."

被 push 时（"为什么你认为 X 而不是 Y"）:
- 不要立刻 flip
- 站住自己理由再 listen
- "我的 reasoning 是 ..., 但你 propose Y 是因为 ...? 让我 reconsider"
```

## 五、附：常见问题 FAQ

#### Q：T-7 还没跑通 mini RL，该弃吗？
A：弃。**没跑通的项目不要拿出来——会成为 liability**。优先保证算法手撕 + paper 深读。

#### Q：英语不好怎么办？
A：1) DeepSeek 国内为主，中文够 2) 提前查清面试语言 3) 不会的英文术语可中英文混用

#### Q：被问到完全没接触过的方向（如 multimodal）怎么办？
A：诚实说"这方向我没深入做过"+ 给你的 transferable 经验（RL/对齐思维）+ "如果加入我会主动学"

#### Q：要不要把 mini RL repo 设公开？
A：T-3 前公开。面试当天可以发链接给面试官（在 chat 里）。

#### Q：被压力面（连续否定）怎么办？
A：不要 defensive。"你这个 push 让我重新思考——确实我的 reasoning 有 [具体] 漏洞。换个角度看 [...]"——展示能 receive feedback 是最大加分。

#### Q：反问环节问敏感问题（如 R1 是否团队主导 vs 个人天才）OK 吗？
A：不要。这类问题在 offer 后 1on1 再问。
