# 04 · 竞品深度分析（8 款主流 AI Chat）

> 这一份给每款产品 **5-part 拆解**：定位 + 数据 + UX 特色 + Take-away for DeepSeek + 我的 1 句话观点。**面试时按 30s/2min/5min 三档随时切换**。
>
> **强烈建议**：T-6 ~ T-5 每款产品**至少 2 小时真用** + 截图记笔记——second-hand 调研永远比不上 first-hand 体感。

## 一、海外 4 款

### Product #1·ChatGPT (OpenAI)

#### 1·定位
> 通用 AI 助手 + 平台（GPT Store、Custom GPT）。目标：**every consumer's first AI**。

#### 2·关键数据（2025-2026 估算）
- MAU：~5 亿（全球第一）
- 付费：Plus $20、Pro $200、Enterprise；付费用户 1500-2000 万
- 主用户：通用 / 学生 / 知识工作者
- 全球分布：北美 + 欧洲 + 印度 + 日韩为主

#### 3·UX 特色（5 个亮点）
1. **Voice Mode**：全屏 immersive 体验，对话感强，移动端杀手锏
2. **Canvas**：右侧 panel 直接编辑文本/代码，避免 chat 一闪而过
3. **GPT Store / Custom GPT**：UGC 长尾场景的 holder
4. **Memory**：跨对话记忆，可见 + 可控
5. **Operator / Atlas / Agent Mode**：浏览器 agent / Computer use

#### 4·Take-away for DeepSeek
- ❤️ **抄**：Voice Mode（移动端关键 retention 杠杆）
- ❤️ **抄**：Memory（让重度用户更粘）
- ⚠️ **不抄**：Custom GPT（UGC 平台门槛高、监管复杂）
- ⚠️ **不抄**：付费墙做太重（国内付费意愿弱）

#### 5·一句话
> "ChatGPT 是 'completeness moat'——单点都被超越，但完整度无对手。DeepSeek 不必抄全，只需在 reasoning + 深度场景做深。"

### Product #2·Claude (Anthropic)

#### 1·定位
> 高质量写作 + Coding + 深度 reasoning。目标：**knowledge worker's preferred AI**。

#### 2·关键数据
- MAU：~1-1.5 亿
- 付费：Pro $20、Max $100-200、Team；付费率高（产品质量+品牌）
- 主用户：开发者（占 30%+）、写作者、研究者
- 流量增长：2025 H2 后 Claude Sonnet 4.x 成开发者首选 → 流量暴涨

#### 3·UX 特色
1. **Artifacts**：右侧 panel render React/HTML/SVG/Mermaid，长输出体验革命
2. **Projects**：10M+ token 持久 context；上传文档 / 集成 GitHub
3. **MCP Tool 接入**：Claude Desktop 内置 MCP，能连 N+ 工具
4. **Computer Use**：截屏 + 鼠键控制，第一个工业级 GUI agent
5. **写作质量品味强**：style consistency + 文学性

#### 4·Take-away for DeepSeek
- ❤️ **抄**：Artifacts（reasoning 输出 / 代码 / 表格 都受益）
- ❤️ **抄**：Projects（重度用户 retention 神器）
- ❤️ **学**：写作质量的产品化（强 prompting + style guide）
- ⚠️ **观望**：Computer Use（技术上 DeepSeek 还差距大）

#### 5·一句话
> "Claude 是 'quality moat'——不追完整度，深耕 'good answers'。DeepSeek 应该走类似路线，重点是 reasoning 和 coding 这两个高价值场景。"

### Product #3·Gemini (Google)

#### 1·定位
> Google 生态原生 AI + 1M+ context + Multimodal。目标：**Google user 的 AI default**。

#### 2·关键数据
- MAU：难独立统计（捆绑 Workspace / Android / Search）
- 付费：Gemini Advanced $20、AI Pro $20-200
- 主用户：Workspace 用户、Android 用户、研究者

#### 3·UX 特色
1. **Deep Research**：长 horizon 自主研究 → 输出 detailed report
2. **NotebookLM**：上传文档 → 生成 audio podcast 对话（独创）
3. **1M+ token context**：超长文档处理
4. **Veo (video gen)**：行业领先视频生成
5. **Gemini Live**：实时多模态对话

#### 4·Take-away for DeepSeek
- ❤️ **学**：Deep Research（专业用户高价值场景）
- ❤️ **学**：NotebookLM 这种"非传统对话形态"
- ⚠️ **不抄**：捆绑生态（DeepSeek 没有这种 distribution）
- ⚠️ **观望**：1M context（成本高）

#### 5·一句话
> "Gemini 是 'distribution moat'——靠 Google 生态分发。DeepSeek 没有这种 luxury，但可以学 Deep Research / NotebookLM 这种 '非聊天形态' 提升 niche 价值。"

### Product #4·Perplexity

#### 1·定位
> AI-native 搜索 + Research 助手。目标：**replace Google search for serious queries**。

#### 2·关键数据
- MAU：~8000 万-1 亿
- 付费：Pro $20、Comet (浏览器) 新品
- 主用户：研究者、记者、专业用户

#### 3·UX 特色
1. **每段都有 citation**——可信度建立
2. **Pro Search**：multi-step research（类似 Deep Research）
3. **Spaces**：项目级 context
4. **Comet browser**：浏览器 agent
5. **Discover**：每日 AI 新闻流（content + AI 结合）

#### 4·Take-away for DeepSeek
- ❤️ **抄**：Citation 强化（DeepSeek 联网搜索可学）
- ❤️ **学**：Discover-like 内容流（提升 DAU 频次）
- ⚠️ **观望**：Comet browser（重投入，不是 DeepSeek 风格）

#### 5·一句话
> "Perplexity 是 'positioning moat'——专攻搜索这个场景做到极致。DeepSeek 可以借鉴 'citation + research mode' 强化可信度。"

## 二、国内 4 款

### Product #5·豆包 (字节跳动)

#### 1·定位
> 全场景 AI 助手（chat + 智能体 + 视频通话 + 播客 + 创作）。目标：**国民级 AI**。

#### 2·关键数据
- MAU：~3-4 亿（国内 No.1）
- 流量：投流 + 字节系导流（抖音、剪映、今日头条）
- 主用户：学生、年轻人、创作者
- 付费：探索中，主要靠免费 + 增值

#### 3·UX 特色
1. **极致的颜值与移动端体验** — UI 友好，新手 0 门槛
2. **视频通话**——一边视频一边对话，AR 风格
3. **AI 播客 / 智能体广场**——UGC 长尾场景
4. **桌面客户端 + 浏览器扩展**——多端覆盖
5. **国内最强 voice**——音色多 + 角色扮演

#### 4·Take-away for DeepSeek
- ❤️ **学**：移动端体验细节（icon / 动画 / 反馈）
- ❤️ **学**：智能体（降低长尾场景门槛）
- ❤️ **学**：voice + 视频通话（移动 retention）
- ⚠️ **不抄**：投流文化 / 红包文化（DeepSeek 文化不匹配）
- ⚠️ **不抄**：娱乐化（DeepSeek 用户偏理性）

#### 5·一句话
> "豆包是 'distribution + breadth moat'——字节生态分发 + 全场景覆盖。DeepSeek 不可能拼 distribution，必须靠 'quality + reasoning' 差异化。"

### Product #6·Kimi (月之暗面)

#### 1·定位
> 长 context + 知识工作 + 探索 reasoning。目标：**深度工作者的 AI**。

#### 2·关键数据
- MAU：~3000 万-5000 万
- 主用户：高度重叠 DeepSeek（学生 / 研究 / 知识工作者）
- 付费：Kimi+ / API
- 流量：曾投流较重，近期 pivoting

#### 3·UX 特色
1. **2M 字超长 context** —— 上传 N 个文档
2. **Kimi 探索版**：reasoning 模型可视化 + 5 步骤展示
3. **Kimi+ 智能体**：垂直场景 agent
4. **学术友好**：论文上传 + 解读
5. **PC 端体验佳**

#### 4·Take-away for DeepSeek
- ❤️ **学**：探索版的 reasoning 可视化方式（5 步骤 visual）
- ❤️ **学**：长文档上传体验
- ⚠️ **DeepSeek 比 Kimi 强的**：模型 reasoning 能力 + 开源 + 海外品牌——要强化这些差异

#### 5·一句话
> "Kimi 和 DeepSeek 是最直接竞品（用户重叠 70%+）。差异化关键是 reasoning 深度 + 模型开放性，DeepSeek 已经赢，要把这个优势 surface 出来。"

### Product #7·腾讯元宝

#### 1·定位
> 混元 + DeepSeek 双模型 + 微信生态分发。目标：**微信用户的 AI default**。

#### 2·关键数据
- MAU：~5000 万-1 亿（接入 DeepSeek 后暴涨）
- 主用户：泛微信用户 / 普通消费者
- 付费：免费 + 部分增值
- 流量：微信 / QQ / 腾讯系导流

#### 3·UX 特色
1. **接入 DeepSeek R1**——某种程度上是 DeepSeek 的"分发版"
2. **多模型切换**——用户可选混元 / R1 / V3
3. **微信生态深度**——可分享到聊天 / 朋友圈
4. **PC + App + 公众号 三端**

#### 4·Take-away for DeepSeek
- ⚠️ **威胁**：元宝靠微信分发 + DeepSeek 模型，相当于"抢 DeepSeek 用户"
- ❤️ **学**：分享到微信的路径（DeepSeek 自家也要做好）
- 💡 **思考**：DeepSeek 是否要"夺回"被元宝拦截的用户？或拥抱（接受 distribution partner 角色）？

#### 5·一句话
> "元宝是 'channel parasitism' —— 靠微信分发 + DeepSeek 引擎。这是 DeepSeek C 端面临的最 nuanced 竞品：既是用户来源也是用户截胡者。"

## 四、DeepSeek vs 8 款竞品的 "战略位置图"

> 二维定位 (X: 完整度 / Y: 深度品质)：

```
高品质质量 ↑
            │ Claude
            │   ●
            │
   DeepSeek ●           ●  ChatGPT
            │
            │           ●  Gemini
            │   ● Kimi
            │   ● Perplexity
            │
            │           ●  豆包
            │   ● 元宝
            │
   ────────────────────────────→ 完整度
                              高
```

> **DeepSeek 当前位置**：低完整度 + 高品质 reasoning。
> **战略选择**：(a) 守在 quality niche；(b) 提升完整度向 Claude 靠拢；(c) 双向都做。
> **推荐**：**优先提升完整度的关键 missing piece**（Voice / Canvas / Memory）+ **保持 reasoning quality 优势**——朝 Claude 位置移动，而不是朝豆包位置（无法对抗字节 distribution）。

## 五、可被面试问的 5 个竞品宏观问题

### Q1·"如果 DeepSeek 只能学一款产品，学谁？"
> "Claude——quality + Artifacts + Projects。理由：(1) 用户重叠最高（理性深度用户）；(2) 产品哲学最相似（quality > breadth）；(3) DeepSeek 还没有 Canvas/Projects 类功能，机会窗口大。"

### Q2·"DeepSeek 面对豆包的 distribution 怎么打？"
> "不正面打。豆包靠字节投流 + 全场景，DeepSeek 复制成本爆炸。差异化路径：(1) 锁定 quality / reasoning niche；(2) 利用海外品牌做差异化（豆包海外弱）；(3) 借开源 + 学术 PR 持续 inbound 流量。"

### Q3·"元宝接入 DeepSeek，DeepSeek 怎么看？"
> "双刃。Pro: 拓展 distribution；Con: 用户 stay in 元宝 → DeepSeek 自家 App 增长被拦截。最佳策略：(1) 自家 App 必须比元宝里的 DeepSeek 体验明显更好（差异化 feature）；(2) 不限制元宝接入（流量带来 brand awareness）；(3) 在元宝接入时争取 'powered by DeepSeek' 强 brand exposure。"

### Q4·"Voice 是 DeepSeek 短板，要补吗？"
> "中期要补，短期不必。理由：(1) Voice 对移动 retention 是大 lever（ChatGPT/豆包 都证明）；(2) 但 voice 自研成本高，DeepSeek 模型团队 priority 应放 reasoning / coding；(3) 折中：先用合作伙伴方案补齐（如 minimax voice），未来再自研。"

### Q5·"如果 OpenAI 突然开源 o3，DeepSeek 怎么办？"
> "开源利好 DeepSeek 整体生态（开源 vs 闭源对抗），但短期挑战：(1) DeepSeek 模型品牌优势弱化（不再是唯一开源 SOTA）；(2) 倒逼 DeepSeek 加快 V4 节奏；(3) 转向'开源 + 中文 native + reasoning' 三重差异化。最坏情况：DeepSeek 模型不再是 USP，但产品 + 品牌可继续。"

## 六、推荐手动深度体验 SOP（再贴一次）

每款 **2-3 小时**，5 步：

1. **冷启动**：注册 / 引导（看 onboarding flow）
2. **基础对话**：日常 Q&A 5 个 turn
3. **复杂任务**：让它（a）写一篇 1000 字报告；（b）解 1 道数学题；（c）翻译长文
4. **多模态**：如果有 voice / image，体验
5. **小众功能**：探索一个被埋藏的功能

**每款写 3+3+1**：
- 3 个 wow 点
- 3 个 卡点
- 1 个 DeepSeek 可偷的点

## 七、一句应试钥匙

> **面试官问 "你怎么看 X 产品" 是 trap——是测试你"有没有真用"。**

> 高分答 = "我连续用了 X 3 天，最 surprised 的是 [极具体 UX detail]，最 frustrated 的是 [极具体 friction]，DeepSeek 可以学 [极具体 idea]。"

> 低分答 = "我读过 X 的文章，他们用户量是 Y..."

> **first-hand 体感 = 这个岗位的硬通货。**
