# 11 · 面试前-中-后全流程行动手册（C 端 PM 版）

> 把 "面试前 / 中 / 后" 3 阶段合在 1 份的实战手册。C 端 PM 准备周期相对短（不需硬啃数学），但 **DeepSeek 深度体验 + 8 款竞品体验 + 1 个 PRD** 是 critical path，不能跳过。推荐 T-7 ~ T+0。
>
> 3 部分：
> - **PART A·T-7 ~ T-0**：7 天 daily 行动 + 项目里程碑
> - **PART B·面试当天**：开场 60 秒 + 各题型临场策略 + 卡壳话术
> - **PART C·T+0 ~ T+14**：复盘 + 感谢邮件 + V0.2 commit + 后续

---

### T-7 启动·"DeepSeek 7 天深度使用"

> **这是整个准备的 cornerstone**——没有 first-hand 体感面试时露馅。

#### Day 1-7 每天 30 分钟 DeepSeek 体验

每天选不同场景 + 记笔记：

| Day | 场景 | 笔记角度 |
| --- | --- | --- |
| Day 1 | 解 1 道数学题（开 R1）| Reasoning UX 体验 |
| Day 2 | 让它写一篇 blog post | 写作质量 / 输出格式 |
| Day 3 | 上传 PDF 让它分析 | 文件处理 |
| Day 4 | 联网搜索 query | Citation / source quality |
| Day 5 | 长对话（10+ turn）| Context 持久性 |
| Day 6 | 移动 App 同样 task | Cross-platform 一致性 |
| Day 7 | "妈妈级"普通 query（"帮我写个邮件"）| 普通用户体验 |

**笔记 format**（每条 30 秒）：

```
- 场景：X
- 体验细节：Y
- friction 点 / wow 点：Z
- 改进 idea：W
```

**目标**：T-3 之前累积 20+ friction 点。

## PART B·面试当天·临场策略

### 一、开场 60 秒模板（必背）

> *面试官说 "请简单自我介绍" 时用。*

```
"您好，我是 [姓名]。

我目前在 [公司/职责] 做 [PM 方向] —— 相关 highlight 3 件事：
  1. [项目 A 1 句话] 比如：lead X 产品 0→1 ship，6 月做到 Y 万 MAU
  2. [项目 B 1 句话] 比如：搭建增长指标体系将 Day-30 留存 X% → Y%
  3. [项目 C 1 句话] 比如：在 [上家] 主导 AI 功能集成，3 月内 ship 5 个 capability features

我对 DeepSeek C 端方向特别感兴趣 ——
特别是 R1 reasoning 怎么 surface 给普通用户、Memory / Projects 怎么提升 retention，
我自己深度用了 1 周整理了 20 个 friction 点 + 3 个改进提案 + 1 个完整 PRD，
今天希望能和团队 deeper exchange 关于产品 vision。"
```

**signal**：
- 3 个 specific 项目（避免泛泛）
- 引用 DeepSeek specific feature （证明真用过）
- 主动提带了 critique deck —— hooks 面试官追问

### 二、各题型临场策略

#### 1·用户洞察题（[05 文档]）

3-step：SCENE → INSIGHT → ACTION

**关键 phrase**：
- "上周我用 DeepSeek 时遇到..."
- "我观察到 3 个洞察..."
- "按 ICE 排优先级..."

#### 2·AI 能力转化题（[06 文档]）

4-step：CAPABILITY → SCENARIO → FEATURE → RISK + METRIC

**关键 phrase**：
- "这个 capability 本质是 X (技术描述)..."
- "boundary 是..."
- "对应 3 类用户场景..."
- "MVP 设计..."

#### 3·增长 / 指标题（[07 文档]）

4-step：拆问题 → 拆指标 → 行动 → 风险

**关键 phrase**：
- "北极星指标我建议..."
- "拆 input：留存 = X × Y × Z..."
- "护栏指标不能动..."

#### 4·A/B 题（[08 文档]）

按 6-element spec：hypothesis / variant / population / metric / sample / decision

**关键 phrase**：
- "我先确认 hypothesis..."
- "primary metric ...; guardrail 是..."
- "样本量我估算 N..."
- "决策 framework..."

#### 5·行为题（[09 文档]）

STAR-C：S → T → A → R → **Critical reflection**

**关键 phrase**：
- "复盘有 N 个 caveat..."
- "如重做我最想改..."
- "meta-lesson 是..."

### 三、卡壳话术（万能 5 句）

```
1. "让我想 30 秒 collect 思路"

2. "我现在不确定 X——可以让我 walk through reasoning 您纠正吗"

3. "30 分钟答 X；给我 1 周深思考会 Y"

4. "这个我读过 paper 但具体数字不记得——给您直觉 + 您可 fact check"

5. "可以 think out loud 吗？现在 partial 想法是..."
```

**绝对不要**：
- ❌ 长时间沉默 (>30s)
- ❌ "不知道" then next
- ❌ 编造数字 / 产品 detail
- ❌ 装懂——PM 面试官闻得出 BS

### 四、加分动作 5 个

1. **主动 reverse connect**："您前面提的 X，让我想到 [对比产品 Y]"
2. **Calibrated confidence**："70% 确定 X，30% 是 Y"
3. **提 alternative**："这是一种解；另一种是 X，trade-off 是..."
4. **承认 limitation**："这个回答 2 个 caveat..."
5. **主动 surface idea**："顺便提，我之前在想 [related idea]..."

### 五、最后 5 分钟反问

```
1. (Strategy) "DeepSeek C 端不投流的坚持核心 conviction 是？"
2. (My fit) "30 天 ship 最 valuable 是什么？"
3. (Product) "DeepSeek 当前最大 product 痛点你认为是？我自己观察到 X，您看怎么样？"
4. (Personal) "您 1 年里 most unexpected surprise 是？"

收尾：
"今天聊得 inspired。其他 follow up 我整理后邮件 OK 吗？"
```

## PART C·T+0 ~ T+14·复盘 + 后续

### T+0 当晚（30 分钟必做）

#### 1·立刻 dump memory（15 分钟）

```markdown
# Interview Dump - YYYY-MM-DD

## Interviewers (姓名 / role / 印象)

## Questions Asked
1. Q: ...
   A: ...
   Self-rating: A/B/C
2. ...

## My Bombs (答不好的)
- Q3: ...
  Better answer (now I think): ...

## Their Signal
- "..." → 拉近 peer 信号
- "..." → resource constraint 信号

## My Follow-ups Ideas
- Q3 应该说...

## Next-Round Concerns
- 数据题 OK
- 系统设计可能弱
```

#### 2·暂不评分

让脑子沉淀 12h，明天 calibrate。

### T+1 (24h 内必做)

#### 1·感谢邮件（每个 senior interviewer 1 封）

模板：

```
Subject: 感谢面试 - DeepSeek C 端 PM - [你的名字]

Hi [面试官名],

非常感谢今天和我深入讨论了 [reasoning UX / 增长 / 竞品 等 specific 话题]。
您提到 "[他们具体一句话]" 让我后来回想 [related idea]，整理了 follow-up：

[1-2 句 better answer 或 follow-up 思考]

我也把今天提到的 DeepSeek 改进 [feature X] PRD draft 发您 review，
非常欢迎您 follow up 让我补充。

期待 next steps。
[你的名字]
[PRD 附件]
```

#### 2·PRD V1.1 commit（重要 differentiator）

针对面试中讨论 update PRD一版：

- 加入面试官的 perspective / 反对意见
- 调整 metric / A/B design 反映 critique
- Email 附发

> 这一动作的价值：99% candidate 面试结束就停；T+1 ship V1.1 == "我 take 这个 conversation seriously" 极强 signal。

| --- | --- | --- | --- | --- |

**3 轮迭代**：1d / 7d / 30d review。

### Offer / Reject 应对

#### Offer

- 不 rush accept；和团队 1-on-1 + 上 Glassdoor
- Negotiate package specific（base / sign-on / equity）
- Notice period 30-60 day

#### Reject

- 礼貌回复 + 请求 feedback（50% HR 会给）
- 复盘改 [00 文档] strategy
- PRD 继续推进——1-2 月 mature 可 reapply

### 长期价值

这 7 天的真正 ROI 不只是 offer——是：
- AI 产品方法论体系化
- 8 款竞品体感成长
- DeepSeek 这家公司洞察
- 写 PRD muscle memory 升级
- 下次 AI PM 面试 8 折准备时间

**Transferable + permanent。即使这次 fail，下次必 stronger。**

## 最后忠告

> **C 端 PM 面试不是 trivia 测试——是 'do you think like a product person AND fit DeepSeek culture' 测试。**

> 前者靠方法论 + 实践经验。
> 后者靠：
> - 不诋毁竞品（赞美字节也提自己 differentiator）
> - 不浮夸数字（calibrated confidence）
> - 不和算法争 authority（'enabler' not 'competitor'）
> - 不焦虑 DAU race（quality > vanity）

> 这种 'mature + grounded + user-first' 的 mindset，是 DeepSeek 这种 'quiet excellence' 公司想要的 C 端 PM。
