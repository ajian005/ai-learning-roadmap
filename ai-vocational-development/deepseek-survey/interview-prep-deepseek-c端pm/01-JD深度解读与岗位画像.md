# 01 · JD 深度解读与岗位画像（DeepSeek C 端 PM 版）

> 目标：把"DeepSeek C 端产品经理" JD 还原为"团队真实需求"，并和已有 4 个 DeepSeek 岗位（Harness PM / Harness Eng / 全栈 Eng / Researcher）做差异对照。

---

## 一、JD 全文（DeepSeek C 端产品经理）

> **岗位职责**
> 1. 主导 DeepSeek C 端产品的功能规划与体验设计，敏锐洞察用户需求，不断推动产品创新和体验优化；
> 2. 关注大模型等相关技术发展，理解 AI 能力边界，将 AI 能力转化为可落地的产品功能；
> 3. 定位并拆解问题，降低用户使用成本，关注用户增长，提升使用深度、粘性和转化；
> 4. 制定并跟踪产品关键指标，分析行业趋势、用户行为和反馈，决策产品优化和迭代方向；
> 5. 高效协同算法、模型、研发团队等，保证产品功能的高质量落地。
>
> **职位要求**
> 1. 具备极强的共情力和同理心，对产品有品味有追求，用户体验敏感度高，主动发现问题，做有用的产品；
> 2. AI 产品深度用户，持续关注创新产品与行业趋势，喜欢动手研究具体的产品；
> 3. 能进行复杂项目的规划、排期、落地与迭代；
> 4. 深度理解国内外大模型技术，视野广阔，同时能聚焦阶段性目标，产品逻辑清晰；
> 5. 认真细致，责任心和自驱力强，具备跨团队的执行力和沟通协作能力。

## 二、岗位画像（一段话浓缩）

> **你是 DeepSeek C 端（网页 / App / 移动端）产品的主理人——主战场是把 DeepSeek 世界级模型能力 (V3/R1/未来 V4) 转化成亿级普通用户感知到的产品价值；竞品是国内豆包/Kimi/腾讯元宝/通义/智谱清言/MiniMax 海螺，海外 ChatGPT/Claude/Gemini/Perplexity。你要做的是用户洞察 + 功能设计 + 增长 + 数据驱动迭代 + 跨团队推动。**

**这个岗位 = ChatGPT consumer PM × 字节豆包 PM × 中国市场 AI 助手 PM 的合成版。**

## 四、主要职责逐句拆解

### 职责 1：主导 DeepSeek C 端产品的功能规划与体验设计，敏锐洞察用户需求，不断推动产品创新和体验优化

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 你不是"维护已有功能"——你是"决定 deepseek.com / App 下一个 12 个月长什么样"。从 user research → 功能定义 → 设计稿协作 → 上线 → 迭代 |
| **隐藏要求** | "**敏锐洞察**"= 不能只看数据；要会做用户访谈、看真实 trajectory、自己重度使用 |
| **考察点** | "你最近 3 个月在哪些 AI 产品里发现了 nontrivial 用户洞察？" |

### 职责 2：关注大模型等相关技术发展，理解 AI 能力边界，将 AI 能力转化为可落地的产品功能

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 模型团队丢一个 "我们 reasoning 能力升了 10%" 给你 → 你想 "这能转成什么用户感知的功能？" → 产出 PRD |
| **隐藏要求** | "**AI 能力边界**"——你要清楚 LLM 什么干不了，避免产品过度承诺导致用户失望 |
| **考察点** | "什么任务 LLM 做不了？" / "R1 reasoning 模型应该怎么 expose 给普通用户？" / "幻觉怎么 product-level 缓解？" |

### 职责 3：定位并拆解问题，降低用户使用成本，关注用户增长，提升使用深度、粘性和转化

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 看到 "Day-7 留存 30%" → 拆 (新用户 vs 老用户 / iOS vs Android / 不同场景) → 找 root cause → 设计 intervention → A/B test → 迭代 |
| **隐藏要求** | "**使用深度**" 是关键词——不仅 DAU，是单用户对话数、token 数、付费转化等 |
| **考察点** | "你怎么设计北极星指标？" / "新用户 onboarding 怎么优化？" / "DeepSeek 现在 Day-30 留存 30%，怎么提到 50%？" |

### 职责 4：制定并跟踪产品关键指标，分析行业趋势、用户行为和反馈，决策产品优化和迭代方向

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | 你不是 ad-hoc 看指标 - 你建立**指标体系**（北极星 + 二级 + 护栏）+ 周报 + 季度 review |
| **隐藏要求** | "**决策**"——数据告诉你 what，你要给出 why + so what + next |
| **考察点** | "你怎么设计指标 dashboard？" / "数据矛盾时怎么判断？" |

### 职责 5：高效协同算法、模型、研发团队等，保证产品功能的高质量落地

| 维度 | 解读 |
| --- | --- |
| **真实活儿** | DeepSeek 团队组成偏研究员/工程师；PM 要 driving without authority |
| **隐藏要求** | DeepSeek 文化是 "无 KPI 无赛马"；PM 不能靠权力推事，靠 buy-in |
| **考察点** | "和算法 / 研究员分歧怎么办？" / "你怎么把研究员的语言翻译给设计师？" |

## 六、JD 里没写但一定会考的"暗线"

### 暗线 1·"为什么 DeepSeek 而不是字节豆包 / 阿里通义 / 腾讯元宝"

**应对**：准备一段 1 分钟 pitch——"DeepSeek 是中国唯一'模型先于产品'的 frontier 公司，C 端 PM 在这里可以做'用模型能力创造用户感知价值'而不是'用运营套路抢用户'。这种产品哲学吸引我。"

### 暗线 2·"DeepSeek C 端目前增长瓶颈是什么"

**应对**：T-7 ~ T-5 看公开数据 (SimilarWeb / QuestMobile / 七麦数据)；准备 3 个 specific 瓶颈假设（如 "App 留存低于豆包，可能因为新功能创新弱" / "网页流量周期性强，可能因为没有 push 机制"）。

### 暗线 3·"如何在'不烧钱不投流'的约束下做增长"

DeepSeek 公开 statement 反复强调不做重运营。C 端 PM 必须能在这个约束下思考。

**应对**：准备 "产品力驱动增长" 的具体打法——更好新手体验、社区分享、SEO 长尾、技术 PR 引流、口碑放大。

### 暗线 4·"如何处理'模型能力领先但产品体验普通'的现实"

DeepSeek R1 数学能力世界级，但很多用户感知不到——这是真实 gap。

**应对**：准备 "能力 surface 化" 的产品思路——把 reasoning 展示出来、加 thinking visualization、做 reasoning 历史回顾、scenario showcase 等。

### 暗线 5·"你能不能讲清 LLM 概念给你妈"

C 端 PM 必须能 translate 技术给普通人。

**应对**：准备 1 段 60 秒的"妈妈能懂的 LLM/Reasoning/Agent 解释"。

## 八、一句应试钥匙

> **每个答案问自己一句："这个回答能不能体现『我是 DeepSeek 重度用户』+『我有共情』+『我懂 AI 能力边界』+『我能拿数据说话』四件事？"**
