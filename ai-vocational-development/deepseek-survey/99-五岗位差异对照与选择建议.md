## 二、自检：你最 fit 哪个？

按下面 5 个维度自评（1-10 分），看你在哪个 cluster 最强。

### 维度 1·学术背景
- 顶校博士 + 顶会论文 → **Researcher** 首选
- 顶校硕士 + ML 工业 2+ 年 → **Researcher / Eng** 都行
- 知名本科 + 工业经验 → **C端 PM / Harness PM / Eng** 都可
- 转型背景（其他 domain 转 AI）→ **C端 PM** 最容易切入

### 维度 2·主要技能
- **数学 + RL 实战** strongest → Researcher
- **Coding (Python + Rust + 系统) + LLM** strongest → Harness Eng / 全栈 Eng
- **工具产品 + 开发者经验** strongest → Harness PM
- **C端产品 + 增长 + AI UX** strongest → C端 PM
- **跨 stack (前后端 + AI 集成)** strongest → 全栈 Eng

### 维度 3·驱动力
- "想做出 SOTA paper" → Researcher
- "想 ship 工具改变 developer 工作流" → Harness PM / Eng
- "想做亿级 C 端产品影响普通人" → C端 PM
- "想搭建支持 AI 团队 productivity 的 infra" → 全栈 Eng

### 维度 4·风险偏好
- 高风险高回报 (research bet 有 fail 风险) → Researcher
- 中风险（PM 决策 trade-off） → C端 PM / Harness PM
- 低风险（工程 ship 较确定）→ Harness Eng / 全栈 Eng

### 维度 5·成长方向
- 想成 ML researcher / scientist → Researcher
- 想成 product leader → C端 / Harness PM
- 想成 senior IC / staff engineer → Harness Eng / 全栈 Eng

## 四、跨岗位申请策略

### 策略 1·"双投" 是否可行？

DeepSeek 通常允许（甚至鼓励）跨岗投。但要注意：

| 跨投组合 | 是否合理 | 注意点 |
| --- | --- | --- |
| C端 PM + Harness PM | ✅ 可双投 | 表明对 PM 整体兴趣，两份不同 PRD |
| Harness Eng + 全栈 Eng | ✅ 可双投 | 工程能力 overlap |
| Researcher + Harness Eng | ⚠️ 注意 | 表明研究偏 production；强调 0→1 + ship |
| Researcher + 全栈 Eng | ⚠️ 不推荐 | 信号 confusing |
| PM + Eng / Researcher | ❌ 不推荐 | trajectory mismatch；hiring manager 困惑 |

### 策略 2·主投 / 副投 priority

- **主投**：你 ICE (Impact × Confidence × Easy) 最高的岗
- **副投**：你 ok 的次选；备用

通常 1 主 + 1 副 已足。

## 六、薪资 / Package 谈判 quick tip

| 岗位 | 初级（应届/2年内）| 中级（3-6年）| 高级（7+年）|

| 全栈 Eng | 40-60W | 60-100W | 100-150W |

### 谈判 tip

- DeepSeek 不发奖金 / 红包文化 → base 占绝对大头
- Equity 稀少（DeepSeek 没上市，未来上市概率中等）
- Sign-on bonus 偶尔有，金额不大
- **不要 negotiate 基于"别处 offer X" 除非真有书面 offer**（DeepSeek 不接受虚假 leverage）

## 八、如果你的 case 是"五岗都符合"

> **这是高质量 candidate 的 luxury problem**。

### 框架·按"长期 trajectory" 决定

```
5 年后你想成为什么？

A) "我想成为 OpenAI / Anthropic 级别的 alignment researcher"
   → Researcher 首选

B) "我想成为 AI 时代的 product visionary（Sam Altman 式）"
   → C端 PM 或 Harness PM 首选

C) "我想成为 staff engineer / 技术合伙人级别 IC"
   → Harness Eng / 全栈 Eng

D) "我想做'技术 + 产品'双栖（如 Dario / Mira Murati）"
   → 先 Researcher 或 Eng 入，3-5 年后切 PM
```

### 框架·按"当前能力 leverage" 决定

```
你最 strong 的 skill 是什么？

1·RL/PyTorch 实操 → Researcher
2·Coding + LLM API → Eng
3·产品判断 + 用户洞察 → PM
4·跨 stack 整合 → 全栈 Eng
```

把你最强的牌打出去——别浪费 transferable 经验。

## 九、最后忠告

> **5 个岗位都是高 bar，但 bar 的类型不同。**
>
> - **Researcher**：bar 是"你能不能成为 peer 不是 candidate"——研究品味、数学深度、paper 一作
> - **C端 PM**：bar 是"你能不能 think like a true product person"——用户共情、第一手体感、PRD 实操
> - **Harness PM**：bar 同 C端 PM 但 audience 换成开发者；要"自己天天用 Cursor / Claude Code"
> - **Harness Eng / 全栈 Eng**：bar 是"你能不能 ship production-quality LLM 产品"——code + 系统设计 + LLM 实战
>
> **DeepSeek 共通 bar**：
> - 自驱（无 KPI 文化）
> - heads-down（少 meeting 多 deep work）
> - quiet excellence（不重 self-promotion）
> - long-term thinking（不追短期 metric）
>
> 选岗位时问自己 3 个问题：
> 1. 我的 strongest 能力是哪类？
> 2. 我的 5 年职业 trajectory 是什么？
> 3. 我对 DeepSeek 这种公司文化是否真心 ok？

> **3 个 yes → go for it。1 个 no → 重新思考。**

> 祝你拿 offer。
